# 跨平台开发规范（Web / Desktop）

## 背景

本项目的 Web 版和 Electron 桌面版共享同一套 `src/` 前端代码库。桌面版在 Web 版基础上通过 Electron 提供原生能力（本地文件访问、托盘、全局快捷键等），两者的功能集和交互存在差异。

为了在"一份代码"的前提下管理差异，采用 **运行时平台检测 + 平台模块统一入口** 的策略。

## 架构总览

```
src/utils/platform.ts      ← 集中式平台检测模块（唯一入口）
electron/preload.js        ← 注入 isDesktop、platform、端口等信息
global.d.ts                ← Window.appEnv 类型声明
```

### 数据流

```
Electron preload.js
  → contextBridge.exposeInMainWorld('appEnv', { isDesktop, platform, ports... })
  → src/utils/platform.ts  从 appEnv 读数，提供响应式/函数式 getter
  → 业务代码               import { platform } from '../utils/platform'
```

## platform 模块 API

```ts
import { platform } from '../utils/platform'

platform.isDesktop       // boolean — 是否 Electron 桌面端
platform.isWeb           // boolean — 是否 Web 浏览器
platform.desktopPlatform // 'darwin' | 'win32' | 'linux' | ''
platform.isMacOS         // boolean — 是否 macOS 桌面端
platform.isWindows       // boolean — 是否 Windows 桌面端
platform.isLinux         // boolean — 是否 Linux 桌面端

platform.apiBaseUrl      // string — API 基础 URL
platform.unblockProxyUrl // string — 去阻塞代理 URL
platform.unblockMatchUrl // string — 音源匹配服务 URL

platform.electronVersion // string — Electron 版本号（Web 端返回 ''）
platform.nodeVersion     // string — Node.js 版本号（Web 端返回 ''）
```

所有 getter 已做 SSR 安全兜底（`typeof window !== 'undefined'`），可直接在模块顶层或 `setup` 中使用。

## 开发规则

### 规则 1：所有平台检测必须走 `platform.ts`

```ts
// ✅ 正确
import { platform } from '../utils/platform'
if (platform.isDesktop) { /* ... */ }

// ❌ 禁止
if ((window as any).appEnv?.isDesktop) { /* ... */ }
if (window.appEnv) { /* ... */ }
```

`window.appEnv` 仅在 `platform.ts` 内部引用，业务代码不直接访问。

### 规则 2：功能差异分级处理

#### 2A — 小差异（条件渲染块）

同一组件/页面内，用 `platform.isDesktop` ／ `platform.isWeb` 条件包裹差异区块。

**适用场景**：设置页多几个选项、TopBar 多一个按钮、播放器多一个控制项。

```vue
<template v-if="platform.isDesktop">
  <ToggleRow label="最小化到托盘" />
</template>
```

#### 2B — 中等差异（平台组件替换）

写一个包装层组件，内部 `v-if` 分发到 Web / Desktop 两个实现。

**适用场景**：设置页、播放页等结构上大部分相同但交互差异较大的页面。

```
src/components/SettingsPage.vue         ← 包装层，内部 v-if 分发
src/components/settings/
  SettingsPageWeb.vue                   ← Web 版实现
  SettingsPageDesktop.vue               ← 桌面版实现（含托盘、自启、代理配置）
```

#### 2C — 大差异（Composable + 平台专用组件）

把平台逻辑提取到 composable，差异化部分用独立组件承载。

**适用场景**：桌面端迷你窗口、下载管理器、本地文件播放、全局快捷键管理。

```ts
// src/composables/useDesktopFeatures.ts
import { platform } from '../utils/platform'
export function useDesktopFeatures() {
  if (!platform.isDesktop) return { /* 空壳 */ }
  // 桌面端专属逻辑（托盘、快捷键、原生通知）
}
```

### 规则 3：新增桌面端功能时，必须判断 `platform.isDesktop`

以下场景必须加平台判断：

- **IPC 调用**：`window.appEnv` 相关、Electron contextBridge 通道
- **原生 API**：`Notification`、`navigator.mediaSession`、`Badge` 等浏览器 API 在 Electron 中可用性不同
- **文件系统**：桌面端的 `dialog.showOpenDialog`、本地文件拖放
- **系统集成**：托盘（`Tray`）、全局快捷键（`globalShortcut`）、自动启动
- **窗口管理**：最小化到托盘、置顶、无边框窗口

### 规则 4：Web 端不能打包 Electron 特有代码

Electron 特有依赖（如 `dialog`、`Tray`、`globalShortcut`）通过 preload + contextBridge 暴露，前端只通过 `window.appEnv` 间接使用。

```ts
// ✅ 正确：在 composable 中按平台加载
import { platform } from '../utils/platform'
if (platform.isDesktop) {
  const result = await window.appEnv.showOpenDialog?.()
}

// ❌ 禁止：直接 import electron 模块
import { dialog } from 'electron'  // Web 端会爆炸
```

### 规则 5：新增需要平台感知的模块后，更新 `global.d.ts`

`global.d.ts` 中 `Window.appEnv` 接口定义了 preload 可注入的所有字段，新增字段时必须同步更新：

```ts
// global.d.ts
declare global {
  interface Window {
    appEnv?: {
      // 已有字段
      apiBaseUrl: string;
      // 新增字段需在此追加
      showOpenDialog?: (options: any) => Promise<any>;
    };
  }
}
```

## preload 桥接规范

`electron/preload.js` 通过 `contextBridge.exposeInMainWorld` 暴露给渲染进程的信息包括：

- **服务端口**：`apiBaseUrl`、`unblockProxyUrl`、`unblockMatchUrl`
- **平台标识**：`isDesktop`、`platform`、`electronVersion`、`nodeVersion`

后续新增桌面端能力时，应在 `preload.js` 中用独立函数注册 contextBridge 通道，不要把所有逻辑堆在 `exposeInMainWorld` 调用里：

```js
// preload.js — 新增通道示例
contextBridge.exposeInMainWorld('appEnv', {
  ...ports,
  isDesktop: true,
  platform: process.platform,
  electronVersion: process.versions.electron,
  nodeVersion: process.versions.node,
})

// 后续按能力分组注册
contextBridge.exposeInMainWorld('desktopAPI', {
  showOpenDialog: (options) => ipcRenderer.invoke('dialog:openFile', options),
  getPlatform: () => process.platform,
})
```
