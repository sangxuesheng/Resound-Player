# 封面图片预加载渐入复用说明

本文档记录项目中为消除 CSS `background-image` 加载 JPEG 封面时逐行渲染（一半空白一半图片）而实现的预加载渐入方案，包括公共资源、使用方式、接入清单与约定。

---

## 1. 问题背景

使用 CSS `background-image: url(...)` 加载远端 JPEG 图片时，浏览器默认逐行渲染（从上到下逐渐出现），在网络较慢时会产生明显的视觉卡顿感（半图半空白）。

**对象场景**：所有通过 `:style="{ backgroundImage: ... }"` 或 CSS 设置封面/背景图的元素。

---

## 2. 公共资源

### 2.1 `src/composables/useBgLoaded.ts`

预加载 composable，接收响应式 URL getter，返回 `loaded` ref。

```ts
import { ref, watch, type WatchSource } from 'vue'

export function useBgLoaded(src: WatchSource<string>) {
  const loaded = ref(false)

  watch(
    src,
    (url) => {
      if (!url) {
        loaded.value = false
        return
      }
      loaded.value = false
      const img = new Image()
      img.onload = () => { loaded.value = true }
      img.onerror = () => { loaded.value = true }
      img.src = url
    },
    { immediate: true },
  )

  return loaded
}
```

**行为说明**：
- URL 为空时 → `loaded = false`，不发起网络请求
- URL 变化时 → 重置 `loaded = false`，用 `new Image()` 在内存中预加载
- 加载完成 → `loaded = true`
- 加载失败 → `loaded = true`（兜底，防止永空白）
- 首次绑定 → `immediate: true` 立即触发

### 2.2 `src/styles/animations.css`

全局 CSS utility 类，直接复用，无需在每个组件中重复声明。

```css
.fade-in-bg {
  opacity: 0;
  transition: opacity 0.25s ease;
}
.fade-in-bg.bg-loaded {
  opacity: 1;
}
```

---

## 3. 使用方式

标准模式三步：

```ts
// 1. 引入 composable
import { useBgLoaded } from '../composables/useBgLoaded'

// 2. 传入响应式 URL getter
const loaded = useBgLoaded(() => someCoverUrl.value)
```

```html
<!-- 3. 模板：.fade-in-bg 提供 opacity 0 + transition，.bg-loaded 由 composable 控制 -->
<span class="fade-in-bg"
      :class="{ 'bg-loaded': loaded }"
      :style="{ backgroundImage: `url(${someCoverUrl})` }"></span>
```

**效果**：图片完全下载后才显示，以 0.25s 淡入，无逐行渲染。

---

## 4. 强制约束

### 4.1 必须使用预加载

新增 `background-image: url()` 的封面元素时，必须使用 `useBgLoaded` + `.fade-in-bg`。

- 禁止直接通过 `:style` 或 CSS 设置 `background-image` 而不做预加载渐入
- 禁止在组件 scoped CSS 中重复定义 `opacity: 0; transition: opacity 0.25s` 等私有渐入规则

### 4.2 必须传响应式 getter

```ts
// ✅ 正确：传入 getter，composable 内部 watch 追踪
const loaded = useBgLoaded(() => someRef.value)

// ❌ 错误：传入静态值，不会响应变化
const loaded = useBgLoaded(someRef.value)
```

### 4.3 URL 为空时兜底

`useBgLoaded` 在 URL 为空时不发起网络请求，返回 `false`。元素应依赖 CSS 自身的 `background-color` 或 fallback 样式展示占位色。

### 4.4 onerror 兜底

`onerror` 也会标记 `loaded = true`，确保图片加载失败时不会永久空白（元素会以 CSS 自身 `background-color` 显示）。

## 6. Code Review 检查项

- 新增或修改封面元素时，是否使用了 `useBgLoaded`？
- 模板元素是否同时挂了 `fade-in-bg` class 和 `bg-loaded` class 绑定？
- 是否删除了组件内重复的私有渐入 CSS（`opacity: 0; transition` / `.bg-loaded`）？
