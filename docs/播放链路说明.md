# 播放链路说明

本文档说明当前项目中统一后的播放链路。

## 1. 目标

播放链路统一的目标是：

- 首页、云盘、歌单详情页都使用同一种播放方式
- 不在首屏批量请求所有播放地址
- 只有在真正播放时才获取单曲 URL

---

## 2. 当前公共播放工具

文件：`src/utils/trackPlayback.ts`

### `resolveTrackUrl(track)`

- 输入：单首歌曲对象
- 作用：按需获取这首歌的播放地址
- 优先使用：
  - `track.url`
- 兜底：
  - 调用 `getSongUrl(track.id)`

### `playTrackWithResolvedUrl(playerStore, track, index)`

- 输入：播放器 store、歌曲对象、索引
- 作用：先解析 URL，再放入播放器并播放
- 适合所有“点歌播放”的场景

---

## 3. 首页播放链路

### 每日推荐

首页每日推荐卡片点击歌曲时：

1. 从 `dailyRecommendSongs` 中取对应歌曲
2. 调用 `playTrackWithResolvedUrl`
3. 公共工具内部按需拿 URL
4. 调用播放器播放

### 结果

- 不再在首页预先请求所有播放地址
- 首页更快

---

## 4. 歌单详情页播放链路

### `PlaylistDetailPage.vue`

#### 播放单曲
- 点击单曲时调用 `playTrackWithResolvedUrl`

#### 播放全部
- 实际上播放列表第一首
- 由公共工具按需拿第一首 URL
- 然后进入播放器

### 结果

- 统一和首页、云盘一致
- 不再依赖首屏全量 URL

---

## 5. 云盘播放链路

### 过去
云盘会在加载阶段给每首歌批量请求 `getSongUrl()`，导致首屏慢。

### 现在
云盘改成：

- 首屏只加载歌曲元信息
- 真正播放时再按需取 URL

### 结果

- 云盘详情页更快
- 播放逻辑和其他页面一致

---

## 6. 播放策略总结

### 首屏策略
- 只显示列表和封面
- 不批量请求 URL

### 播放时策略
- 需要播放哪首，就解析哪首 URL
- 交给公共播放工具统一处理

---

## 7. 当前统一后的体验

- 每日推荐播放
- 云盘播放
- 普通歌单播放

都走同一条逻辑：

1. 获取歌曲对象
2. 按需解析 URL
3. 设置播放列表/当前曲目
4. 播放

---

## 8. 结论

现在播放链路已经从“按页面分别实现”统一成了“公共工具统一处理”。

这带来的主要收益是：

- 更少重复代码
- 更快首屏
- 更一致的播放体验
- 更容易扩展新的歌单样式入口
