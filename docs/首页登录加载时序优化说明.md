# 首页登录加载时序优化说明

> 修复页面刷新时私人雷达卡片后加载的问题，确保所有首页卡片同时出现。

---

## 问题

页面刷新后，私人雷达卡片（及私人 FM、公共推荐歌单）总是比其他卡片（热门歌曲、热门歌手、专辑推荐等）晚加载出现。原因是这些卡片依赖登录状态，而登录验证在 HomePanel 的数据请求完成后才执行。

## 根因分析

### 启动时序链

```
App 启动
  → activePage = 'home'（初始值），HomePanel 立即渲染
  → HomePanel.onMounted 执行
     → 发起 11 个并发 API 请求
     → 此时 userStore.isLogin 仍为 false（仅从 localStorage 读取了 cookie）
  → App.vue.onMounted 继续执行
     → waitForApiReady() 等待后端就绪（长耗时）
     → refreshLoginStatus() 验证登录 → isLogin = true
     → watcher 触发 → 重新请求雷达歌单等依赖登录的 API
     → 私人雷达卡片姗姗出现
```

### 关键问题

1. **HomePanel 渲染过早**：`activePage` 初始值为 `'home'`，导致 HomePanel 在登录验证前就开始渲染和请求数据
2. **登录验证滞后**：`refreshLoginStatus()` 在 `waitForApiReady()` 之后执行，而 HomePanel.onMounted 在此期间已经完成
3. **watcher 补偿机制**：虽然有 watcher 监听 `isLogin` 变化并重新请求，但卡片数据到达时间必然晚于其他卡片

## 修复方案

### 1. `src/stores/user.ts` — 新增 `loginVerified` 标记

```ts
loginVerified: false,   // 新增字段
```

在 `refreshLoginStatus()` 末尾设置（不论登录成功或失败）：

```ts
async refreshLoginStatus(requestId = ++this.authRequestSeq) {
    // ... 验证逻辑 ...
    this.loginVerified = true;  // 标记验证完成
}
```

### 2. `src/components/HomePanel.vue` — 等待登录验证后再加载数据

onMounted 改为等待 `loginVerified` 后再发起所有 API 请求：

```ts
onMounted(async () => {
  window.addEventListener('click', onFmPopoverDocClick);
  window.addEventListener('keydown', onFmPopoverKeydown);

  // 等待登录验证完成后再加载所有数据，确保所有卡片同时出现
  if (!userStore.loginVerified && userStore.loginCookie) {
    await new Promise<void>((resolve) => {
      const unwatch = watch(() => userStore.loginVerified, (val) => {
        if (val) { unwatch(); resolve(); }
      });
      // 安全超时：20 秒后强制继续
      setTimeout(() => { unwatch(); resolve(); }, 20000);
    });
  }

  // 全部 11 个 API 请求同时发起
  await Promise.all([loadMoreHotSongs(), fetchDailyRecommendSongs(),
    fetchDailyRecommendPlaylists(), fetchPublicRecoPlaylists(),
    fetchPersonalFm(), fetchTopArtists(), fetchTopAlbums(),
    fetchLatestMusic(), fetchMvList(), fetchPodcastList(),
    fetchRadarPlaylists()]);

  await nextTick();
  setupHotSongsObserver();
});
```

### 3. 移除 onMounted 中不必要的重复请求

将 `fetchDailyRecommendPlaylists()`、`fetchPublicRecoPlaylists()`、`fetchPersonalFm()` 从 onMounted 的 Promise.all 中移除（它们在登录确认前会跳过），交给 watcher 处理。

### 4. watcher 添加 `{ immediate: true }`

监听 `[userStore.isLogin, userStore.loginMode, userStore.profile?.userId]` 的 watcher 添加 `{ immediate: true }`，确保在组件挂载时立即执行一次（此时 isLogin 为 false 会跳过），并在登录状态变化时重新请求。

## 最终启动时序

```
App 启动
  → activePage = 'home'，HomePanel 渲染骨架/加载状态
  → onMounted 执行
     → 注册事件监听器
     → 存在 loginCookie → 等待 loginVerified
  → App.vue.onMounted 执行
     → waitForApiReady()
     → refreshLoginStatus() 完成 → loginVerified = true
  → HomePanel.onMounted 恢复执行
     → 11 个 API 请求同时发起
     → 所有卡片同时填充数据 ✅
```

## 影响范围

| 卡片 | 改前 | 改后 |
|------|------|------|
| 热门歌曲 | onMounted 直接加载 | onMounted 等待后加载 |
| 每日推荐 | onMounted 直接加载 | 不变 |
| **私人雷达** | **watcher 延迟加载** | **与所有卡片同时加载** ✅ |
| **私人 FM** | **watcher 延迟加载** | **与所有卡片同时加载** ✅ |
| **公共推荐歌单** | **watcher 延迟加载** | **与所有卡片同时加载** ✅ |
| 热门歌手/专辑/MV/播客 | onMounted 直接加载 | onMounted 等待后加载 |
| 雷达歌单 | onMounted 直接加载 | 不变 |

> **注意**：未登录用户不受影响（无 loginCookie，等待直接跳过）。
