# 认证体系说明

本文档统一说明项目内登录认证方式区分、VIP 会员判定、各登录模式的使用场景与权限差异。

---

## 1. 登录模式总览

项目支持三种登录模式，通过 `userStore.loginMode` 区分：

| `loginMode` 值 | 登录方式 | 是否真实登录 | 说明 |
|---|---|---|---|
| `'none'` | 未登录 | — | 无任何认证信息 |
| `'cookie'` | Cookie 登录 / 扫码登录 | ✅ 是 | 真实认证，有完整会话 |
| `'qr'` | 扫码登录 | ✅ 是（运行时暂未使用此值，统一归为 `'cookie'`） | 真实认证，流程同 cookie |
| `'uid'` | 搜索用户登录 | ❌ 否 | 仅通过 UID 查 profile，无真实会话 |

### 1.1 Cookie 登录（`loginMode === 'cookie'`）

- 用户粘贴完整 Cookie 字符串
- 调用 `loginWithCookie(cookie)` → `saveCookie()` → `refreshLoginStatus()`
- Cookie 中包含了完整的登录凭据
- 可调用所有需要鉴权的 API（如 `/vip/info`、获取收藏列表等）

### 1.2 扫码登录（实际归为 `'cookie'`）

- 通过 `LoginPanel.vue` 的 QR 轮询流程
- 扫码成功后调用 `saveCookie()` → `refreshLoginStatus()`
- `refreshLoginStatus()` 内部将 `loginMode` 设为 `'cookie'`（未区分 `'qr'`）
- 因此 `'qr'` 值当前未在运行时实际使用，仅保留类型定义

### 1.3 搜索用户登录（`loginMode === 'uid'`）

- 用户输入目标用户的 UID
- 调用 `loginWithUid(uid)` → `getUserDetail(uid)`
- **仅获取目标用户的公开 profile，不是真实登录**
- Cookie 格式为 `uid=用户ID`
- 只能访问不需要鉴权的公开数据
- **不享有 VIP 权限**

### 1.4 登录模式持久化

```ts
// 存储 key
const LOGIN_COOKIE_KEY = 'ncm_login_cookie';  // localStorage
const LOGIN_MODE_KEY = 'ncm_login_mode';       // localStorage

// 恢复流程（hydrate）
// 1. 从 localStorage 读取 loginMode
// 2. 如果 cookie 以 'uid=' 开头，强制设为 'uid'
```

## 2. 用户状态判定

### 2.1 核心状态字段（`userStore`）

| 字段 | 类型 | 含义 |
|---|---|---|
| `isLogin` | `boolean` | 是否有用户 profile（登录态） |
| `loginMode` | `'none' \| 'cookie' \| 'qr' \| 'uid'` | 登录方式 |
| `isVip` | `boolean` | 当前是否是活跃 VIP 会员 |
| `vipInfo` | `VipInfo \| null` | VIP 详细数据（含到期时间） |
| `loginCookie` | `string` | 存储的认证凭据 |

### 2.2 真实登录 vs 非真实登录

```
真实登录 (cookie/qr)
  ├── isLogin = true
  ├── loginMode = 'cookie'
  └── 可调用需鉴权的 API

搜索用户 (uid)
  ├── isLogin = true
  ├── loginMode = 'uid'  
  ├── Cookie = 'uid=用户ID'
  └── 不可调用需鉴权的 API

未登录 (none)
  ├── isLogin = false
  ├── loginMode = 'none'
  └── 只能访问公开数据
```

判定函数：

```ts
// 是否是真实登录
const isRealLogin = userStore.loginMode === 'cookie' || userStore.loginMode === 'qr';
```

## 3. VIP 会员判定

### 3.1 判定链路

```
fetchVipInfo()
  ├── 优先: getVipInfo(undefined, cookie)  // 带 cookie 认证
  │   └── 返回嵌套格式 { associator, musicPackage, redplus, ... }
  │       └── 包含 expireTime（到期时间）
  │
  ├── 回退: getVipInfo(uid)                 // 无 cookie，仅 UID
  │   └── 返回扁平格式 { redVipLevel, ... }
  │       └── 无 expireTime，仅等级
  │
  └── 最终: 保守判断
      ├── 嵌套格式 + expireTime → 精确判断
      └── 扁平格式（无 expireTime）→ isVip = false
```

### 3.2 活跃会员判断逻辑

```ts
// 核心判断：必须同时满足 vipCode > 0 且在有效期内
const now = Date.now();
this.isVip
  = (associator.vipCode > 0 && (associator.expireTime === -1 || associator.expireTime > now))
  || (musicPackage.vipCode > 0 && (musicPackage.expireTime === -1 || musicPackage.expireTime > now))
  || (redplus.vipCode > 0 && (redplus.expireTime === -1 || redplus.expireTime > now));
```

- `vipCode > 0`：有会员资格
- `expireTime === -1`：永久会员
- `expireTime > now`：会员在有效期内
- **三者任一类型满足即视为活跃会员**

### 3.3 `/vip/info` 响应结构

**嵌套格式（含 cookie 认证时返回）：**

```json
{
  "data": {
    "associator": {
      "vipCode": 1,           // 0=无会员, >0=有会员
      "expireTime": 1777937902098,  // 到期毫秒时间戳, -1=永久
      "vipLevel": 1,          // 会员等级（仅用于展示）
      "isSign": true,          // 是否签约
      "isSignDeduct": false,   // 自动续费
      "isSignIap": false,      // IAP 购买
      "isSignIapDeduct": false // IAP 自动续费
    },
    "musicPackage": { ... },  // 音乐包（同理）
    "redplus": { ... },       // 黑胶VIP+（同理）
    "redVipLevelIcon": "..."
  }
}
```

**扁平格式（无 cookie/UUID 回退时）：**

```json
{
  "data": {
    "redVipLevel": 1,
    "redVipLevelIcon": "..."
  }
}
```

### 3.4 关键结论

| 字段 | 是否可靠 | 说明 |
|---|---|---|
| `associator.vipCode` | ✅ 可靠 | 0=无会员, >0=有会员资格 |
| `associator.expireTime` | ✅ 可靠 | 判断当前是否在有效期内 |
| `associator.vipLevel` | ❌ 不可靠 | 历史累计等级，会员过期后仍可能 > 0 |
| `profile.vipType` | ❌ 不可靠 | 登录 API 返回，会员过期后可能仍残留 |
| `redVipLevel`（扁平格式） | ⚠️ 仅供参考 | 无到期时间，无法确认当前是否活跃 |

## 4. 各场景下的权限对照

| 场景 | `loginMode` | `isLogin` | `isVip` | 可用音质（下载） |
|---|---|---|---|---|
| 未登录 | `'none'` | `false` | `false` | 标准、较高、极高(HQ) |
| 搜索用户（UID） | `'uid'` | `true` | `false` | 标准、较高、极高(HQ) |
| Cookie/QR 登录 + 非会员 | `'cookie'` | `true` | `false` | 标准、较高、极高(HQ) |
| Cookie/QR 登录 + 活跃黑胶VIP | `'cookie'` | `true` | `true` | 全部 9 种 |
| Cookie/QR 登录 + 活跃 SVIP | `'cookie'` | `true` | `true` | 全部 9 种 |
| Cookie/QR 登录 + 会员已过期 | `'cookie'` | `true` | `false` | 标准、较高、极高(HQ) |

## 5. 实际消费场景：下载弹窗权限控制

文件：`src/components/ui/SongActions.vue`

```ts
function isQualityAvailable(level: string): boolean {
  const isRealLogin = userStore.loginMode === 'cookie' || userStore.loginMode === 'qr';
  if (isRealLogin && userStore.isVip) return true;
  return !VIP_ONLY_LEVELS.has(level);
}
```

逻辑拆解：

1. **先判断是否是真实登录**：`loginMode === 'cookie' || 'qr'`
2. **再判断是否是活跃会员**：`userStore.isVip`（由 `fetchVipInfo` 精确判断）
3. **两者都满足** → 所有 9 种音质可下载
4. **否则** → 仅标准/较高/极高(HQ) 3 种免费音质可下载

每次打开下载弹窗时都会调用 `await userStore.fetchVipInfo()` 刷新 VIP 状态，确保判断基于最新数据。

## 7. 维护提示

- 新增登录方式时，需同步更新：`LoginMode` 类型、`loginMode` 设置逻辑、`hydrate` 恢复逻辑
- VIP 判定仅信任 `associator.vipCode + expireTime` 组合，不要依赖 `vipLevel` 或 `profile.vipType`
- 所有调用 `getVipInfo` 的地方需确保传入 `cookie` 参数，否则返回未认证数据
- 下载弹窗打开时通过 `await fetchVipInfo()` 主动刷新，不要依赖登录时的缓存状态
