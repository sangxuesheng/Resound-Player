# 下载流程说明

本文档说明项目内不同登录状态下下载弹窗的音质可用性、文件大小获取源、实际下载执行链路的差异。

---

## 1. 用户类型定义

项目中有三种用户类型，影响下载时的行为：

| 类型 | `loginMode` | `isLogin` | `isVip` | 说明 |
|------|------------|-----------|---------|------|
| **VIP 用户** | `cookie` / `qr` | `true` | `true` | 真实登录 + 会员 |
| **非 VIP 用户** | `cookie` / `qr` | `true` | `false` | 真实登录但非会员 |
| **非真实登录** | `uid` / `none` | 可能 `true`(uid) 或 `false`(none) | `false` | 搜索用户登录或未登录 |

判断逻辑（`SongActions.vue` 第 406 行）：

```typescript
const isRealLogin = userStore.loginMode === 'cookie' || userStore.loginMode === 'qr';
```

### 1.1 登录模式说明

| 登录模式 | 对应 API | cookie 值 | 真实凭据 |
|---------|---------|-----------|---------|
| `cookie` | 手动输入 Cookie | 完整 cookie 字符串 | ✅ 是 |
| `qr` | 扫码登录 | 完整 cookie 字符串 | ✅ 是 |
| `uid` | 搜索用户登录 | `uid={用户ID}` | ❌ 仅用户 ID |
| `none` | 未登录 | 空 | ❌ |

详见 [`认证体系说明.md`](./认证体系说明.md)。

### 1.2 VIP 检测链路

`userStore.fetchVipInfo()` 分两条路径：

1. **真实登录（cookie/qr）**：携带完整 cookie 调用 `getVipInfo()`，返回嵌套格式（含 `expireTime`），可精确判断会员过期时间
2. **uid 模式**：cookie 仅为 `uid=用户ID`，不是真实凭据，跳过 cookie 认证路径，用 uid 查询扁平格式（仅有 `redVipLevel`）
   - 扁平格式不能严格证明会员身份，`isVip` 强制设为 `false`

## 2. 下载弹窗总体流程

```
用户点击下载按钮
  → openDownloadPopup()
    → userStore.fetchVipInfo()    // 刷新 VIP 状态
    → showDownloadPopup = true
      → watch(showDownloadPopup)
        → fetchDownloadSizes()    // 获取文件大小
        → fetchLyricsInfo()       // 获取歌词信息
  → 用户选择音质 → handleDownload()
    → downloadSong(level, label)  // 下载音频
    → downloadSelectedLyrics()    // 下载歌词
```

## 3. 音质可用性判断 (`isQualityAvailable`)

```typescript
// SongActions.vue 第 108-111 行
function isQualityAvailable(level: string): boolean {
  const isRealLogin = userStore.loginMode === 'cookie' || userStore.loginMode === 'qr';
  if (isRealLogin && userStore.isVip) return true;  // VIP：全部可用
  return !VIP_ONLY_LEVELS.has(level);                 // 非VIP：仅免费音质
}
```

### 3.1 VIP 等级与音质映射

| 音质 | API level | 需要的最低 VIP 等级 | 显示标记 |
|------|-----------|-------------------|---------|
| 标准 | `standard` | 免费 | - |
| 较高 | `higher` | 免费 | - |
| 极高(HQ) | `exhigh` | 免费 | - |
| 无损(SQ) | `lossless` | 黑胶VIP | `黑胶VIP` |
| Hi-Res | `hires` | 黑胶VIP | `黑胶VIP` |
| 高清臻音 | `jyeffect` | SVIP (redplus) | `SVIP` |
| 沉浸环绕声 | `sky` | SVIP (redplus) | `SVIP` |
| 杜比全景声 | `dolby` | SVIP (redplus) | `SVIP` |
| 超清母带 | `jymaster` | SVIP (redplus) | `SVIP` |

详见 [`音质体系说明.md`](./音质体系说明.md)。

### 3.2 各用户类型可用音质

| 用户类型 | 可用音质 |
|---------|---------|
| VIP | 全部 9 个音质 |
| 非 VIP（真实登录） | standard, higher, exhigh |
| 非真实登录（uid/none） | standard, higher, exhigh |

## 4. 文件大小获取源 (`fetchDownloadSizes`)

这是**核心差异点**，不同用户类型使用不同的数据源获取文件大小。

### 4.1 决策逻辑

```typescript
// SongActions.vue 第 135-137 行
const isRealLogin = userStore.loginMode === 'cookie' || userStore.loginMode === 'qr';
const useOfficial = isRealLogin && userStore.isVip;
```

### 4.2 各用户类型的源

| 用户类型 | `useOfficial` | 获取方式 | 请求次数 | 文件大小精度 |
|---------|:-----------:|---------|---------|------------|
| **VIP** | `true` | `getSongUrlV1(songId, level, cookie)` — 对每个可用音质分别请求官方 API | 9 次（每个音质 1 次） | 各音质精确大小 |
| **非 VIP** | `false` | `tryUnblockMatch(songId, sources)` — 从音源替换获取单一文件大小 → 复用给所有可用音质 | 1 次 | 所有音质显示相同大小 |
| **非真实登录** | `false` | 同上 | 1 次 | 所有音质显示相同大小 |

### 4.3 VIP 用户大小获取流程

```
getSongUrlV1(id, "standard", cookie)      → standard 大小
getSongUrlV1(id, "higher", cookie)        → higher 大小
getSongUrlV1(id, "exhigh", cookie)        → exhigh 大小
getSongUrlV1(id, "lossless", cookie)      → lossless 大小
getSongUrlV1(id, "hires", cookie)         → hires 大小
getSongUrlV1(id, "jyeffect", cookie)      → jyeffect 大小
getSongUrlV1(id, "sky", cookie)           → sky 大小
getSongUrlV1(id, "dolby", cookie)         → dolby 大小
getSongUrlV1(id, "jymaster", cookie)      → jymaster 大小
```

每个音质独立显示精确文件大小。

### 4.4 非 VIP / 非真实登录用户大小获取流程

```
tryUnblockMatch(id, ["bodian","kugou","migu","qq","bilibili"])
  └─ 返回 { url, source, br, size }
     └─ 将 size 赋值给所有可用音质（standard, higher, exhigh）
     └─ 可用音质显示相同文件大小
```

音源替换失败时的兜底：

```
tryUnblockMatch 返回空
  └─ 回退到遍历 qualityOptions，逐个调用 getSongUrlV1 获取官方大小
```

### 4.5 音源替换服务

| 项目 | 说明 |
|------|------|
| 服务端 | `server/unblock-match-server.mjs` |
| 端口 | 38763 |
| 前端代理 | `/unblock-api` → `http://127.0.0.1:38763` |
| 匹配 API | `GET /match?id={songId}&sources={src1},{src2},...` |
| 可用源 | `bodian`, `kugou`, `migu`, `qq`, `bilibili` |
| 返回字段 | `url`, `source`, `br`, `size` |
| 健康检查 | `GET /health`（3s 超时，60s 冷却，连续 3 次失败标记不可用） |

详见 [`src/api/unblock.ts`](../src/api/unblock.ts)。

## 5. 下载执行链路 (`downloadSong`)

音频下载的线路对不同用户类型**基本一致**，按以下优先级尝试：

### 5.1 步骤

```
1. 官方源下载
   getSongUrlV1(songId, level, cookie)
   └─ 获取官方音频 URL
     ├─ music.126.net CDN → 直接 fetch（支持 CORS）
     └─ music.163.com → 跳过直连，走代理

2. 代理下载
   fetch(`/dl-proxy?url={url}&cookie={cookie}`)
   └─ Vite middleware，转发 Cookie，返回 Access-Control-Allow-Origin: *

3. 音源替换兜底（官方源失败时）
   tryUnblockMatch(songId, sources)
   └─ 获取替换 URL
     └─ fetch(`/dl-proxy?url={matchUrl}&cookie=`)
     └─ 文件扩展名固定为 .mp3
```

### 5.2 `/dl-proxy` 工作原理

Vite 中间件，定义在 `vite.config.ts` 第 15-44 行：

| 功能 | 说明 |
|------|------|
| 用途 | 解决 CORS 限制，转发下载请求 |
| 协议 | 自动适配 http/https |
| Cookie 转发 | 通过 query 参数传递 |
| CORS 头 | 始终返回 `Access-Control-Allow-Origin: *` |
| 内容类型 | 透传源响应的 `content-type` 和 `content-length` |

### 5.3 各用户类型下载行为差异

| 阶段 | VIP | 非 VIP | 非真实登录 |
|------|:---:|:------:|:---------:|
| 调用 `getSongUrlV1` | ✅ 携带完整 cookie | ✅ 携带完整 cookie | ✅ 携带空 cookie |
| 官方源可能返回付费曲目的完整 URL | ✅（VIP 可播） | ❌（可能返回试听片段） | ❌（无 cookie） |
| 音源替换兜底 | ✅ 触发 | ✅ 触发 | ✅ 触发 |
| 最终降级 | 官方 URL 优先 | 音源替换结果优先 | 音源替换结果优先 |

## 6. 歌词下载 (`downloadSelectedLyrics`)

歌词下载流程对所有用户类型**一致**：

| 歌词类型 | 数据源 | 说明 |
|---------|-------|------|
| 主歌词 (LRC) | `getSongLyric(id)` → `body.lrc.lyric` | 默认勾选 |
| 翻译歌词 (LRC) | `getSongLyric(id)` → `body.tlyric.lyric` | 有内容才显示 |
| 音译歌词 (LRC) | `getSongLyric(id)` → `body.romalrc.lyric` | 有内容才显示 |
| 逐字歌词 (YRC) | `getSongLyric(id)` → `body.yrc.lyric` | 优先旧 API，无结果时尝试 `/lyric/new` |

文件命名格式：`{歌曲名} - {歌手}.lrc` / `{歌曲名} - {歌手}.yrc`

## 8. 常见问题

### Q: 为什么非 VIP 用户所有可用音质显示相同文件大小？

因为非 VIP 用户不逐个请求官方 API 获取各音质大小，而是通过音源替换匹配获取单一文件的 size，然后复用给所有可用音质。这是一个优化，避免对非 VIP 无意义地请求 9 次官方 API。

### Q: 下载时官方源失败一定会走音源替换吗？

不一定。音源替换有健康管理机制：替换服务器连续 3 次匹配失败后会被标记为不可用，60 秒内不再尝试。此后仅走官方源。

### Q: uid 登录（搜索用户）能下载吗？

能。`uid` 模式的 `isRealLogin` 为 `false`，走的是非 VIP 路径。音质限制为 standard/higher/exhigh，文件大小通过音源替换获取，下载时调用 `getSongUrlV1` 携带空 cookie。
