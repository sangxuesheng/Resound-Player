# 添加到播放列表功能说明

在歌单详情页与播客详情页的 hero actions 区域增加"添加到播放列表"按钮，支持歌曲与播客节目混合在同一播放队列中，并为付费播客提供通知与自动跳过能力。

---

## 1. 核心 Store 方法

文件：`src/stores/player.ts`

### 1.1 `appendToQueue(tracks)` — 追加到播放列表末尾

```ts
appendToQueue(tracks: any[]): number
```

**职责**：将一批 track 追加到当前播放列表末尾，不替换已有队列。

**行为**：
- 调用 `formatTrack()` 格式化每个条目
- 按组合键 `(id:source)` 去重（避免歌曲 id 与播客 id 碰撞）
- 允许歌曲和播客混合在同一队列中（无类型过滤）
- 返回实际追加数量

**去重键示例**：
- 歌曲：`12345:song`
- 播客：`12345:podcast`
- 两者 id 相同但 source 不同 → 视为不同条目，各自保留

**参考位置**：`player.ts` 中 `setPlaylist()` 方法之后。

```ts
function addAllToQueue() {

  if (!list.length) return;
  const added = playerStore.appendToQueue(list);
  if (added > 0) {
    showToast(`已添加 ${added} 首至播放列表`, 'success', 3000);
```

## 3. 播客详情页

文件：`src/components/PodcastDetailPage.vue`

### 3.1 模板

与歌单页相同模式，在 `<template #actions>` 中新增按钮：

```html
<AnimatedAppear tag="button" variant="control" rhythm="actions" class-name="add-to-queue" type="button" @click="addAllToQueue">添加到播放列表</AnimatedAppear>
```

### 3.2 处理函数

```ts
function addAllToQueue() {
  const items = normalizedItems.value;
  if (!items.length) return;
  const tracks = items
    .map((item) => {
      const raw = item.raw;
      if (!item.trackId) return null;
      const programId = Number(raw?.id || raw?.voiceId || raw?.programId || raw?.program?.id || 0);
      const createTime = Number(raw?.createTime || raw?.program?.createTime || 0);
      const podcastMeta: Record<string, any> = {};
      if (podcastRid.value) podcastMeta.rid = podcastRid.value;
      if (programId) podcastMeta.programId = programId;
      if (createTime) podcastMeta.createTime = createTime;
      // 携带付费/会员徽标信息，供播放列表展示
      const feeBadge = item.badges?.find((b) =>
        b.tone === 'paid' || b.tone === 'vip' || b.tone === 'purchased'
      );
      if (feeBadge) {
        podcastMeta.feeBadge = feeBadge.label;
        podcastMeta.feeTone = feeBadge.tone;
      }
      return {
        id: item.trackId,            // mainTrackId（音频 track ID）
        name: item.name,
        ar: [{ name: hero.value.name || '播客' }],
        al: { picUrl: item.coverUrl }, // 供播放列表封面使用
        source: 'podcast',
        description: item.description,
        podcast: Object.keys(podcastMeta).length ? podcastMeta : undefined,
      };
    })
    .filter(Boolean);
  if (!tracks.length) return;
  const added = playerStore.appendToQueue(tracks);
  if (added > 0) {
    showGlobalToast(`已添加 ${added} 条至播放列表`, 'success', 3000);
  }
}
```

**关键点**：
- `id` 必须用 `item.trackId`（即 `mainTrackId`），而非 raw item 的 `id`（那是节目 ID）
- `al.picUrl` 用 `item.coverUrl`（`normalizedItems` 已解析过的封面链）
- `ar` 用 `hero.value.name`（电台/节目名），而非分类名
- 导入 `showGlobalToast` 来自 `src/stores/loginModal`

### 3.3 播客条目规约

播客条目放入播放列表时，必须满足以下字段：

| 字段 | 值 | 作用 |
|------|-----|------|
| `id` | `mainTrackId` | 音频 track ID，用于播放器请求音源 |
| `name` | 节目名 | 显示在播放列表 |
| `ar` | `[{ name: '节目名' }]` | 显示在播放列表副标题行 |
| `al.picUrl` | 封面 URL | 显示在播放列表封面 |
| `source` | `'podcast'` | 用于类型标识、组合键去重、UI 徽标判定 |
| `description` | 节目描述 | 透传给播放器 |
| `podcast.rid` | 电台 ID | 评论区、播客相关 UI |
| `podcast.feeBadge` | 付费/会员/已购买 | 在播放列表中显示付费徽标 |
| `podcast.feeTone` | paid/vip/purchased | 付费徽标的主题色变体 |

## 4. App.vue 播客播放路径补充

文件：`src/App.vue`

### 4.1 `normalizePodcastPlayableTrack` 补充

当用户直接从播客详情页点击播放（而非通过"添加到播放列表"）时，走的路径是 `playPodcastItem` → `normalizePodcastPlayableTrack` → `setPlaylist` → `playByIndex`。

此函数需要携带 `feeBadge` / `feeTone`，否则播放器的付费播客检测不会触发。

内联检测逻辑（与 `PodcastDetailPage` 的 `resolveFeeMeta` 一致）：

```ts
const feeType = Number(item?.voiceFeeType ?? item?.feeType ?? ...);
const payed = Number(item?.payed ?? item?.paid ?? 0);
const buyed = Boolean(item?.buyed ?? item?.purchased ?? ...) || payed > 0;
const needPay = Boolean(item?.needPay ?? ...) || price > 0;
let feeBadge, feeTone;
if (buyed) { feeBadge = '已购买'; feeTone = 'purchased'; }
else if (feeType === 8 || feeType === 16) { feeBadge = '会员'; feeTone = 'vip'; }
else if (feeType > 0 || needPay) { feeBadge = '付费'; feeTone = 'paid'; }
```

在返回的 `podcast` 对象的所有分支中都携带 `feeBadge` / `feeTone`。

```ts
type PodcastMeta = {

```

### 6.2 `playTrack` 付费检测

在 `playTrack` 方法最开头添加检测：

```ts
if (track.source === 'podcast' && track.podcast?.feeTone === 'paid') {
  showGlobalToast('该节目为付费内容，请前往播客详情页购买后收听', 'warning', 4000);
  this.loading = false;
  if (this.paidContentSkip) {
    await this.next();       // 跳过 → 自动播下一首
  } else {
    this.isPlaying = false;  // 停止 → 保持暂停
    this.persist();
  }
  return;                    // 不执行后续 API 请求
}
```

### 6.3 `paidContentSkip` 状态

```ts
paidContentSkip: true  // 默认开启
```

- 用户设置 → 持久化（`persist()` / `hydrate()`）
- 生命周期同 `autoplayNext`、`playMode` 等设置项

## 9. 混合队列的注意事项

### 9.1 去重策略

组合键 `(id:source)` — 歌曲 `123:song` 和播客 `123:podcast` 视为不同条目。

### 9.2 封面显示

- 歌曲：`track.al?.picUrl`（API 返回的标准字段）
- 播客：在 `addAllToQueue` 中手动设置 `al: { picUrl: coverUrl }`

### 9.3 播放器行为

- 播放器已支持 per-track 的 `source` 判断
- 评论区、歌词等 UI 组件根据当前 track 的 `source` 独立决策
- 混合队列不会破坏已有播放器行为

### 9.4 付费检测覆盖路径

| 入口 | 路径 | 是否需要传 feeBadge |
|------|------|---------------------|
| 歌单页 → 播放 | `playAll` → `setPlaylist` → `playByIndex` | 不涉及（歌曲无付费概念） |
| 播客页 → 添加到播放列表 → 点击播放 | `appendToQueue` → `playByIndex` | ✅ 在 `addAllToQueue` 中传递 |
| 播客页 → 直接点击播放 | `playPodcastItem` → `normalizePodcastPlayableTrack` → `setPlaylist` | ✅ 在 `normalizePodcastPlayableTrack` 中传递 |
