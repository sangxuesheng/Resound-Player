# 登录态鉴权与全局 Toast 复用说明

## 概述

项目中所有需要登录态的操作（收藏歌单、收藏专辑、订阅播客、关注歌手、关注用户、收藏至歌单、上传至云盘等）统一使用同一套鉴权机制。由以下入口覆盖所有场景：

| 入口 | 定位 | 鉴权方式 |
|---|---|---|
| `useEntitySubscribe` composable | 收藏歌单 / 收藏专辑 / 订阅播客 / 关注歌手 | 内置在 `toggle()` 中 |
| `useUserFollow` composable | 关注用户（三态） | 内置在 `toggle()` 中 |
| `BookmarkIconButton.vue` | 收藏按钮（标准收藏入口组件） | 组件自身内置鉴权 |
| `useAuthAction` composable | 收藏至歌单等需要鉴权的操作 | 业务函数手动调用 |
| `SongActions.vue` | 上传至云盘按钮 | 直接调用 `showLoginModal` / `showGlobalToast` |

所有入口内部走同一套逻辑：**未登录→弹登录窗，uid 登录→全局 toast 提示**。

## 实现结构

```
src/stores/loginModal.ts        ← 全局 toast 状态
src/components/LoginModal.vue   ← 全局 toast UI（Teleport 到 body）
src/composables/useAuthAction.ts ← 鉴权 composable
src/composables/useEntitySubscribe.ts ← 实体收藏/订阅 composable（二态，内置鉴权）
src/composables/useUserFollow.ts ← 用户关注 composable（三态，内置鉴权）
src/components/ui/BookmarkIconButton.vue ← 收藏按钮（内置鉴权）
src/components/ui/EntitySubscribeButton.vue ← 实体收藏/订阅按钮（由 composable 驱动）
src/components/ui/UserFollowButton.vue ← 用户关注按钮（由 composable 驱动）
src/components/ui/SongActions.vue ← 上传至云盘按钮（直接调用鉴权）
```

### 全局 Toast

toast 统一由 `LoginModal.vue` 渲染（Teleport 到 body），任何组件或 composable 只需调用 `showGlobalToast()` 即可触发，无需父页面帮忙渲染。

```ts
import { showGlobalToast, showLoginModal } from '../stores/loginModal';

// 显示 toast（自动 4s 消失，重复调用会重置计时器）
showGlobalToast('消息内容', 'warning');           // 默认 warning（黄色）
showGlobalToast('操作成功', 'success', 3000);     // success（绿色）
showGlobalToast('操作失败', 'error', 3000);       // error（红色）

// 弹登录窗
showLoginModal('like');      // 收藏意图
showLoginModal('playlist');  // 添加到歌单意图
```

### `useAuthAction` composable

用于收藏至歌单等需要手动鉴权的操作。

```ts
import { useAuthAction } from '../composables/useAuthAction';

// 在 setup 中初始化
const { checkAuth, showToast } = useAuthAction(
  '搜索用户方式登录不支持收藏功能，请使用扫码或 Cookie 登录',  // uid 登录时的 toast 文案
  'playlist',                // 未登录时弹窗的 intent 标识
);

// 在操作函数开头调用
async function addToPlaylist() {
  if (!checkAuth()) return;   // 鉴权失败则提前返回（弹窗或 toast）
  // 鉴权通过，执行实际操作
}

// 显示成功/失败的 toast（走全局 toast）
showToast('已添加至播放列表', 'success', 3000);
showToast('添加失败，请重试', 'error', 3000);
```

#### 返回值

| 返回值 | 类型 | 说明 |
|---|---|---|
| `checkAuth` | `() => boolean` | 鉴权函数，返回 true 表示可继续操作 |
| `showToast` | `(msg, type?, duration?) => void` | 显示全局 toast |

### `BookmarkIconButton`（收藏按钮）

标准收藏按钮组件，鉴权逻辑已内置于 `toggleSaved()` 内部。业务页面只需传入 `songId`，可选传入 `liked` prop 以优先使用页面数据中的收藏状态，无需额外处理鉴权。

```vue
<BookmarkIconButton :song-id="song.id" :liked="userStore.likedSongIds.includes(song.id)" />
```

三种场景的自动处理：

| 场景 | 行为 |
|---|---|
| 未登录 | 自动弹登录窗（intent: `like`） |
| uid 登录（搜索用户） | 自动显示全局 toast "搜索用户方式登录不支持收藏" |
| cookie/qr 登录 | 正常执行收藏调用 |

### `SongActions.vue`（上传至云盘按钮）

上传至云盘按钮直接调用 `showLoginModal` / `showGlobalToast` 进行鉴权。

```ts
// 检查登录状态
if (!userStore.isLogin) {
  showLoginModal('none');
  return;
}
if (userStore.loginMode !== 'cookie' && userStore.loginMode !== 'qr') {
  showGlobalToast('搜索用户方式登录不支持上传云盘功能，请使用扫码或 Cookie 登录', 'warning', 5000);
  return;
}
// 鉴权通过，执行上传操作
```

三种场景的处理：

| 场景 | 行为 |
|---|---|
| 未登录 | 弹出登录对话框 |
| uid 登录（搜索用户） | 显示全局 toast "搜索用户方式登录不支持上传云盘功能，请使用扫码或 Cookie 登录" |
| cookie/qr 登录 | 正常执行上传 |

### `EntitySubscribeButton` + `useEntitySubscribe`（实体收藏/订阅）

用于收藏歌单、收藏专辑、订阅播客、关注歌手。鉴权逻辑内置于 `useEntitySubscribe` 的 `toggle()` 中，业务页面只需调用 composable 并绑定按钮组件。

```vue
<script setup lang="ts">
import EntitySubscribeButton from './ui/EntitySubscribeButton.vue';
import { useEntitySubscribe } from '../composables/useEntitySubscribe';

const albumIdRef = computed(() => album.value?.id || undefined);
const { isSubscribed, isLoading, toggle } = useEntitySubscribe({
  type: 'album',
  id: albumIdRef,
  initialSubscribed: computed(() => album.value?.subscribed ?? false),
});
</script>

<template>
  <EntitySubscribeButton
    type="album"
    text
    :subscribed="isSubscribed.value"
    :loading="isLoading.value"
    @toggle="toggle"
  />
</template>
```

三种场景的自动处理：

| 场景 | 行为 |
|---|---|
| 未登录 | 自动弹登录窗（intent: `subscribe`） |
| uid 登录（搜索用户） | 自动显示全局 toast "当前登录方式不支持此操作，请使用扫码或 Cookie 登录" |
| cookie/qr 登录 | 正常执行 API 调用 |

### `UserFollowButton` + `useUserFollow`（关注用户）

用于关注用户（三态：未关注 / 已关注 / 互相关注）。鉴权逻辑内置于 `useUserFollow` 的 `toggle()` 中。

```vue
<script setup lang="ts">
import UserFollowButton from './ui/UserFollowButton.vue';
import { useUserFollow } from '../composables/useUserFollow';
import { userStore } from '../stores/user';

const userIdRef = computed(() => props.userId || undefined);
const { status, isLoading, toggle } = useUserFollow({ id: userIdRef });
</script>

<template>
  <UserFollowButton
    v-if="userId && userId !== userStore.profile?.userId"
    :status="status.value"
    :loading="isLoading.value"
    @toggle="toggle"
  />
</template>
```

三种场景的自动处理：

| 场景 | 行为 |
|---|---|
| 未登录 | 自动弹登录窗（intent: `subscribe`） |
| uid 登录（搜索用户） | 自动显示全局 toast "当前登录方式不支持此操作，请使用扫码或 Cookie 登录" |
| cookie/qr 登录 | 正常执行 API 调用 |

## 鉴权规则

```ts
function checkAuth(): boolean {
  if (!userStore.isLogin) {
    showLoginModal(loginIntent);   // 弹出登录弹窗
    return false;
  }
  if (userStore.loginMode !== 'cookie' && userStore.loginMode !== 'qr') {
    showGlobalToast(toastMsg, 'warning', 5000);  // 显示 uid 登录提示
    return false;
  }
  return true;  // 鉴权通过
}
```
