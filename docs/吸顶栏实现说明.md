# 吸顶栏实现说明

本项目详情页的吸顶栏采用**连续进度映射**方案，滚动位置平滑映射为 `--sticky-progress`（0→1），通过 CSS `clip-path` + 内联 inline style 驱动多层元素的独立变换，实现类似 Apple Music 的渐进式吸顶效果。

---

## 一、架构层次

```
页面模板（如 ArtistDetailPage.vue）
│
├─ useDetailStickyState()          ← 管理 isSticky 布尔状态
│
├─ DetailStickyHeroHeader.vue      ← 吸顶头部核心组件
│   ├─ position: sticky; top: 0    ← 始终粘性定位
│   ├─ clip-path 渐进裁剪          ← 折叠时只留顶部 ~80px 可见
│   ├─ JS 计算 progress (0→1)      ← requestAnimationFrame 逐帧更新
│   ├─ 6 组内联样式同时驱动        ← 封面/元数据/描述/按钮独立控制
│   └─ PageHeroHeader.vue          ← 内部 grid 布局（媒体列 + 内容列）
│
└─ 详情页 Tab 栏（如 .artist-tabs）  ← 独立 sticky，紧贴折叠后的头部
    └─ position: sticky; top: 80px ← 在吸顶栏下方保持置顶
```

## 三、连续进度机制

### 3.1 滚动 → progress 映射

```typescript
const PROGRESS_SCROLL_DISTANCE = 324;  // 从完全展开到完全折叠所需的滚动像素

function updateProgress() {
  const pt = scrollHost.scrollTop;
  const progress = Math.max(0, Math.min(1, pt / PROGRESS_SCROLL_DISTANCE));
  headerWrapEl.style.setProperty('--sticky-progress', String(progress));
  // ... 6 组内联样式更新
}
```

- `progress = 0`：完全展开
- `progress = 1`：完全折叠（只剩顶部 ~80px 可见）

### 3.2 六组独立变换

每个元素组有独立的变换参数，形成层次化视差效果：

| 元素 | 变换效果 (progress 0→1) | 说明 |
|---|---|---|
| **媒体列** (`.page-hero-header__media`) | `translateX(-18px) + scale(0.84)` | 封面列向左收缩 |
| **封面壳** (`.hero-media-shell`) | `opacity 1→0, translateX(-10px) translateY(-4px) scale(0.76)` | 封面淡出缩小 |
| **封面图** (`img.cover`) | `opacity 1→0, scale(0.7), saturate(0.88), blur(6px), borderRadius 16→22px` | 模糊+圆角变化 |
| **元数据区** (`.hero-meta-shell`) | `opacity 1→0, translateY(-8px)` | 向上淡出 |
| **描述区** (`.desc`) | `opacity 1→0, translateY(-8px), maxHeight 120→0px` | 收缩淡出 |
| **操作按钮区** (`.hero-actions-shell`) | `opacity 1→0, translateY(-6px)` | 吸顶态 absolute 定位到右上角 |

### 3.3 滚动事件处理

```typescript
function onScroll() {
  cancelAnimationFrame(progressRAF);
  progressRAF = requestAnimationFrame(updateProgress);
}
```

使用 `requestAnimationFrame` 保证与浏览器渲染帧同步，避免过度计算。

## 四、clip-path 裁剪机制

### 4.1 原理

当 `.is-sticky-header` 激活时：

```css
.playlist-detail-header-wrap.is-sticky-header {
  clip-path: inset(0 0 calc(var(--sticky-progress, 0) * (100% - 80px)) 0);
}
```

- `progress=0` → `inset(0 0 0 0)`：全部可见
- `progress=1` → `inset(0 0 calc(100% - 80px) 0)`：只保留顶部 80px
- 中间值平滑过渡

### 4.2 为什么需要 clip-path

- 头部元素使用 `position: sticky; top: 0` + `min-height: var(--sticky-header-height)` 保持文档流高度恒定
- 内部元素通过 inline style 渐进淡出/缩小后，clip-path 确保不可见区域完全透明，让下方列表内容正常显示
- 不破坏文档流 → 列表永不跳动

### 4.3 吸顶态样式总览

```css
.playlist-detail-header-wrap.is-sticky-header {
  z-index: 30;
  background: color-mix(in srgb, var(--bg-surface) 82%, transparent);  /* 毛玻璃 */
  backdrop-filter: blur(10px) saturate(132%);                          /* 模糊背景 */
  box-shadow: 0 8px 20px rgba(15, 23, 42, 0.1);                       /* 阴影 */
  border-radius: 0 0 16px 16px;                                        /* 底部圆角 */
  clip-path: inset(0 0 calc(var(--sticky-progress) * (100% - 80px)) 0);/* 渐进裁剪 */
}
```

## 五、Sticky 状态管理：useDetailStickyState.ts

### 5.1 职责

独立管理 `isSticky` 布尔状态，与 `DetailStickyHeroHeader` 内部的 progress 计算解耦。

### 5.2 参数

```typescript
interface UseDetailStickyStateOptions {
  scrollHostSelector?: string | (() => string | undefined);  // 默认 '.content'
  headerWrapSelector?: string;                               // 默认 '.playlist-detail-header-wrap'
  enterThreshold?: number;                                   // 默认 12px（进入吸顶阈值）
  exitThreshold?: number;                                    // 默认 4px（退出吸顶阈值）
  safetyGap?: number;                                        // 默认 32px
}
```

### 5.3 返回值

```typescript
{
  isSticky: Ref<boolean>;  // 是否吸顶
  refresh: () => void;      // 手动刷新（如切换页面时重新绑定 DOM 引用）
}
```

### 5.4 关键逻辑

- 进入 / 退出使用不同阈值，防止抖动
- 内容高度不足时不进入吸顶态
- `requestAnimationFrame` 节流
- 自动寻找最近的滚动容器（向上遍历父级查找 `overflow: auto/scroll`）

### 5.5 典型用法

```typescript
const { isSticky, refresh } = useDetailStickyState({
  scrollHostSelector: () => props.scrollHostSelector || '.content',
});

// 监听 artistId 变化时刷新
watch(
  () => props.artistId,
  () => {
    requestAnimationFrame(() => refresh());
  },
);
```

## 六、详情页 Tab 栏吸顶

### 6.1 问题

`DetailStickyHeroHeader` 吸顶折叠后，紧随其后的 Tab 栏（如"热门歌曲/专辑/MV/简介"）在滚动时会被吸顶栏遮挡。

### 6.2 解决方案

让 Tab 栏独立使用 `position: sticky; top: 80px`，紧贴折叠后的头部下方。

```css
.artist-tabs {
  position: sticky;
  top: 80px;          /* 与折叠后头部可见高度一致 */
  z-index: 20;        /* 低于头部 z-index: 30，高于内容 */
  background: var(--bg-solid);  /* 不透明背景，遮挡滚动内容 */
  padding: 18px 0 20px;  /* 与头部/内容的间距 */
  margin: 0;
}
```

### 6.3 层级关系

```
z-index: 30  →  DetailStickyHeroHeader（吸顶态）
z-index: 20  →  .artist-tabs / .user-tabs（吸附在头部下方）
z-index: 1   →  页面主体内容（正常滚动）
```

### 6.4 受影响的页面

| 页面 | Tab 容器类名 | 文件 |
|---|---|---|
| 歌手详情 | `.artist-tabs` | `ArtistDetailPage.vue` |
| 用户详情 | `.user-tabs` | `UserDetailPage.vue` |

## 七、PageHeroHeader 布局组件

作为 `DetailStickyHeroHeader` 内部的布局容器，使用 CSS Grid 实现媒体列 + 内容列左右排列。

```css
.page-hero-header {
  display: grid;
  grid-template-columns: var(--hero-media-width, 308px) minmax(0, 1fr);
  gap: 22px;
  align-items: center;
}
```

吸顶态下，`DetailStickyHeroHeader` 会覆盖此布局：

```css
.playlist-detail-header-wrap.is-sticky-header :deep(.page-hero-header) {
  grid-template-columns: 0 minmax(0, 1fr) !important;  /* 媒体列宽度坍缩为 0 */
}
```

## 九、差异对比：本方案 vs 其他实现

| 维度 | 本项目（连续式） | 二段式（CSS transition） | 简单 sticky 开关 |
|---|---|---|---|
| 视觉过渡 | 连续映射，无跳变 | 两态 CSS transition | 突变 |
| 封面视差 | 三层嵌套不同速率 | 无 | 无 |
| 毛玻璃 | `backdrop-filter` | 无 | 无 |
| 独立元素控制 | 6 组独立 inline style | 整体 height 缩放 | 仅类名切换 |
| 文档流稳定性 | `min-height` 恒定 | `height` 变化 | `sticky` 属性切换 |
| 适用场景 | 详情页 Hero 头部 | 列表详情头部 | 分类页 tabs |

## 十一、维护指南

### 11.1 新增详情页

1. 直接使用 `DetailStickyHeroHeader.vue`，无需自行实现吸顶
2. 使用 `useDetailStickyState()` 管理吸顶布尔状态
3. 如果页面有 Tab 栏，添加 `position: sticky; top: 80px; z-index: 20; background: var(--bg-solid)`
4. 优先使用 `detail-page.css` 中的公共样式

### 11.2 修改吸顶行为

- 调整折叠速度：修改 `PROGRESS_SCROLL_DISTANCE`（当前 324px）
- 调整折叠后高度：修改 clip-path 中的 `80px`
- 调整封面视差参数：修改 `updateProgress()` 中各元素的变换系数

| 文件 | 职责 |
|---|---|
