# 吸顶栏实现说明

本项目详情页的吸顶栏采用 **`position: sticky` + 显式 height transition** 方案，滚动位置通过 RAF 映射为 `--sticky-progress`（0→1），驱动背景色/毛玻璃/内容动画的连续渐变，实现类似网易云音乐的渐进式置顶缩放效果。

---

## 一、架构层次

```
页面模板（如 PlaylistDetailPage.vue）
│
├─ useDetailStickyState()               ← 管理 isSticky 布尔 + --sticky-progress 连续值
│
├─ DetailStickyHeroHeader.vue           ← 吸顶头部核心组件
│   ├─ position: sticky; top: 0         ← 始终粘性定位
│   ├─ margin-left/right: calc(-16px)   ← 始终左右铺满（正常态 + 吸顶态）
│   ├─ height: --header-full-height → 134px transition
│   ├─ --sticky-progress 驱动背景/模糊  ← color-mix + calc 连续变化
│   ├─ overflow: hidden                 ← 裁剪过渡中的内容
│   ├─ PageHeroHeader.vue               ← 内部 grid 布局（媒体列 + 内容列）
│   └─ #tabs slot → .header-tabs-area   ← 标签栏嵌入头部，flex 末尾
│
├─ App.vue (.content::before)           ← 唯一 blur 背景来源
│
└─ 各详情页组件（PlaylistDetailPage 等） ← 负责同步 --cover-bg-url 到 .content
```

## 二、核心组件：DetailStickyHeroHeader.vue

### 2.1 Props

| Prop | 类型 | 默认 | 说明 |
|---|---|---|---|
| `sticky` | `boolean` | — | 是否进入吸顶态（由外部 composable 控制） |
| `embedded` | `boolean` | — | 嵌入模式（偏移量微调） |
| `loading` | `boolean` | — | 加载中 |
| `ready` | `boolean` | — | 数据就绪 |
| `error` | `string` | — | 错误信息 |
| `loadingText` | `string` | — | 加载提示文字 |

### 2.2 Slots

| Slot | 内容 |
|---|---|
| `#media` | 封面媒体区域（通常传入 HeroCoverMedia） |
| `#title` | 标题文本 |
| `#meta` | 元数据行、描述文字等 |
| `#actions` | 操作按钮（播放全部、收藏等） |
| `#tabs` | 标签栏（如歌手页的热门歌曲/专辑/MV/简介）|

### 2.3 模板结构

```html
<header.playlist-detail-header-wrap>      ← display: flex; flex-direction: column
  <PageHeroHeader>                         ← flex: 1（吸顶态自动压缩）
    #media → .hero-media-col
      .hero-media-shell                    ← 封面壳
        <slot name="media" />              ← 实际封面元素
    #content → .hero-main-shell
      .hero-title-shell <slot name="title" />
      .hero-meta-shell <slot name="meta" />
      .hero-actions-shell--under-cover     ← 操作按钮
        <slot name="actions" />
  </PageHeroHeader>
  .header-tabs-area                        ← flex-shrink: 0，吸附底部
    <slot name="tabs" />                   ← 标签栏嵌入头部
</header>
```

动作按钮位于 `#content` slot 内的 `.hero-main-shell` 底部，吸顶态切换为 flex 行，标题和操作按钮并排。

标签栏通过 `#tabs` slot 嵌入头部，flex 吸附在 hero 内容下方。无 tabs 时 `.header-tabs-area` 不渲染，header 回退为纯 hero 模式。

## 三、滚动进度机制

### 3.1 滚动 → progress 映射

```typescript
const PROGRESS_DISTANCE = 324;  // 从完全展开到完全折叠所需的滚动像素

function update() {
  const st = scrollHost.scrollTop;
  const progress = Math.max(0, Math.min(1, st / PROGRESS_DISTANCE));
  headerEl.style.setProperty('--sticky-progress', String(progress));
}
```

- `progress = 0`：完全展开，背景透明
- `progress = 1`：完全折叠，背景纯色 `var(--bg-solid)`

### 3.2 progress 驱动的 CSS 属性

`--sticky-progress` 在 header 基础态 CSS 中驱动所有渐变动画，无需 `.is-sticky-header` 类的介入：

```css
.playlist-detail-header-wrap {
  background: color-mix(in srgb, var(--bg-solid) calc(var(--sticky-progress, 0) * 100%), transparent);
  backdrop-filter: blur(calc(var(--sticky-progress, 0) * 10px)) saturate(calc(1 + var(--sticky-progress, 0) * 0.32));
}
```

| progress | 背景 | backdrop-filter | 封面 scale | 封面 translateY |
|---|---|---|---|---|
| 0 | 完全透明 | blur(0) | 1.0（原始大小） | 0 |
| 0.5 | 50% `var(--bg-solid)` | blur(5px) | 0.9 | -8px |
| 1 | 100% `var(--bg-solid)` | blur(10px) | 0.8 | -16px |

封面缩小和上移直接作用于 `.cover-motion-shell`，跟随滚动位置实时变化，与 `.is-sticky-header` 类无关。

另外 `.hero-media-shell` 的 `opacity` 也由 progress 驱动：从 `1` 渐变为 `0.92`（轻微淡入淡出感）。

### 3.3 滚动事件处理

```typescript
function onScroll() {
  cancelAnimationFrame(rafId);
  rafId = requestAnimationFrame(update);
}
```

使用 `requestAnimationFrame` 每帧写入 1 个 CSS 变量（`--sticky-progress`），对比原实现的 6+ 组 inline style，写入量最小化。

## 四、显式高度过渡 + overflow 裁剪

### 4.1 原理

吸顶态头部的高度从 `--header-full-height`（~380px）过渡到 `134px`（含 tabs 时为 `134px`，无 tabs 时 `80px`）：

```css
.playlist-detail-header-wrap {
  height: var(--header-full-height, 380px);
  transition: height 0.3s cubic-bezier(0.33, 0, 0.1, 1);
}
.playlist-detail-header-wrap.is-sticky-header {
  height: 80px;
  overflow: hidden;
}
```

- `height` 使用 CSS transition 平滑过渡，驱动文档流高度变化
- `overflow: hidden` 裁剪过渡过程中超出 134px 的内容
- 内部元素的 opacity/transform/scale 过渡与高度过渡同步运行

### 4.2 高度变体

吸顶态高度根据页面类型自动适配：

| 页面类型 | 吸顶态高度 | 说明 |
|---|---|---|
| 无 tabs 的详情页（歌单/专辑） | `80px` | 仅 mini 行 |
| 有 tabs 的详情页（歌手） | `134px` | mini 行 + 紧凑 tabs 行 |

有 tabs 时 header 自动获得 `has-header-tabs` class，内部 `display: flex; flex-direction: column` 分配 `PageHeroHeader`（`flex: 1`）和 `.header-tabs-area`（`flex-shrink: 0`）的空间。

原实现通过 `clip-path` 裁剪多余空间，因为使用了 `min-height` 保持文档流恒定。当前方案去掉 `min-height`，改用显式 `height: 324px → 80px` 过渡，文档流跟随高度变化自然缩小，无需裁剪。

## 五、吸顶态视觉

### 5.1 吸顶态样式总览

```css
.playlist-detail-header-wrap.is-sticky-header {
  z-index: 30;
  height: 80px;                              /* 无 tabs 时 */
  overflow: hidden;
  margin: 0 calc(var(--space-4) * -1) 0;
  padding: 22px var(--space-4) 8px;
  border-radius: 0 0 16px 16px;
  box-shadow: 0 8px 20px rgba(15, 23, 42, 0.1);
}
.playlist-detail-header-wrap.is-sticky-header.has-header-tabs {
  height: 134px;                             /* 有 tabs 时 */
  padding: 22px var(--space-4) 0;
}
```

背景和 backdrop-filter 不在此处设置——它们已由 `--sticky-progress` 在基础态中驱动。

### 5.2 普通态也铺满（吸顶态与普通态统一横向布局）

header 在普通态和吸顶态的 `margin-left`/`margin-right` 保持一致，均为 `calc(var(--space-4) * -1)`，确保始终从 `.content` 的 padding 边缘铺满：

```css
.playlist-detail-header-wrap {
  margin-left: calc(var(--space-4) * -1);
  margin-right: calc(var(--space-4) * -1);
  padding-left: var(--space-4);
  padding-right: var(--space-4);
}
```

吸顶态仅改变 `margin-top`/`margin-bottom`，横向 margin 不变，因此过渡时不会出现水平方向的布局抖动。

### 5.2 封面

封面在滚动时连续缩小并上移（由 `--sticky-progress` 在基础态中驱动，与 `.is-sticky-header` 类无关）：

- `.cover-motion-shell`：`scale` 从 `1` 渐变至 `0.8`，`translateY` 从 `0` 渐变至 `-16px`
- `.hero-media-shell`：`opacity` 从 `1` 渐变至 `0.92`

到达吸顶态后，grid 进一步将封面所在列缩小至 72px，形成最终小方形：

```css
.playlist-detail-header-wrap.is-sticky-header :deep(.page-hero-header) {
  grid-template-columns: 72px minmax(0, 1fr) !important;
  gap: 14px !important;
  align-items: center;
}
.playlist-detail-header-wrap.is-sticky-header :deep(img.cover) {
  border-radius: 6px;
  opacity: 0.95;
}
```

### 5.3 标题与操作按钮

吸顶态时 `.hero-main-shell` 切换为 flex 行布局，标题 stretch 左侧，操作按钮在右侧：

```css
.playlist-detail-header-wrap.is-sticky-header :deep(.hero-main-shell) {
  display: flex;
  flex-direction: row;
  align-items: center;
}
.playlist-detail-header-wrap.is-sticky-header :deep(.hero-title-shell) {
  min-width: 0;
  flex: 1 1 auto;
}
.playlist-detail-header-wrap.is-sticky-header .hero-actions-shell--under-cover {
  display: flex;
  align-items: center;
  flex-shrink: 0;
}
```

### 5.4 标签栏（tabs-in-header）

有 `#tabs` slot 时，header 自动获得 `has-header-tabs` class。吸顶态下 tabs 区域固定在 header 底部：

```css
.playlist-detail-header-wrap.is-sticky-header .header-tabs-area {
  padding: 8px var(--space-4) var(--space-1);
  border-radius: 0 0 16px 16px;
}
```

吸顶态下 tab 按钮自动压缩为紧凑态（高度从 38px 缩至 28px，字号缩小）：

```css
.playlist-detail-header-wrap.is-sticky-header :deep(.artist-tab) {
  height: 28px;
  padding: 0 10px;
  font-size: 12px;
}
.playlist-detail-header-wrap.is-sticky-header :deep(.artist-tabs) {
  gap: 8px;
}
```

tabs 背景不单独设色，直接透出 header 由 `--sticky-progress` 驱动的渐变背景，视觉上与 header 一体。

**正常态 → 吸顶态过渡**：数据就绪后 `watch(ready)` 通过 `nextTick(remeasureHeight)` 重新测量包含 tabs 的 header 全高，`--header-full-height` 自动更新（fallback `380px`）。

## 六、背景模糊架构

> 本节的改动是本次优化核心：将原先分散在卡片和 content 两个层级的两套 blur 合并为唯一 blur 来源，消除视觉层分离感。

### 6.1 分层演变

详情页的背景模糊经历了三次简化：

| 阶段 | 模糊来源 | 层数 | 问题 |
|------|---------|------|------|
| 初始 | `content::before` + `panel::before` + `--bg-surface` 卡片面 | **3 层** | 三套独立渲染，边界无法对齐，有明显的"卡片浮在背景上"的边框感 |
| 归并 | 移除 `panel::before`，保留 `content::before` + 透明卡片 | **2 层** | 但 `--cover-bg-url` 设在卡片上，`content::before` 拿不到 → blur 无封面图，变灰色 |
| 当前 | **仅 `content::before`** 作为唯一 blur，卡片完全透明 | **1 层** | 无层分离感，封面 URL 通过 watcher 同步到 `.content` |

### 6.2 当前架构

```
App.vue — .content::before
  └── 唯一 blur 来源（360px 高，scale(1.1) 模糊，深色渐变遮罩）
  └── 封面图来自 --cover-bg-url，由详情页 watcher 同步

详情页（PlaylistDetailPage / PodcastDetailPage）
  └── 无 ::before（不再独立渲染 blur）
  └── 无 background（透明，透出 content blur）
  └── 通过 watcher 将封面 URL 同步到 .content
     └── onBeforeUnmount 清理，避免残留到下一页
```

### 6.3 卡片（`.playlist-detail-page`）当前样式

```css
.playlist-detail-page {
  width: 100%;
  box-sizing: border-box;
  position: relative;
  /* 无 background、无 border、无 border-radius、无 padding、无 overflow */
}
```

- `box-shadow` 已从该元素移除（保留在 `.panel`/`.daily-page` 上，它们未被使用）
- `background` 为透明，透出下方的 `content::before` blur
- `border-radius`/`overflow` 不再需要——卡片不再作为独立视觉层

### 6.4 封面 URL 同步机制

由于 CSS 自定义属性只能从父级向子级传递，`--cover-bg-url` 设在卡片上时 `.content` 的 `::before` 无法访问。因此需要在详情页中将变量同步到 `.content`：

```typescript
// PlaylistDetailPage.vue / PodcastDetailPage.vue
watch(
  () => playlist.value?.coverImgUrl,  // 或 hero.value.coverUrl（播客页）
  (url) => {
    const el = document.querySelector('.content') as HTMLElement | null;
    if (!el) return;
    if (url?.trim()) {
      el.style.setProperty('--cover-bg-url', `url("${url.trim()}")`);
    } else {
      el.style.removeProperty('--cover-bg-url');
    }
  },
  { immediate: true },
);

// 离开时清理，避免残留到下一个详情页
onBeforeUnmount(() => {
  const el = document.querySelector('.content') as HTMLElement | null;
  el?.style.removeProperty('--cover-bg-url');
});
```

已接入的页面：
- `PlaylistDetailPage.vue`
- `PodcastDetailPage.vue`

### 6.5 注意事项

- `fallbackCover` 等 computed 中使用的变量**必须定义在 computed 之前**，否则 `{ immediate: true }` 的 watcher 会在 setup 阶段触发 TDZ 错误
- 当不显示详情页时（回到首页、搜索结果页等），`content::before` 的 `--cover-bg-url` 回退到 `none`（由 `contentStyle` 控制），封面模糊自然消失

## 七、Sticky 状态管理：useDetailStickyState.ts

### 7.1 职责

管理 `isSticky` 布尔状态，同时每帧计算 `--sticky-progress`（0→1）并写入 header 元素。

### 7.2 参数

```typescript
interface UseDetailStickyStateOptions {
  scrollHostSelector?: string | (() => string | undefined);  // 默认 '.content'
  headerWrapSelector?: string;                               // 默认 '.playlist-detail-header-wrap'
}
```

### 7.3 返回值

```typescript
{
  isSticky: Ref<boolean>;  // 是否吸顶
  refresh: () => void;     // 手动刷新（如切换页面时重新绑定 DOM 引用）
}
```

### 7.4 关键逻辑

- RAF 节流的 scroll listener（仅写入 1 个 CSS 变量）
- `scrollTop / PROGRESS_DISTANCE` 映射为 0→1
- **首次激活**时内容高度必须足够（`scrollRange >= requiredRange` 保护），防止内容不足时强行吸顶
- **已吸顶后**仅靠 `scrollTop >= initialOffset` 判断退出（不检查 scrollRange），避免 tab 切换时内容高度波动导致反复 toggle
- 测量 header 的 `initialOffset` 判断 sticky 触发时机

### 7.5 典型用法

```typescript
const { isSticky, refresh } = useDetailStickyState();
```

## 十一、维护指南

### 11.1 新增详情页

1. 直接使用 `DetailStickyHeroHeader.vue`，无需自行实现吸顶
2. 使用 `useDetailStickyState()` 管理吸顶布尔状态
3. 如果页面有返回按钮，直接使用 `class="playlist-detail-back"`（公共样式已处理过渡）
4. **如果页面有 Tab 栏**：将 tabs 模板放入 `<template #tabs>` slot 传入 `DetailStickyHeroHeader`，tabs 会自动嵌入 header 底部吸顶。不要自行写独立 sticky 定位的 tabs
5. **Tab 内容切换**：优先使用 `v-show` 而非 `v-if/v-else-if/v-else`，避免切换 tab 时 body 高度变化导致吸顶抖动。如果使用 `v-if`，需配合 `min-height` 保证充足滚动空间
6. **设置最小高度**：为 body 容器设置 `min-height: calc(100vh - 200px)` 或类似值，确保各 tab 内容不足时仍有足够 scrollRange 维持吸顶
7. 优先使用 `detail-page.css` 中的公共样式

### 11.2 背景模糊接入指南

详情页的封面模糊背景当前由 `.content::before` 作为唯一来源提供。新增详情页时需注意：

1. **不要再给卡片自身加 `::before` blur** — 统一走 content 级 blur，避免层分离
2. **必须将封面 URL 同步到 `.content` 元素**，否则 content blur 无法获取 `--cover-bg-url`
3. 同步方式参考标准模板（需要 `watch` + `onBeforeUnmount` 清理）：

```typescript
watch(
  () => /* 封面 URL 响应式来源 */,
  (url) => {
    const el = document.querySelector('.content') as HTMLElement | null;
    if (!el) return;
    if (url?.trim()) {
      el.style.setProperty('--cover-bg-url', `url("${url.trim()}")`);
    } else {
      el.style.removeProperty('--cover-bg-url');
    }
  },
  { immediate: true },
);

onBeforeUnmount(() => {
  const el = document.querySelector('.content') as HTMLElement | null;
  el?.style.removeProperty('--cover-bg-url');
});
```

4. **注意 TDZ**：`fallbackCover` 等被 computed 引用的变量必须定义在 computed 之前（因为 `immediate: true` 的 watcher 会在 setup 阶段触发求值）
5. 页面销毁时必须在 `onBeforeUnmount` 中清理 `--cover-bg-url`，避免残留到下一个详情页

已接入的详情页：
- `PlaylistDetailPage.vue`
- `PodcastDetailPage.vue`

### 11.3 修改吸顶行为

- 调整折叠速度：修改 `PROGRESS_DISTANCE`（当前 324px）

## 十二、性能优化：移除布局型 transition

### 12.1 问题

吸顶头部从普通态过渡到吸顶态时出现卡顿。原因是 `height`/`padding`/`margin`/`grid-template-columns`/`font-size`/`width` 等触发布局重排（Layout）的 CSS transition 与滚动同时运行，浏览器每帧需执行 Layout → Paint → Composite 完整管线。

### 12.2 修复

移除所有布局型 transition，仅保留合成器友好属性：

| 移除的 transition | 位置 |
|---|---|
| `height`, `padding`, `margin` | `.playlist-detail-header-wrap` 容器 |
| `margin` | `.hero-media-shell` |
| `max-width` | `.hero-meta-shell` |
| `font-size`, `line-height`, `margin` | `.hero-title-shell .title` |
| `grid-template-columns`, `gap` | `.page-hero-header` |
| `width`, `min-width` | `.page-hero-header__media` |

### 12.3 保留的 transition

仅保留以下合成器友好属性（只触发 Composite，不触发 Layout）：

- `opacity`
- `transform`
- `border-radius`
- `box-shadow`
- `filter`

### 12.4 效果

滚动时浏览器渲染管线从 `Layout → Paint → Composite` 缩短为仅 `Composite`，消除了布局计算开销。

### 12.5 代码注释规范

修改 `DetailStickyHeroHeader.vue` 时，请遵循以下规范：

- 新增 transition 优先使用合成器友好属性（`opacity` / `transform` / `filter`）
- 如需添加布局型属性（`height` / `padding` / `margin` / `width` / `font-size` 等），必须在对应 CSS 块中添加注释说明
- 移除布局型 transition 时，将旧值以注释形式保留（例如 `/* transition 历史: height / padding / margin 曾在此 transition */`），便于后续回溯
