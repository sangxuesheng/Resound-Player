# 吸顶栏实现说明

本项目详情页的吸顶栏采用 **`position: sticky` + CSS calc 驱动**方案，滚动位置通过 RAF 映射为 `--sticky-progress`（0→1），驱动背景色/毛玻璃/内容动画的连续渐变，实现类似网易云音乐的渐进式置顶缩放效果。

---

## 一、架构层次

```
独立模式（如 PlaylistDetailPage）:
  .content::before                   ← 唯一 blur 背景来源（App.vue）
    └─ .playlist-detail-page         ← 滚动容器（overflow: clip auto + overflow-anchor: none）
         ├─ .playlist-detail-back    ← 返回按钮，progress 驱动 opacity 淡出
         ├─ DetailStickyHeroHeader   ← 吸顶头部（position: sticky; top: 0; z-index: 30）
         │    ├─ 封面 media（grid 列宽 308px → 72px）
         │    ├─ 内容列（title + meta + actions）
         │    └─ #tabs slot → 标签栏嵌入头部底部
         └─ .playlist-detail-body    ← 列表内容（flex: 1）

嵌入模式（UserSplitView 右侧面板）:
  .content::before                   ← 唯一 blur 背景来源（App.vue）
    └─ .split-stage                  ← CSS Grid 分屏容器
         ├─ .left-panel              ← 左侧用户列表
         └─ .detail-panel            ← 右侧详情面板（overflow-y: auto; height: 100%）
              └─ .detail-body-wrap   ← 内容包裹层（height: 100%）
                   └─ .playlist-detail-page.playlist-detail-page--embedded
                        ├─ .playlist-detail-back
                        ├─ DetailStickyHeroHeader.detail-sticky-header--embedded
                        └─ .playlist-detail-body
```

## 三、滚动进度机制

### 3.1 滚动 → progress 映射

```typescript
const PROGRESS_DISTANCE = embedded ? 120 : 324;

function update() {
  const st = scrollHost.scrollTop;
  const progress = Math.max(0, Math.min(1, st / PROGRESS_DISTANCE));
  host.style.setProperty('--sticky-progress', String(progress));
}
```

- `progress = 0`：完全展开，背景透明
- `progress = 1`：完全折叠，背景纯色 `var(--bg-solid)`
- 嵌入模式（UserSplitView 右侧面板）使用 120px 而非 324px，因面板高度有限

### 3.2 progress 驱动的 CSS 属性

```css
.playlist-detail-header-wrap {
  background: color-mix(in srgb, var(--bg-solid) calc(var(--sticky-progress, 0) * 100%), transparent);
  backdrop-filter: blur(calc(var(--sticky-progress, 0) * 10px)) saturate(calc(1 + var(--sticky-progress, 0) * 0.32));
}
```

| progress | 背景 | backdrop-filter | 封面 scale | 封面 translateY |
|---|---|---|---|---|
| 0 | 完全透明 | blur(0) | 1.0（原始大小） | 0 |
| 0.5 | 50% `var(--bg-solid)` | blur(5px) | 0.9 | -8px |
| 1 | 100% `var(--bg-solid)` | blur(10px) | 0.8 | -16px |

### 3.3 滚动事件处理

```typescript
function onScroll(): void {
  cancelAnimationFrame(rafId);
  rafId = requestAnimationFrame(() => update());
}
```

使用 `requestAnimationFrame` 每帧写入 1 个 CSS 变量（`--sticky-progress`），delta < 0.005 时跳过写入。

## 四、高度变化机制

### 4.1 原理

头部高度通过 CSS calc 由 `--sticky-progress` 直接计算得出，**无 height transition**。所有依赖 `--sticky-progress` 的属性在同一帧同时变化，消除 temporal mismatch：

```css
height: calc(var(--header-full-height, 380px) - var(--sticky-progress, 0) * (var(--header-full-height, 380px) - var(--collapsed-height)));
```

`transition` 仅保留纯视觉效果：

```css
transition:
  border-radius 0.3s cubic-bezier(0.33, 0, 0.1, 1),
  box-shadow 0.3s cubic-bezier(0.33, 0, 0.1, 1);
```

### 4.2 Scroll anchoring 防护

所有滚动容器（`.playlist-detail-page`）上设置：

```css
overflow-anchor: none;
```

防止头部高度变化触发 scrollTop 反馈振荡。

### 4.3 高度变体

| 页面类型 | 吸顶态高度 | 说明 |
|---|---|---|
| 无 tabs 的详情页 | `80px` | 仅 mini 行 |
| 有 tabs 的详情页（歌手/歌单/专辑/播客） | `134px` | mini 行 + 紧凑 tabs 行 |

有 tabs 时 header 自动获得 `has-header-tabs` class，`--collapsed-height` 从 80px 切换为 134px。

### 4.4 Header 高度重测

数据就绪后 `watch(ready)` → `nextTick(remeasureHeight)` 重新测量包含 tabs 的 header 全高，`--header-full-height` 自动更新（fallback 380px）。

## 六、背景模糊架构

### 6.1 唯一 blur 来源

仅 `App.vue` 的 `.content::before` 提供模糊背景，所有详情页不再各自渲染 `::before`：

```
App.vue — .content::before
  └── 唯一 blur（360px 高，scale(1.1) 模糊，深色渐变遮罩）
  └── 封面图来自 --cover-bg-url，由 useDetailStickyState 自动同步

详情页（所有）
  └── 无 ::before
  └── 无 background（透明，透出 content blur）
  └── 通过 composable 将封面 URL 同步到 .content
     └── onBeforeUnmount 自动清理
```

### 6.2 封面 URL 同步

`useDetailStickyState` 统一接管 `--cover-bg-url` 同步，各页面不再自行实现 watcher：

```typescript
// 在 composable 内部
if (coverUrl) {
  watch(coverUrl, (url) => {
    const el = document.querySelector('.content') as HTMLElement | null;
    if (!el) return;
    if (url?.trim()) {
      el.style.setProperty('--cover-bg-url', `url("${url.trim()}")`);
    } else {
      el.style.removeProperty('--cover-bg-url');
    }
  }, { immediate: true });
}

// onBeforeUnmount 时自动清理
contentEl?.style.removeProperty('--cover-bg-url');
```

### 6.3 `--blur-opacity` 同步

当 scroll host 不是 `.content` 自身时（即独立滚动容器场景），每帧同步 `--blur-opacity`，使 blur 随滚动淡出：

```typescript
function syncBlurOpacity(progress: number): void {
  const contentEl = document.querySelector('.content');
  if (contentEl) {
    const blurOpacity = Math.max(0, 1 - progress * 1.5);
    contentEl.style.setProperty('--blur-opacity', String(blurOpacity));
  }
}
```

## 七、统一状态管理：useDetailStickyState.ts

### 7.1 API

```typescript
export function useDetailStickyState(
  coverUrl?: ComputedRef<string>,   // 可选，封面 URL，自动同步 blur 背景
  embedded = false,                  // 可选，是否嵌入 UserSplitView
): { refresh: () => void }
```

- **无配置参数**：scroll host 固定为 `.playlist-detail-page`（独立模式）或 `.detail-panel`（嵌入模式），sticky class target 固定为 `.playlist-detail-header-wrap`
- **自动生命周期管理**：mount 时绑定 scroll 事件，unmount 时清理 `--sticky-progress`、`--blur-opacity`、`is-sticky-header` class、`--cover-bg-url`

### 7.2 典型用法

```typescript
// 独立模式
const { refresh } = useDetailStickyState(
  computed(() => playlist.value?.coverImgUrl?.trim() || ''),
);

// 嵌入模式
const { refresh } = useDetailStickyState(
  computed(() => playlist.value?.coverImgUrl?.trim() || ''),
  !!props.embedded,
);
```

### 7.3 关键常量

| 场景 | PROGRESS_DISTANCE | scroll host |
|---|---|---|
| 独立模式 | 324px | `.playlist-detail-page` |
| 嵌入模式 | 120px | `.detail-panel` |

### 7.4 已接入的详情页

### 9.3 数据流

UserPanel 通过 `:embedded="true"` 传递给内嵌的详情页（PlaylistDetailPage / AlbumDetailPage / PodcastDetailPage），composable 据此切换 scroll host 为 `.detail-panel` 并缩小 PROGRESS_DISTANCE。

### 9.4 嵌入模式下的虚拟滚动与无限加载

PlaylistDetailPage 使用虚拟滚动渲染歌曲列表，通过监听滚动容器的 `scroll` 事件计算 `songListScrollTop`，确定当前可视区应渲染的歌曲范围。嵌入模式下需同步调整以下机制：

#### 9.4.1 虚拟滚动滚动宿主

独立模式下虚拟滚动监听 `.playlist-detail-page` 的 scroll 事件。嵌入模式下 `playlist-detail-page` 设置 `overflow: visible`，不自已滚动，需改为监听 `.detail-panel`：

```typescript
onMounted(() => {
  const scrollHostSelector = props.embedded ? '.detail-panel' : '.playlist-detail-page';
  const page = document.querySelector(scrollHostSelector);
  if (page) {
    cachedPageEl = page as HTMLElement;
    page.addEventListener('scroll', pageScrollListener, { passive: true });
  }
});
```

#### 9.4.2 IntersectionObserver root

无限滚动（`loadMoreSongs`）通过 `IntersectionObserver` 监听 sentinel 元素触发。默认 `root: null`（视口），但嵌入模式下 sentinel 位于 `.detail-panel` 溢出区域内，不在视口中。需要指定 `root`：

```typescript
const scrollRoot = props.embedded
  ? document.querySelector('.detail-panel')
  : document.querySelector('.playlist-detail-page');

scrollObserver = new IntersectionObserver(callback, { root: scrollRoot, rootMargin: '400px' });
```

#### 9.4.3 `currentToken` 作用域

`fetchDetail` 和 `loadMoreSongs` 使用 `currentToken` 防止并发请求的响应污染。原代码将 `currentToken` 声明为 `fetchDetail` 内的局部 `const`，`loadMoreSongs` 引用时实为 `undefined`，导致 `undefined !== fetchToken` 恒为 `true`，`loadMoreSongs` 始终 return。修复为模块级变量：

```typescript
let fetchToken = 0;
let currentToken = 0;  // 模块级，两个函数都可访问

async function fetchDetail(id: number) {
  currentToken = ++fetchToken;  // 赋值而非 const 声明
  // ...
}
```

#### 9.4.4 初始歌曲加载上限

`getPlaylistDetail(id, s, cookie)` 的 `s` 参数为初始返回歌曲数上限，原为 `s=100`。对于超过 100 首歌的歌单需靠无限滚动加载剩余歌曲。批量加载至 `s=1000`，减少对无限滚动的依赖：

```typescript
const { data } = await getPlaylistDetail(id, 1000, userStore.loginCookie || undefined);
```

## 十、Temporal Dead Zone 防范

`useDetailStickyState` 内部立即执行 `watch(coverUrl, ..., { immediate: true })`，在 setup 阶段就会访问 `coverUrl.value`。因此 **coverUrl 依赖的所有变量必须定义在 composable 调用之前**。

错误示例（PodcastDetailPage 曾遇到的问题）：

```typescript
// ❌ hero 在 useDetailStickyState 之后定义
const { refresh } = useDetailStickyState(computed(() => hero.value.coverUrl?.trim() || ''));
const hero = computed(() => ...);
```

正确顺序：

```typescript
// ✓ hero 在 useDetailStickyState 之前定义
const hero = computed(() => ...);
const { refresh } = useDetailStickyState(computed(() => hero.value.coverUrl?.trim() || ''));
```

## 十三、维护指南

### 13.1 新增详情页

1. 直接使用 `DetailStickyHeroHeader.vue`，无需自行实现吸顶
2. 模板根容器使用 `class-name="playlist-detail-page"`，获得共享滚动容器样式
3. 使用 `useDetailStickyState(computed(() => coverUrl))`，blur 同步自动处理
4. 如果页面有返回按钮，使用 `class="playlist-detail-back"`（公共样式已处理过渡）
5. **如果页面有 Tab 栏**：将 tabs 放入 `<template #tabs>` slot，tabs 自动嵌入 header 底部吸顶
6. 如果页面会嵌入 UserSplitView，模板加 `:class="[embedded && 'playlist-detail-page--embedded']"`，composable 传 `!!props.embedded`
7. 为 body 容器设置 `flex: 1; min-height: 0`（引用 `detail-page.css` 的 `.playlist-detail-body`）
8. Tab 内容切换优先使用 `v-show`，避免切换时 body 高度变化导致吸顶抖动

### 13.2 修改吸顶行为

- 调整折叠速度：修改 `PROGRESS_DISTANCE`（独立 324px / 嵌入 120px）
- 调整无 tabs 折叠高度：修改 `.is-sticky-header` 的 `height: 80px`
- 调整有 tabs 折叠高度：修改 `.is-sticky-header.has-header-tabs` 的 `height: 134px`
- 调整嵌入模式面板 padding：同时更新 `.detail-panel` 的 `padding`、`.detail-sticky-header--embedded` 的 `margin`、组合选择器的 `width: calc(100% + 36px)`

### 13.4 维护规范

- **禁止为布局型属性**（`height`、`padding`、`margin`、`grid-template-columns`、`font-size` 等）添加 CSS transition
- 仅允许为纯视觉效果添加 transition（`opacity`、`transform`、`filter`、`border-radius`、`box-shadow`）
- 新增详情页时确保滚动容器上设置了 `overflow-anchor: none`
- 封面 URL 依赖的变量必须定义在 `useDetailStickyState` 之前（规避 TDZ）
- 离开页面时 composable 自动清理 `--cover-bg-url`
