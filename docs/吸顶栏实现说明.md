# 吸顶栏实现说明

本项目详情页的吸顶栏采用 **`position: sticky` + 显式 height transition** 方案，滚动位置通过 RAF 映射为 `--sticky-progress`（0→1），驱动背景色/毛玻璃/内容动画的连续渐变，实现类似网易云音乐的渐进式置顶缩放效果。

---

## 一、架构层次

```
PlaylistDetailPage.vue
│
├─ useDetailStickyState({ scrollHostSelector: '.playlist-detail-page' })
│                                    ← 管理 isSticky + --sticky-progress，监听详情页滚动
│
├─ .playlist-detail-page               ← 滚动容器（overflow: clip auto）
│   │
│   ├─ DetailStickyHeroHeader.vue      ← 吸顶头部核心组件
│   │   ├─ position: sticky; top: 0    ← 在 .playlist-detail-page 内粘性定位
│   │   ├─ margin-left/right: -16px    ← 左右铺满（单滚动容器无需穿透到 .content）
│   │   ├─ height: --header-full-height → 80px（不透明）
│   │   ├─ --sticky-progress 驱动背景/模糊
│   │   ├─ PageHeroHeader.vue          ← 内部 grid（媒体列 + 内容列）
│   │   └─ #tabs slot → .header-tabs-area
│   │
│   ├─ .playlist-detail-body (flex:1)
│   │   └─ .song-list--virtual         ← 虚拟滚动容器（overflow: visible）
│   │       └─ spacer + 绝对定位项
│   │
│   └─ App.vue (.content::before)      ← 唯一 blur 背景来源
```

## 二、核心组件：DetailStickyHeroHeader.vue

### 2.1 Props

| Prop | 类型 | 默认 | 说明 |
|---|---|---|---|
| `sticky` | `boolean` | — | 是否进入吸顶态（由外部 composable 控制） |
| `embedded` | `boolean` | — | 嵌入模式（偏移量微调） |
| `loading` | `boolean` | — | 加载中 |
| `ready` | `boolean` | — | 数据就绪 |
| `error` | `string` | — | 错误信息 |
| `loadingText` | `string` | — | 加载提示文字 |

### 2.2 Slots

| Slot | 内容 |
|---|---|
| `#media` | 封面媒体区域（通常传入 HeroCoverMedia） |
| `#title` | 标题文本 |
| `#meta` | 元数据行、描述文字等 |
| `#actions` | 操作按钮（播放全部、收藏等） |
| `#tabs` | 标签栏（如歌手页的热门歌曲/专辑/MV/简介）|

### 2.3 模板结构

```html
<header.playlist-detail-header-wrap>      ← display: flex; flex-direction: column
  <PageHeroHeader>                         ← flex: 1（吸顶态自动压缩）
    #media → .hero-media-col
      .hero-media-shell                    ← 封面壳
        <slot name="media" />              ← 实际封面元素
    #content → .hero-main-shell
      .hero-title-shell <slot name="title" />
      .hero-meta-shell <slot name="meta" />
      .hero-actions-shell--under-cover     ← 操作按钮
        <slot name="actions" />
  </PageHeroHeader>
  .header-tabs-area                        ← flex-shrink: 0，吸附底部
    <slot name="tabs" />                   ← 标签栏嵌入头部
</header>
```

动作按钮位于 `#content` slot 内的 `.hero-main-shell` 底部。普通态按文档流排列在 meta 下方；吸顶态（`.is-sticky-header`）切换为 grid 布局，标题和操作按钮同行并排，meta 在第二行。

标签栏通过 `#tabs` slot 嵌入头部，flex 吸附在 hero 内容下方。无 tabs 时 `.header-tabs-area` 不渲染，header 回退为纯 hero 模式。

## 三、滚动进度机制

### 3.1 滚动 → progress 映射

```typescript
const PROGRESS_DISTANCE = 324;  // 从完全展开到完全折叠所需的滚动像素

function update() {
  const st = scrollHost.scrollTop;
  const progress = Math.max(0, Math.min(1, st / PROGRESS_DISTANCE));
  headerEl.style.setProperty('--sticky-progress', String(progress));
}
```

- `progress = 0`：完全展开，背景透明
- `progress = 1`：完全折叠，背景纯色 `var(--bg-solid)`

### 3.2 progress 驱动的 CSS 属性

`--sticky-progress` 在 header 基础态 CSS 中驱动所有渐变动画，无需 `.is-sticky-header` 类的介入：

```css
.playlist-detail-header-wrap {
  background: color-mix(in srgb, var(--bg-solid) calc(var(--sticky-progress, 0) * 100%), transparent);
  backdrop-filter: blur(calc(var(--sticky-progress, 0) * 10px)) saturate(calc(1 + var(--sticky-progress, 0) * 0.32));
}
```

| progress | 背景 | backdrop-filter | 封面 scale | 封面 translateY |
|---|---|---|---|---|
| 0 | 完全透明 | blur(0) | 1.0（原始大小） | 0 |
| 0.5 | 50% `var(--bg-solid)` | blur(5px) | 0.9 | -8px |
| 1 | 100% `var(--bg-solid)` | blur(10px) | 0.8 | -16px |

封面缩小和上移直接作用于 `.cover-motion-shell`，跟随滚动位置实时变化，与 `.is-sticky-header` 类无关。

另外 `.hero-media-shell` 的 `opacity` 也由 progress 驱动：从 `1` 渐变为 `0.92`（轻微淡入淡出感）。

### 3.3 滚动事件处理

```typescript
function onScroll() {
  cancelAnimationFrame(rafId);
  rafId = requestAnimationFrame(update);
}
```

使用 `requestAnimationFrame` 每帧写入 1 个 CSS 变量（`--sticky-progress`），对比原实现的 6+ 组 inline style，写入量最小化。

## 四、显式高度过渡 + overflow 裁剪

### 4.1 原理

吸顶态头部的高度从 `--header-full-height`（~380px）过渡到 `134px`（含 tabs 时为 `134px`，无 tabs 时 `80px`）。采用**双层过渡时长**方案：

| 滚动状态 | height transition | 目的 |
|---------|-------------------|------|
| 滚动中 | 60ms | 消除跳动感，同时极短时长确保不卡顿 |
| 停止滚动后 200ms | 350ms | 视觉平滑过渡 |

#### 60ms 过渡（滚动中，始终启用）

```css
.playlist-detail-header-wrap {
  height: var(--header-full-height, 380px);
  transition:
    height 0.06s cubic-bezier(0.33, 0, 0.1, 1),
    border-radius 0.3s cubic-bezier(0.33, 0, 0.1, 1),
    box-shadow 0.3s cubic-bezier(0.33, 0, 0.1, 1);
}
.playlist-detail-header-wrap.is-sticky-header {
  height: 80px;
  overflow: hidden;
}
```

60ms 仅约 4 帧，每帧布局开销 ~1-3ms，远低于 60fps 的 16ms 帧预算。

#### 350ms 升级（滚动停止后）

`useDetailStickyState` composable 在每次 scroll 事件中移除 `sticky-transition-ready` 类，并在 200ms 无 scroll 后重新添加：

```typescript
// useDetailStickyState.ts
function onScroll(): void {
  cancelAnimationFrame(rafId);
  rafId = requestAnimationFrame(update);
  // 滚动中关闭平滑 transition（降级为默认 60ms）
  headerEl?.classList.remove('sticky-transition-ready');
  // 200ms 无 scroll 后重新启用
  clearTimeout(scrollStopTimer);
  scrollStopTimer = setTimeout(() => {
    headerEl?.classList.add('sticky-transition-ready');
  }, 200);
}
```

```css
.playlist-detail-header-wrap.sticky-transition-ready {
  transition:
    height 0.35s cubic-bezier(0.33, 0, 0.1, 1),
    border-radius 0.3s cubic-bezier(0.33, 0, 0.1, 1),
    box-shadow 0.3s cubic-bezier(0.33, 0, 0.1, 1);
}
```

两层保护确保不卡顿往复：
- **activatedAt 锁（300ms）**：防止 isSticky 在激活后快速振荡
- **滚动中断保护**：过渡进行中若用户再次滚动，`sticky-transition-ready` 立即移除，transition 切回 60ms

- `overflow: hidden` 裁剪过渡过程中超出 80px 的内容
- 内部元素的 opacity/transform/scale 过渡与高度过渡同步运行
- `will-change: height` 提示浏览器预分配合成层

### 4.2 高度变体

吸顶态高度根据页面类型自动适配：

| 页面类型 | 吸顶态高度 | 说明 |
|---|---|---|
| 无 tabs 的详情页（歌单/专辑） | `80px` | 仅 mini 行 |
| 有 tabs 的详情页（歌手） | `134px` | mini 行 + 紧凑 tabs 行 |

有 tabs 时 header 自动获得 `has-header-tabs` class，内部 `display: flex; flex-direction: column` 分配 `PageHeroHeader`（`flex: 1`）和 `.header-tabs-area`（`flex-shrink: 0`）的空间。

原实现通过 `clip-path` 裁剪多余空间，因为使用了 `min-height` 保持文档流恒定。当前方案去掉 `min-height`，改用显式 `height: 324px → 80px` 过渡，文档流跟随高度变化自然缩小，无需裁剪。

## 五、吸顶态视觉

### 5.1 吸顶态样式总览

```css
.playlist-detail-header-wrap.is-sticky-header {
  z-index: 30;
  height: 80px;                              /* 无 tabs 时 */
  overflow: hidden;
  margin: 0 calc(var(--space-4) * -1) 0;
  padding: 22px var(--space-4) 8px;
  border-radius: 0 0 16px 16px;
  clip-path: inset(0 round 0 0 16px 16px);   /* 独立裁切圆角，避免父级 overflow:clip 裁掉 */
  box-shadow: 0 8px 20px var(--shadow-sticky);
}
.playlist-detail-header-wrap.is-sticky-header.has-header-tabs {
  height: 134px;                             /* 有 tabs 时 */
  padding: 22px var(--space-4) 0;
}
```

背景和 backdrop-filter 不在此处设置——它们已由 `--sticky-progress` 在基础态中驱动。

### 5.2 普通态也铺满（吸顶态与普通态统一横向布局）

header 在普通态和吸顶态的 `margin-left`/`margin-right` 保持一致，均为 `calc(var(--space-4) * -1)`，确保始终从 `.content` 的 padding 边缘铺满：

```css
.playlist-detail-header-wrap {
  margin-left: calc(var(--space-4) * -1);
  margin-right: calc(var(--space-4) * -1);
  padding-left: var(--space-4);
  padding-right: var(--space-4);
}
```

吸顶态仅改变 `margin-top`/`margin-bottom`，横向 margin 不变，因此过渡时不会出现水平方向的布局抖动。

> **PlaylistDetailPage 场景**：由于 `.playlist-detail-page` 自身已通过 `margin-left: -16px; width: calc(100% + 32px)` 延伸至视口边缘，header 的负 margin 会导致超出父级裁切边界，连带左下/右下圆角被裁掉。因此在 PlaylistDetailPage 中通过 scoped CSS 将 header 的 `margin-left/margin-right` 重置为 0，配合 `clip-path: inset(0 round 0 0 16px 16px)` 保证圆角可见。

### 5.2 封面

封面在滚动时连续缩小并上移（由 `--sticky-progress` 在基础态中驱动，与 `.is-sticky-header` 类无关）：

- `.cover-motion-shell`：`scale` 从 `1` 渐变至 `0.8`，`translateY` 从 `0` 渐变至 `-16px`
- `.hero-media-shell`：`opacity` 从 `1` 渐变至 `0.92`

到达吸顶态后，grid 进一步将封面所在列缩小至 72px，形成最终小方形：

```css
.playlist-detail-header-wrap.is-sticky-header :deep(.page-hero-header) {
  grid-template-columns: 72px minmax(0, 1fr) !important;
  gap: 14px !important;
  align-items: center;
}
.playlist-detail-header-wrap.is-sticky-header :deep(img.cover) {
  border-radius: 6px;
  opacity: 0.95;
}
```

### 5.3 标题与操作按钮

普通态 `.hero-main-shell` 为 flex column 布局（38585db 样式），标题在上、meta 居中、操作按钮在底部自然排列。

吸顶态（`.is-sticky-header`）切换为 grid 布局，title 与 actions 同行并排，meta 在第二行：

```css
.playlist-detail-header-wrap.is-sticky-header :deep(.hero-main-shell) {
  display: grid;
  grid-template-columns: 1fr auto;
  grid-template-rows: auto auto;
  grid-template-areas:
    "title actions"
    "meta   meta";
}
.playlist-detail-header-wrap.is-sticky-header :deep(.hero-title-shell) {
  grid-area: title;
}
.playlist-detail-header-wrap.is-sticky-header :deep(.hero-actions-shell--under-cover) {
  grid-area: actions;
  align-self: start;
}
.playlist-detail-header-wrap.is-sticky-header :deep(.hero-meta-shell) {
  grid-area: meta;
}
```

### 5.4 标签栏（tabs-in-header）

有 `#tabs` slot 时，header 自动获得 `has-header-tabs` class。吸顶态下 tabs 区域固定在 header 底部：

```css
.playlist-detail-header-wrap.is-sticky-header .header-tabs-area {
  padding: 8px var(--space-4) var(--space-1);
  border-radius: 0 0 16px 16px;
}
```

吸顶态下 tab 按钮自动压缩为紧凑态（高度从 38px 缩至 28px，字号缩小）：

```css
.playlist-detail-header-wrap.is-sticky-header :deep(.artist-tab) {
  height: 28px;
  padding: 0 10px;
  font-size: 12px;
}
.playlist-detail-header-wrap.is-sticky-header :deep(.artist-tabs) {
  gap: 8px;
}
```

tabs 背景不单独设色，直接透出 header 由 `--sticky-progress` 驱动的渐变背景，视觉上与 header 一体。

**正常态 → 吸顶态过渡**：数据就绪后 `watch(ready)` 通过 `nextTick(remeasureHeight)` 重新测量包含 tabs 的 header 全高，`--header-full-height` 自动更新（fallback `380px`）。

## 六、背景模糊架构

> 本节的改动是本次优化核心：将原先分散在卡片和 content 两个层级的两套 blur 合并为唯一 blur 来源，消除视觉层分离感。

### 6.1 分层演变

详情页的背景模糊经历了三次简化：

| 阶段 | 模糊来源 | 层数 | 问题 |
|------|---------|------|------|
| 初始 | `content::before` + `panel::before` + `--bg-surface` 卡片面 | **3 层** | 三套独立渲染，边界无法对齐，有明显的"卡片浮在背景上"的边框感 |
| 归并 | 移除 `panel::before`，保留 `content::before` + 透明卡片 | **2 层** | 但 `--cover-bg-url` 设在卡片上，`content::before` 拿不到 → blur 无封面图，变灰色 |
| 当前 | **仅 `content::before`** 作为唯一 blur，卡片完全透明 | **1 层** | 无层分离感，封面 URL 通过 watcher 同步到 `.content` |

### 6.2 当前架构

```
App.vue — .content::before
  └── 唯一 blur 来源（360px 高，scale(1.1) 模糊，深色渐变遮罩）
  └── 封面图来自 --cover-bg-url，由详情页 watcher 同步

详情页（PlaylistDetailPage / PodcastDetailPage）
  └── 无 ::before（不再独立渲染 blur）
  └── 无 background（透明，透出 content blur）
  └── 通过 watcher 将封面 URL 同步到 .content
     └── onBeforeUnmount 清理，避免残留到下一页
```

### 6.3 滚动容器（`.playlist-detail-page`）当前样式

```css
.playlist-detail-page {
  display: flex;
  flex-direction: column;
  height: 100%;
  min-height: 0;
  overflow: clip auto;
  overflow-clip-margin: 16px;
  margin-left: calc(var(--space-4) * -1);
  margin-right: calc(var(--space-4) * -1);
  width: calc(100% + var(--space-4) * 2);
}
```

- `overflow: clip auto` 使页面自身成为唯一滚动容器：纵向溢出自适应滚动，横向溢出被 clip 裁切
- `overflow-clip-margin: 16px` 扩展裁切边界，配合 `clip-path: inset(0 round 0 0 16px 16px)` 确保底部圆角不被父级裁掉
- `margin-left/right: -16px` + `width: calc(100% + 32px)` 将元素延伸至 `.content` 的 padding 区域，使 `overflow-x: clip` 裁切边界对齐视口边缘
- `display: flex; flex-direction: column; height: 100%;` 结合 `flex: 1` 子元素，撑满 `.content` 的同时约束虚拟滚动容器高度
- 无 `background`（透明，透出 `.content::before` blur）
- `overflow-x: clip` 替代传统的 `overflow-x: hidden`，避免与 `overflow-y: auto` 的 `visible→auto` 隐式转换问题

### 6.4 封面 URL 同步机制

由于 CSS 自定义属性只能从父级向子级传递，`--cover-bg-url` 设在卡片上时 `.content` 的 `::before` 无法访问。因此需要在详情页中将变量同步到 `.content`：

```typescript
// PlaylistDetailPage.vue / PodcastDetailPage.vue
watch(
  () => playlist.value?.coverImgUrl,  // 或 hero.value.coverUrl（播客页）
  (url) => {
    const el = document.querySelector('.content') as HTMLElement | null;
    if (!el) return;
    if (url?.trim()) {
      el.style.setProperty('--cover-bg-url', `url("${url.trim()}")`);
    } else {
      el.style.removeProperty('--cover-bg-url');
    }
  },
  { immediate: true },
);

// 离开时清理，避免残留到下一个详情页
onBeforeUnmount(() => {
  const el = document.querySelector('.content') as HTMLElement | null;
  el?.style.removeProperty('--cover-bg-url');
});
```

已接入的页面：
- `PlaylistDetailPage.vue`
- `PodcastDetailPage.vue`

### 6.5 `--blur-opacity` 同步（仅 PlaylistDetailPage）

PlaylistDetailPage 中，`.content` 不再滚动（`.playlist-detail-page` 是滚动容器），因此 `.content::before` 的 blur 背景始终固定在顶部，不会随滑动消失。

**解决方案**：在 `useDetailStickyState` composable 中，当检测到 scroll host 不是 `.content` 时（即 PlaylistDetailPage 场景），每帧同步 `--blur-opacity` 到 `.content` 元素：

```typescript
// useDetailStickyState.ts — syncBlurOpacity()
function syncBlurOpacity(progress: number): void {
  if (!shouldSyncBlur) return;  // 仅非 .content scroll host 时执行
  const contentEl = document.querySelector('.content');
  if (contentEl) {
    const blurOpacity = Math.max(0, 1 - progress * 1.5);
    contentEl.style.setProperty('--blur-opacity', String(blurOpacity));
  }
}
```

App.vue 中对应的 CSS 将 `<itt>` 改为读取 CSS 变量：

```css
.content.content--hero-sticky::before {
  opacity: var(--blur-opacity, 1);
}
```

当 `progress` 从 0→1 时，`--blur-opacity` 从 1→0（乘 1.5 系数使 blur 比 header 背景稍快淡出），其他详情页不受影响（走 fallback `1`）。

### 6.6 注意事项

- `fallbackCover` 等 computed 中使用的变量**必须定义在 computed 之前**，否则 `{ immediate: true }` 的 watcher 会在 setup 阶段触发 TDZ 错误
- 当不显示详情页时（回到首页、搜索结果页等），`content::before` 的 `--cover-bg-url` 回退到 `none`（由 `contentStyle` 控制），封面模糊自然消失

## 七、Sticky 状态管理：useDetailStickyState.ts

### 7.1 职责

每帧计算 `--sticky-progress`（0→1）并写入滚动容器，驱动所有 CSS 渐变动画。同时可选同步 `is-sticky-header` class（通过 `stickyClassTarget` 选项）。

### 7.2 参数

```typescript
interface UseDetailStickyStateOptions {
  scrollHostSelector?: string | (() => string | undefined);  // 默认 '.content'
  headerWrapSelector?: string;                               // 默认 '.playlist-detail-header-wrap'
  stickyClassTarget?: string;                                // 可选：progress >= 0.95 时自动添加 is-sticky-header
}
```

**在 PlaylistDetailPage 中**，scrollHostSelector 传入了 `.playlist-detail-page` 而非默认的 `.content`，因为虚拟滚动重构后 `.playlist-detail-page` 自身即为滚动容器：

```typescript
const { refresh } = useDetailStickyState({
  scrollHostSelector: () => '.playlist-detail-page',
  stickyClassTarget: '.playlist-detail-header-wrap',
});
```

对于其他不使用虚拟滚动的详情页（Album / Artist / Language / UserDetailPage），仍使用默认的 `.content` scroll host。

### 7.3 返回值

```typescript
{
  refresh: () => void;  // 手动刷新（如切换页面时重新绑定 DOM 引用）
}
```

> `isSticky` 布尔状态已移除，改为纯 `--sticky-progress` 连续驱动 + `stickyClassTarget` class 同步。

### 7.4 关键逻辑

- **RAF 节流**的 scroll listener（仅写入 1 个 CSS 变量 `--sticky-progress`）
- **Delta 过滤**：`--sticky-progress` 变化量 < 0.005 时跳过写入，减少 inline style 写入频率
- **progress 计算**：`progress = clamp(scrollTop / PROGRESS_DISTANCE, 0, 1)`，从 `initialOffset`（=0）开始计算
- **`is-sticky-header` class**：当 `stickyClassTarget` 设置时，progress >= 0.95 自动添加，progress < 0.95 自动移除
- **Scroll host 动态检测**：mount 时检测滚动容器是否不是 `.content`，决定是否同步 `--blur-opacity`（仅 PlaylistDetailPage 需要）
- **组件卸载时**清理 `--sticky-progress`、`--blur-opacity`（仅 PlaylistDetailPage 场景）、`is-sticky-header` class

### 7.5 典型用法

```typescript
const { refresh } = useDetailStickyState();
```

## 八、返回按钮防闪烁方案

### 8.1 历史演进

| 阶段 | 实现 | 问题 |
|------|------|------|
| 初始 | `v-if="!isSticky"` + CSS transition | 从 DOM 移除导致布局突变 → 闪烁 |
| 方案一 | `:class` + `.back-fade` 用 `max-height/margin/padding` 坍缩 | 布局切换仍触发布局滚动锚定调整 → 闪烁 |
| 当前 | 留在文档流 + `.back-fade` 仅用 `opacity` + activatedAt 冷启动锁 | 消除布局变化 + 防溢出 |

### 8.2 根因与原理

**根因**：`.back-fade` 通过 `max-height: 0 / margin: 0 / padding: 0` 坍缩空间。当 `isSticky` 变为 `true` 时，返回按钮收缩 + header 高度变化（380px→80px）叠加 → 滚动容器总高度骤减 ~358px → 浏览器 scroll anchoring 自动调整 `scrollTop` → 触发新一轮 scroll 事件 → `isSticky` 短暂翻转 → 按钮"闪一下"。

**修复策略：两重保护**

1. **`.back-fade` 仅操作 `opacity`**：`opacity: 0; pointer-events: none` 不触发布局变化，消除滚动锚定干扰源
2. **activatedAt 冷启动锁**：在 `useDetailStickyState` 中，吸顶激活后 300ms 内不允许 `isSticky` 退出，防止 header 高度变化导致的 scrollTop 钳位误触发

```html
<div :class="['playlist-detail-back', { 'back-fade': isSticky }]">
  <button class="back-btn" @click="emit('back')">← {{ props.backLabel }}</button>
</div>
```

```css
.playlist-detail-back,
.artist-detail-back {
  max-height: 58px;
  overflow: visible;
  padding-top: 2px;
  margin: calc(var(--space-2) - 2px) 0 var(--space-4);
}
.playlist-detail-back.back-fade,
.artist-detail-back.back-fade {
  opacity: 0;
  pointer-events: none;
}
```

### 8.3 涉及的页面

- `PlaylistDetailPage.vue`
- `AlbumDetailPage.vue`
- `ArtistDetailPage.vue`
- `LanguageDetailPage.vue`

### 8.4 维护要点

- 新增详情页时，返回按钮直接使用 `class="playlist-detail-back"`（公共样式已处理）
- **不要在 `.back-fade` 中添加布局触发属性**（`max-height` / `margin` / `padding` / `display` / `overflow` / `visibility`），否则滚动锚定问题会复现
- 如需调整返回按钮样式，修改 `detail-page.css` 中的 `.playlist-detail-back` 规则即可

## 九、虚拟滚动与吸顶态共存架构

### 9.1 问题

引入虚拟滚动后，`.playlist-detail-page` 内的 `.song-list--virtual` 原本使用 `overflow-y: auto` 独立滚动。这导致：

- `.content` 不再溢出（`.playlist-detail-page` 被 `height: 100%` 约束）
- `useDetailStickyState` 监听 `.content` 的 scroll 事件永远不触发
- `position: sticky` 无对应滚动容器，吸顶态失效

### 9.2 解决方案

放弃 `.song-list--virtual` 的内部滚动，改为 **`.playlist-detail-page` 作为唯一滚动容器**：

```
.content（overflow-y: auto，但子元素 height:100% 不溢出 → 无滚动）
  └─ .playlist-detail-page（overflow: clip auto, height: 100%）
       ├─ DetailStickyHeroHeader（position: sticky; top: 0）← 在此容器内吸顶
       └─ .playlist-detail-body（flex: 1）
            └─ .song-list--virtual（overflow: visible; flex: 1）
                 └─ spacer（height: totalListHeight）← 超高溢出 → 触发页面滚动
```

### 9.3 关键技术细节

**溢出传播链**：spacer 的 `totalListHeight` > `.song-list--virtual` 的 flex 约束高度 → `.song-list--virtual` 的 `overflow: visible` 让溢出差可见 → 沿 flex 链向上传播到 `.playlist-detail-page` → `overflow-y: auto` 创建滚动条。

**虚拟滚动位置追踪**：虚拟滚动不再使用内部 `@scroll` 事件，而是监听 `.playlist-detail-page` 的 scroll，通过 `page.scrollTop - list.offsetTop` 计算列表内的滚动位置。引用在 `onMounted` 中缓存，避免每帧 `document.querySelector`：

```typescript
/* 缓存滚动宿主引用，避免每帧 document.querySelector */
let cachedPageEl: HTMLElement | null = null;

function onSongListScroll(): void {
  const list = songListRef.value;
  if (cachedPageEl && list) {
    songListScrollTop.value = Math.max(0, cachedPageEl.scrollTop - list.offsetTop);
  }
}

onMounted(() => {
  const page = document.querySelector('.playlist-detail-page');
  if (page) {
    cachedPageEl = page as HTMLElement;
    page.addEventListener('scroll', () => onSongListScroll(), { passive: true });
  }
});
```

**裁切边界**：虚拟列表绝对定位项在视口外的区域由 `.playlist-detail-page` 的 `overflow-y: auto` 统一裁切，`.song-list--virtual` 不需要独立的 `overflow: clip`。

### 9.4 更新要点（维护参考）

| 项目 | 说明 |
|------|------|
| 滚动容器 | `.playlist-detail-page`（`overflow: clip auto` + `overflow-clip-margin: 16px`） |
| scroll host 选择器 | `useDetailStickyState({ scrollHostSelector: '.playlist-detail-page' })` |
| 虚拟列表容器 | `.song-list--virtual` 无 `overflow` 限制（`overflow: visible`） |
| 虚拟列表高度 | 由 flex: 1 约束（配合 `min-height: 400px`） |
| 位置计算 | `page.scrollTop - list.offsetTop`（每帧计算，自动适应吸顶态高度变化） |
| 圆角方案 | `clip-path: inset(0 round 0 0 16px 16px)` — 独立于父级 `overflow: clip` 的裁切上下文 |
| header 铺满 | `.playlist-detail-page` 的负 margin 已延伸至视口边缘，PlaylistDetailPage 中 header 负 margin 被 scoped CSS 强制归零（`margin-left: 0 !important`），避免裁切圆角 |

### 13.1 新增详情页

7. 优先使用 `detail-page.css` 中的公共样式

### 13.2 背景模糊接入指南

详情页的封面模糊背景当前由 `.content::before` 作为唯一来源提供。新增详情页时需注意：

1. **不要再给卡片自身加 `::before` blur** — 统一走 content 级 blur，避免层分离
2. **必须将封面 URL 同步到 `.content` 元素**，否则 content blur 无法获取 `--cover-bg-url`
3. 同步方式参考标准模板（需要 `watch` + `onBeforeUnmount` 清理）：

```typescript
watch(
  () => /* 封面 URL 响应式来源 */,
  (url) => {
    const el = document.querySelector('.content') as HTMLElement | null;
    if (!el) return;
    if (url?.trim()) {
      el.style.setProperty('--cover-bg-url', `url("${url.trim()}")`);
    } else {
      el.style.removeProperty('--cover-bg-url');
    }
  },
  { immediate: true },
);

onBeforeUnmount(() => {
  const el = document.querySelector('.content') as HTMLElement | null;
  el?.style.removeProperty('--cover-bg-url');
});
```

4. **注意 TDZ**：`fallbackCover` 等被 computed 引用的变量必须定义在 computed 之前（因为 `immediate: true` 的 watcher 会在 setup 阶段触发求值）
5. 页面销毁时必须在 `onBeforeUnmount` 中清理 `--cover-bg-url`，避免残留到下一个详情页

## 十四、height transition 恢复：60ms 防跳动 + 350ms 平滑

### 14.1 背景

`height` transition 曾因"小幅滑动时触发布局重排导致头部弹跳"被全面移除。当前通过三层保护重新引入：

| 保护层 | 作用 |
|--------|------|
| activatedAt 锁（300ms） | 防止 isSticky 振荡，消除 transition 反复触发的根因 |
| 默认 60ms 极短时长 | 仅 4 帧布局开销（每帧 ~1-3ms），消除跳动但几乎不增加渲染压力 |
| scroll-stop 升级（200ms） | 停止滚动后自动升级为 350ms，平滑完成最终过渡 |

### 14.2 当前 transition 清单

```css
/* 默认态（滚动中）：60ms height + 300ms 合成器属性 */
.playlist-detail-header-wrap {
  transition:
    height 0.06s cubic-bezier(0.33, 0, 0.1, 1),
    border-radius 0.3s cubic-bezier(0.33, 0, 0.1, 1),
    box-shadow 0.3s cubic-bezier(0.33, 0, 0.1, 1);
}

/* 滚动停止后：升级为 350ms height */
.playlist-detail-header-wrap.sticky-transition-ready {
  transition:
    height 0.35s cubic-bezier(0.33, 0, 0.1, 1),
    border-radius 0.3s cubic-bezier(0.33, 0, 0.1, 1),
    box-shadow 0.3s cubic-bezier(0.33, 0, 0.1, 1);
}
```

以下布局型属性仍保持无 transition（避免额外布局开销）：

- `padding`, `margin`（`.playlist-detail-header-wrap`）
- `font-size`, `line-height`, `margin`（`.hero-title-shell .title`）
- `grid-template-columns`, `gap`（`.page-hero-header`）
- `width`, `min-width`（`.page-hero-header__media`）

### 14.3 滚动中断保护

过渡进行中若用户再次滚动，`sticky-transition-ready` 被立即移除，transition 切回 60ms。正在进行的 350ms 过渡被 60ms 过渡覆盖，剩余差值在 60ms 内完成，保证新滚动事件不受旧过渡影响。

### 14.4 代码注释规范

修改 `DetailStickyHeroHeader.vue` 时，请遵循以下规范：

- 新增 transition 优先使用合成器友好属性（`opacity` / `transform` / `filter`）
- 布局型 transition（`height` / `padding` / `margin` / `width` / `font-size` 等）只在滚动停止后启用
