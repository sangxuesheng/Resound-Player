# 首页专属推荐卡片布局改造说明

## 改动概要

将首页「你的专属推荐」区域从 CSS Grid 布局改为 flex-wrap 布局，解决卡片尺寸不一致的问题。

## 涉及文件

- `src/components/HomePanel.vue` — 模板 + CSS 修改
- `src/styles/theme.css` — 引用 spacing token

## 修改前

### 模板结构

```html
<div class="exclusive-recommend-grid">
  <AnimatedAppear
    v-for="item in exclusiveRecoPlaylists.slice(0, 18)"
    ...
  >
    <GradientCard ... />
  </AnimatedAppear>
</div>
```

### CSS

```css
.exclusive-recommend-grid {
  display: grid;
  grid-template-columns: repeat(6, 1fr);
  gap: var(--space-3);
}
```

## 修改后

### CSS

```css
.exclusive-recommend-grid {
  display: flex;
  flex-wrap: wrap;
  gap: var(--space-3);
}
.exclusive-recommend-grid > * {
  width: calc((100% - 5 * var(--space-3)) / 6);
  min-width: 0;
}
```

## 原因分析

首页有以下两类推荐卡片区域：

| 区域 | 布局方式 | 列数 | 特点 |
|---|-----|-----|-----|
| 最新专辑推荐 | flex (`HorizontalScrollRail` + `albums-row`) | 自动（约 5 列） | 卡片等宽 |
| 你的专属推荐（改前） | CSS Grid `repeat(6, 1fr)` | 固定 6 列 | 卡片偏窄且高度不一致 |

**卡片尺寸不一致的原因：**

1. **列宽差异**：`1fr` 给出精确 1/6 宽度（~138px），而 albums 区 flex 布局每列约 **165px**，专属推荐卡片更窄
2. **info 区高度差异**：部分卡片有 subtitle（创建者按钮）、部分没有，较窄的卡片会放大这种视觉差异
3. **gap 差异**：grid 使用 12px（space-3），albums 区列间距使用 16px（space-4）

**flex-wrap 方案的优势：**

- 每项 `width: calc((100% - 5 * var(--space-3)) / 6)` 精确控制 1/6 宽度，与 albums 区行为一致
- `flex-wrap: wrap` 自动换行：≤6 一行，7-12 两行，13-18 三行（上限 18）
- gap 统一使用 `--space-3`（12px），与 albums 区内卡片间距一致

## 相关 Token

| Token | 值 | 用途 |
|---|---|---|
| `--space-3` | 12px | grid/flex gap |
| `--space-4` | 16px | albums 区横向列间距（供对比参考） |

## 重复模式

当需要「固定 N 列 + 自动换行」的卡片网格时，可复用此模式：

```css
.grid-container {
  display: flex;
  flex-wrap: wrap;
  gap: var(--space-3);
}
.grid-container > * {
  width: calc((100% - (N - 1) * var(--space-3)) / N);
  min-width: 0;
}
```

其中 `N` 为每行期望列数，gap 替换为对应 spacing token。

## 局限性

- 每行最后一行不足 N 项时，末尾会有空白（与 grid 行为一致）
- 不适用于需要严格跨行对齐的场景（应用使用 grid 而非 flex-wrap）
