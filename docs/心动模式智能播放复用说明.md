# 心动模式 / 智能播放复用说明

## 概述

心动模式是项目独有的一种智能播放模式，基于网易云音乐智能推荐 API，从用户歌单中选取歌曲作为种子，推荐类似风格的歌曲并自动播放。

区别于顺序/单曲/随机三种循环模式，心动模式由专属入口触发，不参与 `cyclePlayMode` 循环。

## 功能结构

```
用户登录
  └─ userStore 拉取 playlists（创建的、收藏的、喜欢的音乐）
      └─ 按钮根据 playlists 判断是否有歌单可用
          ├─ 有歌单 → 可点击 → 调 API 获取推荐 → 自动播放
          └─ 无歌单 → 可点击 → toast "您的歌单列表暂无心动推荐"
```

### 触发方式（TopBar Sparkles 按钮）

```
点击 Sparkles 按钮
  ├─ loading 中 → disabled（SVG 快速旋转）
  ├─ 未登录 → checkAuth() → 弹出登录弹窗
  ├─ UID 登录 → checkAuth() → toast "智能播放需要扫码或 Cookie 方式登录"
  └─ Cookie/QR 登录
       ├─ userStore.playlists 无曲目歌单 → toast "您的歌单列表暂无心动推荐"
       └─ 有曲目歌单 → 取第一个有曲目歌单的 id 作为 pid →
            ├─ 有当前歌曲 → 用 currentSongId 作为种子
            └─ 无当前歌曲 → 调用 getPlaylistTrackAll 取歌单第一首作为种子
            → 调 API → 替换播放列表 → 从第一首开始播放
```

## 数据流

```
TopBar.vue
  └─ handleIntelligencePlay()
      └─ checkAuth() → useAuthAction
      └─ 找 userStore.playlists 中有曲目的歌单
      └─ playerStore.currentPlaylistId = pid
      └─ playerStore.playIntelligenceList()
          └─ (无种子时) getPlaylistTrackAll({ id: pid, limit: 1 })
          └─ getIntelligenceList({ id: seedSongId, pid, cookie })
          └─ 映射 songInfo → Track[]
          └─ setPlaylist(tracks, 0, pid)
          └─ playByIndex(0)
```

## API 接口

| 函数 | 端点 | 说明 |
|------|------|------|
| `getIntelligenceList({ id, pid, sid?, cookie? })` | `GET /playmode/intelligence/list` | 基于种子歌曲从指定歌单推荐歌曲 |
| `getPlaylistTrackAll({ id, limit, cookie? })` | `GET /playlist/track/all` | 获取歌单曲目列表（用于无播放时取种子） |

### 请求参数

| 参数 | 必选 | 说明 |
|------|------|------|
| `id` | 是 | 种子歌曲 ID |
| `pid` | 是 | 歌单 ID（必须是当前用户创建/收藏的标准歌单） |
| `sid` | 否 | 要开始播放的歌曲 ID |

### 响应格式

```json
{
  "code": 200,
  "data": [
    {
      "id": 27836251,
      "alg": "Alg_AI_One_33894312:27836251_sameTag-RH",
      "recommended": false,
      "songInfo": {
        "name": "歌曲名称",
        "ar": [{ "id": 30149841, "name": "毛新琳" }],
        "al": { "picUrl": "...", "name": "评弹", "id": 2685038 },
        "id": 27836251,
        "dt": 395000
      }
    }
  ]
}
```

**注意：** `name`、`ar`、`al` 不在顶层，嵌套在 `songInfo` 内部。映射到 `Track` 类型时需要提取：

```ts
const info = item.songInfo || {};
formatTrack({
  id: info.id || item.id,
  name: info.name,
  ar: info.ar,
  al: info.al,
});
```

### 错误码

| code | 说明 |
|------|------|
| 200 | 成功 |
| 301 | 需要登录 |
| 400 | 不支持该歌单类型（歌单不属于当前用户或非标准类型） |

## 核心 Store 状态

### `playerStore`（`src/stores/player.ts`）

| 属性 | 类型 | 说明 |
|------|------|------|
| `currentPlaylistId` | `number` | 当前播放上下文歌单 ID（持久化） |
| `currentIntelligenceLoading` | `boolean` | 心动模式 API 请求中 |
| `isIntelligenceActive` | `boolean` | 是否处于心动模式（持久化） |

### 关键方法

```ts
// 启动心动模式：获取推荐 → 替换播放列表 → 自动播放
playIntelligenceList(): Promise<string | null>

// 设置播放列表（第三个参数可选，用于记录歌单上下文）
setPlaylist(list, startIndex, playlistId?)
```

### `uiStore`（`src/stores/ui.ts`）

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `showIntelligenceIndicator` | `boolean` | `true` | 是否在控制中心显示心动模式图标 |

## 可视化接入点

心动模式图标在所有 3 个播放器控制组件中展示：

| 组件 | 位置 | CSS 类 | 条件 |
|------|------|--------|------|
| `TopBar.vue` | 搜索按钮右侧 | `.msg--active` | 激活时粉色边框+背景+慢速旋转 |
| `PlayerBar.vue` | 音量滑块与音质按钮之间 | `.icon.intel-icon` | `isIntelligenceActive && showIntelligenceIndicator` |
| `PlayerExpanded.vue` | 迷你控制中心：音量与收藏之间 | `.ctrl.intel-icon` | 同上 |
| `PlayerExpanded.vue` | 底部控制中心：播放列表按钮前 | `.con-btn.intel-icon` | 同上 |

### 视觉规范

- **默认态：** 无图标显示
- **Loading 态：** SVG 快速旋转（1s/圈），`msg--loading`
- **激活态（心动模式中）：** SVG 慢速旋转（5s/圈），粉色 `#ec4899`
  - TopBar 按钮：粉色边框 + 粉色背景 + 粉色光晕
  - 控制中心按钮：粉色 SVG，仅 hover 显示背景

### 控制中心显隐设置

设置页 → 外观 → "控制中心心动图标" 开关（`uiStore.showIntelligenceIndicator`），关闭后 PlayerBar 和 PlayerExpanded 的心动图标隐藏，TopBar 入口按钮不受影响。

## 登录鉴权

复用 `useAuthAction` composable：

```ts
const { checkAuth } = useAuthAction(
  '智能播放需要扫码或 Cookie 方式登录',  // UID 登录时的 toast 文案
  'playlist',                           // 未登录时的弹窗 intent
);
```

鉴权链路与 `BookmarkIconButton`、`SongActions` 等组件一致。

## 歌单 ID 传递方式

`PlaylistDetailPage.vue` 在调用 `setPlaylist` 时传入歌单 ID：

```ts
playerStore.setPlaylist(tracks.value, 0, props.playlistId);
```

其他页面（HomePanel、SearchPage、AlbumDetailPage 等）不传歌单 ID 时，`currentPlaylistId` 保持上次值或默认 0。点击心动按钮时，从 `userStore.playlists` 中查找第一个有曲目的歌单作为 pid。

### 歌单校验

点击时校验持久化的 `currentPlaylistId` 是否属于当前用户：

```ts
if (playerStore.currentPlaylistId) {
  const belongsToUser = userStore.playlists.some(p => p.id === playerStore.currentPlaylistId);
  if (!belongsToUser) playerStore.currentPlaylistId = 0;
}
```

## 退出机制

| 触发方式 | 行为 |
|---------|------|
| 点击激活态按钮 | `clearPlaylist()` → 停止播放、清空列表 → 退出 |
| 切换其他歌单播放 | watch `currentPlaylistId` 变化 → 自动退出 + toast |
| 退出登录 | `userStore.logout()` → 调用 `playerStore.clearPlaylistContext()` |

## 设置开关

**路径：** 设置页 → 外观 → "控制中心心动图标"

- 默认开启
- 持久化到 localStorage key `tm_show_intel_indicator`
- 关闭后 PlayerBar / PlayerExpanded 两处图标隐藏，TopBar 入口按钮不受影响

## 种子歌曲选取策略

| 场景 | 种子来源 |
|------|---------|
| 有当前播放歌曲 | `playerStore.currentSongId` |
| 无当前播放歌曲 | `getPlaylistTrackAll({ id: pid, limit: 1 })` 取歌单第一首 |

## 维护建议

- 新增调用 `setPlaylist` 的页面时，如果该页面有明确的歌单上下文，应传入 `playlistId`（第三个参数）
- 新增播放器控制组件时，使用 `playerStore.isIntelligenceActive` + `uiStore.showIntelligenceIndicator` 条件渲染心动图标
- 心动模式指示器使用 `.intel-icon` CSS 类，需同时添加 SVG 旋转动画和粉色主题色
