# 实体收藏/关注组件复用说明

## 概述

项目中所有对「实体」的收藏/订阅/关注操作（区别于对单曲的收藏），统一使用同一套架构。包括：

| 功能 | 页面 | 按钮文字（未操作） | 按钮文字（已操作） |
|---|---|---|---|
| 收藏歌单 | PlaylistDetailPage | 收藏歌单 | 取消收藏歌单 |
| 收藏专辑 | AlbumDetailPage | 收藏专辑 | 取消收藏 |
| 订阅播客 | PodcastDetailPage | 订阅播客 | 取消订阅 |
| 关注歌手 | ArtistDetailPage | 关注 | 已关注 |
| 关注用户 | UserDetailPage | 关注 | 已关注 / 互相关注 |

两套系统并存：
- **二态系统**（`useEntitySubscribe` + `EntitySubscribeButton`）：歌单/专辑/播客/歌手，收藏/未收藏两态
- **三态系统**（`useUserFollow` + `UserFollowButton`）：用户关注，未关注/已关注/互相关注三态

## 实现结构

```
src/api/music.ts                           ← API 层
src/stores/user.ts                         ← Store 层（登录时拉取订阅列表）
src/composables/useEntitySubscribe.ts      ← 二态 composable
src/composables/useUserFollow.ts           ← 三态 composable
src/components/ui/EntitySubscribeButton.vue ← 二态按钮组件
src/components/ui/UserFollowButton.vue     ← 三态按钮组件
```

## 二态系统：收藏/订阅/关注歌手

### API 层

| 函数 | 接口 | 说明 |
|---|---|---|
| `togglePlaylistSubscribe({ id, subscribe, cookie })` | `/playlist/subscribe?t=1/2&id=xxx` | t=1 收藏, t=2 取消 |
| `toggleAlbumSubscribe({ id, subscribe, cookie })` | `/album/sub?t=1/0&id=xxx` | t=1 收藏, t=0 取消 |
| `toggleArtistSubscribe({ id, subscribe, cookie })` | `/artist/sub?t=1/0&id=xxx` | t=1 收藏, t=0 取消 |
| `toggleDjSubscribe({ rid, subscribe, cookie })` | `/dj/sub?t=1/0&rid=xxx` | t=1 订阅, t=0 取消 |

### Store 层

登录时自动拉取订阅列表，存入 `userStore`：

| 数组 | 填充方式 |
|---|---|
| `subscribedPlaylistIds` | `fetchPlaylists()` 内联从 `/user/playlist` 提取 `p?.subscribed === true` 的歌单 ID |
| `subscribedAlbumIds` | `fetchSubscribedAlbums()` 调用 `/album/sublist` |
| `subscribedArtistIds` | `fetchSubscribedArtists()` 调用 `/artist/sublist` |
| `subscribedDjIds` | `fetchSubscribedDjs()` 调用 `/dj/sublist` |

所有 fetch 方法在 `refreshLoginStatus` / `loginWithUid` / `restoreUidLogin` 中通过 `Promise.allSettled` 并行调用。

### Composable 接口

```ts
type EntityType = 'playlist' | 'album' | 'artist' | 'podcast';

interface UseEntitySubscribeOptions {
  type: EntityType;
  id: Ref<number | undefined> | ComputedRef<number | undefined>;
  initialSubscribed?: boolean | Ref<boolean>;  // 可选，页面数据已有时传进来避免闪烁
}

function useEntitySubscribe(options: UseEntitySubscribeOptions): {
  isSubscribed: Ref<boolean>;   // 是否已收藏/订阅/关注
  isLoading: Ref<boolean>;      // 操作中
  toggle: () => Promise<void>;  // 切换收藏状态（内置鉴权）
}
```

**状态优先级**：store 数组（登录时拉取）> `initialSubscribed`（页面数据）> `false`

### 组件 Props

```vue
<EntitySubscribeButton
  :type="'playlist'"
  :subscribed="isSubscribed"
  :loading="isLoading"
  :text="true"           // 可选，文字模式（默认心形图标）
  @toggle="toggle"
/>
```

| Prop | 类型 | 默认 | 说明 |
|---|---|---|---|
| `type` | `EntityType` | — | `playlist` / `album` / `artist` / `podcast` |
| `subscribed` | `boolean` | — | 当前收藏状态 |
| `loading` | `boolean` | — | 操作中 |
| `text` | `boolean` | `false` | 文字模式。`true` 时显示纯文字按钮，`false` 时显示心形图标 |

**渲染模式**：

| type | text=false（默认） | text=true |
|---|---|---|
| `playlist` | 心形图标 | 收藏歌单 / 取消收藏歌单 |
| `album` | 心形图标 | 收藏专辑 / 取消收藏 |
| `podcast` | 心形图标 | 订阅播客 / 取消订阅 |
| `artist` | 关注 / 已关注（文字+图标，不受 `text` 影响） | 同上 |

### 页面接入示例

```vue
<script setup lang="ts">
import EntitySubscribeButton from './ui/EntitySubscribeButton.vue';
import { useEntitySubscribe } from '../composables/useEntitySubscribe';

const albumIdRef = computed(() => album.value?.id || undefined);
const subscribeState = useEntitySubscribe({
  type: 'album',
  id: albumIdRef,
  initialSubscribed: computed(() => album.value?.subscribed ?? false),
});
</script>

<template>
  <DetailStickyHeroHeader>
    <template #actions>
      <EntitySubscribeButton
        v-if="album?.id"
        type="album"
        text
        :subscribed="subscribeState.isSubscribed.value"
        :loading="subscribeState.isLoading.value"
        @toggle="subscribeState.toggle"
      />
      <button class="play-all" @click="playAll">播放全部</button>
    </template>
  </DetailStickyHeroHeader>
</template>
```

## 三态系统：关注用户

### API 层

| 函数 | 接口 | 说明 |
|---|---|---|
| `toggleUserFollow({ id, follow, cookie })` | `/follow?t=1/0&id=xxx` | t=1 关注, t=0 取消 |
| `getUserMutualFollow(uid, cookie)` | `/user/mutualfollow/get?uid=xxx` | 查询是否互关 |
| `getUserFollows(uid, opts)` | `/user/follows?uid=xxx` | 批量获取关注列表（登录时填充 store） |

### Store 层

`subscribedUserIds: number[]` — 登录时通过 `fetchSubscribedUsers()` 调用 `/user/follows` 填充。

### Composable 接口

```ts
type FollowStatus = 'none' | 'following' | 'mutual';

function useUserFollow(options: {
  id: Ref<number | undefined> | ComputedRef<number | undefined>;
}): {
  status: Ref<FollowStatus>;    // 三种状态
  isLoading: Ref<boolean>;      // 操作中
  toggle: () => Promise<void>;  // 切换关注状态（内置鉴权）
}
```

**三种状态**：
- `none` — 未关注
- `following` — 已关注（对方未回关）
- `mutual` — 互相关注

**syncToken 防竞态**：composable 内部维护 `syncToken`，避免 store 数组变化触发的 watcher 回调覆盖 `toggle` 刚设置的正确状态。

### 组件 Props

```vue
<UserFollowButton
  :status="status"
  :loading="isLoading"
  @toggle="toggle"
/>
```

| Prop | 类型 | 默认 | 说明 |
|---|---|---|---|
| `status` | `FollowStatus` | — | `'none'` / `'following'` / `'mutual'` |
| `loading` | `boolean` | — | 操作中 |

**三种渲染态**：

| status | 图标 | 文字 | 颜色 |
|---|---|---|---|
| `none` | + | 关注 | 默认 |
| `following` | ✓ | 已关注 | 绿色强调 |
| `mutual` | ⇄ | 互相关注 | 绿色强调 |

### 页面接入示例

```vue
<script setup lang="ts">
import UserFollowButton from './ui/UserFollowButton.vue';
import { useUserFollow } from '../composables/useUserFollow';
import { userStore } from '../stores/user';

const userIdRef = computed(() => props.userId || undefined);
const followState = useUserFollow({ id: userIdRef });
</script>

<template>
  <DetailStickyHeroHeader>
    <template #actions>
      <UserFollowButton
        v-if="userId && userId !== userStore.profile?.userId"
        :status="followState.status.value"
        :loading="followState.isLoading.value"
        @toggle="followState.toggle"
      />
      <button class="play-all" @click="openFirstPlaylist">打开首个歌单</button>
    </template>
  </DetailStickyHeroHeader>
</template>
```

## 鉴权机制

两个 composable 的 `toggle()` 函数均内置鉴权：

```
用户点击按钮 → toggle() → 鉴权检查
  ├─ 未登录 → showLoginModal('subscribe') → 弹出登录弹窗
  ├─ uid 登录（搜索用户，非 cookie/qr）→ showGlobalToast(…) → 弹出黄色 toast 提示
  └─ cookie/qr 登录 → 正常执行 API 调用
```

无需在页面中额外处理鉴权。

## 状态持久化（刷新不丢失）

刷新页面后订阅状态的恢复链路：

| 类型 | 登录时填充 | 页面加载后 |
|---|---|---|
| 歌单 | `fetchPlaylists()` 提取 `subscribedPlaylistIds` | 复用 store 数组 |
| 专辑 | `fetchSubscribedAlbums()` 提取 `subscribedAlbumIds` | 复用 store 数组 |
| 歌手 | `fetchSubscribedArtists()` 提取 `subscribedArtistIds` | 复用 store 数组 |
| 播客 | `fetchSubscribedDjs()` 提取 `subscribedDjIds` | 复用 store 数组 |
| 用户 | `fetchSubscribedUsers()` 提取 `subscribedUserIds` | 复用 store 数组，再调 `/user/mutualfollow/get` 判断互关 |
