# 播客简介时间标签跳转实现说明

## 功能概述

播客播放时，歌词区域显示播客单集简介（`.podcast-desc`）。简介文本中通常包含时间标签（如 `30:00`、`1:30:00`），点击这些时间标签后播放器跳转到对应时间位置。如果播放器处于暂停状态，跳转后自动恢复播放。

## 实现方案

全部实现在 `src/components/LyricsPanel.vue` 中，分三层：模板解析、逻辑处理、样式表现。

### 模板层

将原来的纯文本插值改为 `v-html` 渲染 + 点击事件代理：

```html
<div v-else-if="!lyricLines.length && podcastDescription"
     class="podcast-desc"
     v-html="podcastDescriptionHtml"
     @click="onPodcastDescClick">
</div>
```

- `v-html` 用于渲染带可点击 span 的 HTML（内容来自 API，视为可信）
- `@click` 通过事件代理捕获内部 span 点击

### 逻辑层

#### 时间标签解析

```ts
const timestampRe = /(\b\d{1,2}:\d{2})(?::(\d{2}))?\b/g;
```

正则匹配两种格式：
- `MM:SS` — 如 `30:00`、`05:30`
- `H:MM:SS` — 如 `1:30:00`、`12:05:30`

#### 文本转秒数

```ts
function parseTimestampToSeconds(text: string): number {
  const parts = text.split(':').map(Number);
  if (parts.length === 2) return parts[0] * 60 + parts[1];
  if (parts.length === 3) return parts[0] * 3600 + parts[1] * 60 + parts[2];
  return 0;
}
```

#### HTML 生成

```ts
const podcastDescriptionHtml = computed(() => {
  const desc = podcastDescription.value;
  if (!desc) return '';
  const escaped = desc
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;');
  return escaped.replace(timestampRe, (match) => {
    const sec = parseTimestampToSeconds(match);
    return `<span class="podcast-ts" data-sec="${sec}">${match}</span>`;
  });
});
```

1. 先做 HTML 转义（防止简介文本中的特殊字符破坏页面）
2. 对转义后的文本用正则找到所有时间标签
3. 每个时间标签替换为一个 `<span class="podcast-ts" data-sec="秒数">` 元素
4. 计算属性：播客切换时自动重新计算

#### 点击处理

```ts
function onPodcastDescClick(e: MouseEvent) {
  const target = (e.target as HTMLElement)?.closest?.('.podcast-ts');
  if (!target) return;
  const sec = Number(target.getAttribute('data-sec'));
  if (isNaN(sec) || sec < 0) return;
  playerStore.seek(sec);
  // 如果处于暂停状态则自动恢复播放
  if (!playerStore.isPlaying) {
    nextTick(() => playerStore.togglePlay());
  }
}
```

- 通过 `closest('.podcast-ts')` 向上查找被点击的 span，不会受内部文本节点干扰
- 读取 `data-sec` 属性获取秒数
- 调用 `playerStore.seek(sec)` 跳转
- 暂停态下点击后调用 `togglePlay()` 自动恢复

### 样式层

```css
.podcast-desc :deep(.podcast-ts),
:deep(.podcast-ts) {
  cursor: pointer;
  color: var(--accent, #c39c76);
  font-weight: 600;
  border-bottom: 1px dashed color-mix(in srgb, var(--accent, #c39c76) 40%, transparent);
  transition: all 120ms ease;
}
.podcast-desc :deep(.podcast-ts:hover),
:deep(.podcast-ts:hover) {
  background: color-mix(in srgb, var(--accent, #c39c76) 12%, transparent);
  border-bottom-color: var(--accent, #c39c76);
}
.podcast-desc :deep(.podcast-ts:active),
:deep(.podcast-ts:active) {
  background: color-mix(in srgb, var(--accent, #c39c76) 22%, transparent);
}
```

由于 `v-html` 渲染的内容不受 scoped style 直接作用，使用 `:deep()` 穿透选择器定位 `.podcast-ts`。

视觉风格：
- accent 色文字 + 圆点虚线下划线（表示可点击）
- hover 时浅色背景填充 + 下划线变为实色
- active 时背景加深

## 影响范围

- **仅限播客**：通过 `isCurrentPodcast` computed 守卫，仅 `source === 'podcast'` 的曲目生效
- **无歌词时显示**：时间标签只在「无歌词 + 有简介」状态下渲染（与现有 `.podcast-desc` 显示逻辑一致）
- **歌词状态不受影响**：有歌词的播客节目走正常歌词渲染流程，简介不显示

## 验证清单

1. 播放播客（无歌词）→ 简介区域显示文字，时间标签以 accent 色高亮
2. 鼠标悬浮时间标签 → 显示下划线和浅色背景
3. 点击时间标签 → 播放器跳转到对应时间
4. 暂停状态下点击时间标签 → 播放器跳转并自动恢复播放
5. 切换到普通歌曲 → 播客简介不显示，时间标签功能不干扰
6. 简介中的非时间文本不受影响（不会被误识别为时间标签）
