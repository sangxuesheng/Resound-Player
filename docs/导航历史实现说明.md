# 导航历史（前进/后退）实现说明

## 概述

导航历史系统支持用户在 TopBar 上使用前进/后退按钮，按页面访问顺序导航。系统独立于页面内部的 `backTo*` 返回按钮，两者并行工作，但通过 `navHistory.replace()` 保持同步。

## 核心 API

`src/composables/useNavigationHistory.ts` 提供以下接口：

| 方法 | 行为 |
|---|---|
| `push(entry)` | 将当前页面压入 backStack，设置新页面为 current，清空 forwardStack |
| `back()` | 将 current 压入 forwardStack，弹出 backStack 顶端条目作为新 current |
| `forward()` | 将 current 压入 backStack，弹出 forwardStack 顶端条目作为新 current |
| `replace(entry)` | 替换 current 为新的 entry，不操作 backStack/forwardStack |
| `reset(entry)` | 清空 backStack/forwardStack，设置 current |

**注意：** `push` 仅在 `current` 存在时才会把旧页面压入 backStack。因此首次导航前必须初始化 `current`（见下文）。

## 集成方式

### 1. 初始化

在 `App.vue` 的 `onMounted` 末尾调用：

```ts
navHistory.replace({ page: 'home' });
```

这设置了初始 `current`，使得第一次 `push()` 时「首页」能被压入 backStack。

### 2. open* 函数中记录导航

所有 `open*` 函数在修改页面状态之前调用 `navHistory.push()`，并携带**恢复页面所需的关键参数**：

```ts
function openPlaylistDetail(playlistId: number, ...) {
  navHistory.push({
    page: 'playlist-detail',
    params: {
      playlistId,
      returnPage: returnPage || 'home',   // 恢复页面内返回按钮标签
    },
  });
  // ... 修改状态
  activePage.value = 'playlist-detail';
}
```

### 3. 恢复状态 — navigateToEntry

当用户点击 TopBar 后退/前进按钮时，调用 `navigateToEntry(entry)` 恢复页面状态：

```ts
function navigateToEntry(entry: { page: string; params?: Record<string, any> }) {
  const { page, params } = entry;
  activePage.value = page;

  // 恢复关键 ID（触发 watcher 加载数据）
  if (params?.playlistId) activePlaylistId.value = params.playlistId;
  if (params?.albumId) activeAlbumId.value = params.albumId;
  // ... 其他 ID

  // 恢复扩展状态
  if (params?.dailyInjectedPlaylist) dailyInjectedPlaylist.value = params.dailyInjectedPlaylist;
  if (params?.keyword) uiStore.searchKeyword = params.keyword;

  // 按页面类型恢复 returnPage
  if (params?.returnPage) {
    if (page === 'playlist-detail') activePlaylistReturnPage.value = params.returnPage;
    else if (page === 'album-detail') activeAlbumReturnPage.value = params.returnPage;
    // ...
  }
  scrollContentToTop();
}
```

### 4. backTo* / goBackFrom* 函数中使用 replace

页面内「返回」按钮通过 `backTo*` 或 `goBackFrom*` 函数实现。这些函数使用 `navHistory.replace()` 同步 navHistory 的 `current`，防止与 TopBar 前进/后退产生混乱：

```ts
function backToPlaylist() {
  dailyInjectedPlaylist.value = null;
  const target = activePlaylistReturnPage.value || 'playlist';
  navHistory.replace({ page: target });  // 仅替换 current，不操作栈
  activePage.value = target;
}
```

**为什么用 replace 而不是 push？** 使用 `push` 会向 backStack 额外压入当前页面，导致用户按一次后退回到同一页面，需要再按一次才能回到上一个页面。`replace` 仅修改 current，后续后退行为不受影响。

### 5. onSelectMenu 同页守卫

侧栏点击事件需防止重复压入：

```ts
function onSelectMenu(key: string) {
  if (activePage.value === key) return;  // 已在目标页，跳过
  navHistory.push({ page: key });
  activePage.value = key;
  // ...
}
```

## 参数传递规范

每个 `open*` 函数的 `navHistory.push()` 应携带恢复该页面所需的参数。当前参数清单：

| 页面 | 参数 | 说明 |
|---|---|---|
| `playlist-detail` | `playlistId`, `returnPage`, `dailyInjectedPlaylist` | `dailyInjectedPlaylist` 用于每日推荐自定义歌单 |
| `album-detail` | `albumId`, `returnPage` | |
| `artist-detail` | `artistId`, `returnPage` | |
| `rank-detail` | `rankId` | |
| `user-detail` | `userId`, `returnPage` | |
| `search` | `keyword` | |
| `song-comment` | `songId`, `returnPage` | |
| `settings` | `settingsTab` | |
| `language-detail` | `language`, `returnPage` | |
| `mv-play` | `mvItem`, `mvReturnPage` | `mvItem` 为 MV 数据对象 |
| `podcast-detail` | 无（复杂异步，不做参数恢复） | |
| `playlist` | `category` | |
| `podcast-category` | `podcastCategoryId` | |
| 其他简单页面 | 无 | |

## 新增页面接入步骤

1. 在 `open*` 函数中调用 `navHistory.push({ page, params })`，params 包含恢复页面所需的数据
2. 在 `navigateToEntry` 中新增对应参数的恢复逻辑
3. 在对应的 `backTo*` 或 `goBackFrom*` 函数中添加 `navHistory.replace({ page })`
4. 如果页面是侧栏入口，在 `onSelectMenu` 中确认同页守卫生效（无需额外操作）

## 生命周期

- 历史存储在内存中（`ref<HistoryEntry[]>`），页面刷新后丢失
- `navHistory.replace({ page: 'home' })` 在 `onMounted` 中执行，确保刷新后首次导航可后退到首页
- 最大历史深度无硬限制，由用户浏览深度自然决定（通常不超过数十条）
