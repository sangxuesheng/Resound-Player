# 首页组件开发记录

> 记录了最新专辑推荐、推荐播客、推荐 MV 三个首页组件的开发变更，以及网格布局系统的移除。

---

## 1. 布局系统变更：网格 → 流式

### 变更内容

首页布局从 CSS Grid 固定行高系统改为 Flex 流式纵向排列。

| 项 | 改前 | 改后 |
|---|---|---|
| `home-grid` | `display: grid` + `grid-auto-rows: 72px` | `display: flex; flex-direction: column; gap: 12px` |
| `.card` | `overflow: hidden` 裁切溢出内容 | 无 overflow 限制，高度由内容决定 |
| `.card.list` | `overflow-y: auto` 内部滚动 | 移除，列表自然撑开 |
| 组件定位 | `grid-row: span N` 固定 N 行 | 纵向依次排列，高度自适应 |
| 相关常量 | `GRID_COLS`、`GRID_ROW_HEIGHT`、`GRID_GAP` | 全部移除 |
| 相关函数 | `gridStyle` computed、`cellStyle()` | 全部移除 |

### 删除的代码

- 模板：`:style="gridStyle"`、`:style="cellStyle(widget)"`
- 脚本：`GRID_COLS`, `GRID_ROW_HEIGHT`, `GRID_GAP` 常量、`gridStyle` 计算属性、`cellStyle` 函数
- CSS：`.home-grid` 的 grid 相关属性、`.card` 的 `overflow: hidden`、`.card.list` 的滚动样式

## 2. 最新专辑推荐（albums）

### 布局变更

将专辑区从**竖向固定网格**改为**横向滚动轨道**，并移到分类组件下方全宽显示。

```
改前：右侧竖向排列（4列网格）
改后：分类下方全宽，HorizontalScrollRail 横向滚动
```

### 模板结构

```html
<HorizontalScrollRail content-layout="flex" content-class="albums-row">
  <div v-for="(group, gi) in albumGroups" :key="gi" class="albums-column">
    <GradientCard v-for="item in group" tag="button" ... />
  </div>
</HorizontalScrollRail>
```

### 关键实现

- **组件**: `HorizontalScrollRail` + `GradientCard`
- **数据**: `getTopAlbums()` 或 `getNewestAlbums()` 兜底，取 12 张
- **布局**: 每列 2 张卡片（column pairs），`content-layout="flex"`
- **卡片宽度**: `:deep(.albums-row) > * { width: 220px }`
- **无限滚动**: 已移除（原 IntersectionObserver 分页逻辑不再需要）

### 删除的代码

| 项 | 位置 | 原因 |
|---|---|---|
| `visibleAlbumCount` / `albumPageSize` / `albumSentinel` / `albumCardRef` / `albumObserver` | script 变量 | 网格分页不再需要 |
| `visibleAlbums` computed | script | 全部展示 |
| `setAlbumCardRef` / `setAlbumSentinelRef` | script 函数 | 无需 ref |
| `loadMoreAlbums` / `ensureAlbumScrollable` / `setupAlbumObserver` | script 函数 | 滚动加载已移除 |
| `albumObserver.disconnect()` | `onBeforeUnmount` | 观察者已移除 |
| `.card.albums` overflow 样式 / `.album-grid` CSS | scoped CSS | 不再使用 |

## 3. 推荐播客（podcast）

### 新增组件

在最新音乐组件下方、推荐 MV 上方新增播客推荐区域。

```
最新音乐（latest-music）
推荐播客（podcast）      ← 新增
推荐 MV（mv）
```

### 模板结构

```html
<div class="podcast-grid an-stagger-media">
  <div v-for="(item, idx) in podcastList" :key="item.id" class="mv-card"
       :style="{ '--i': idx }" @click="openPodcast(item)">
    <GradientCard tag="button" :cover="item.picUrl" :name="item.name" hover-play-size="sm">
      <template #subtitle>
        <span v-if="item.creatorName">{{ item.creatorName }}</span>
      </template>
    </GradientCard>
  </div>
</div>
```

### 关键实现

| 项 | 说明 |
|---|---|
| **组件** | `GradientCard`（带 hover 播放按钮的封面卡片） |
| **数据** | `getDjRecommend()` → `data.djRadios`，取前 9 个 |
| **布局** | 5 列网格，`grid-template-columns: repeat(5, 1fr)` |
| **高度** | 自适应（流式布局下由内容决定） |
| **交互** | 点击卡片 emit `open-podcast-detail` |
| **入场动画** | `.an-stagger-media > *` + `--i` 变量控制逐卡片弹入 |

### 新增代码

- `import { getDjRecommend }` — API 导入
- `podcastList` / `podcastLoading` / `podcastError` — 响应式状态
- `fetchPodcastList()` — 数据获取函数，字段归一化
- `openPodcast()` — 点击跳转函数
- `emit('open-podcast-detail', item.id)` — 事件
- `{.podcast-grid}` — 5 列网格 CSS
- 布局配置：`podcast: y=21, w=12, h=7`（h 值仅在 editor 中生效）

## 4. 推荐 MV（mv）

### 新增组件

在推荐播客下方新增 MV 推荐区域。

### 模板结构

```html
<div class="mv-grid an-stagger-media">
  <div v-for="(item, idx) in mvList" :key="item.id" class="mv-card"
       :style="{ '--i': idx }" @click="openMv(item)">
    <MvHoverPoster :src="item.cover" :alt="item.name"
                   :count="item.playCount" :index="idx" />
    <div class="mv-card__info">
      <div class="mv-card__name">{{ item.name }}</div>
      <span v-if="item.artistName" class="mv-card__artist">{{ item.artistName }}</span>
    </div>
  </div>
</div>
```

### 关键实现

| 项 | 说明 |
|---|---|
| **组件** | `MvHoverPoster`（16:9 海报 + hover 播放按钮 + 播放量） |
| **数据** | `getAllMvs({ limit: 12 })` → `data.data`，取 12 个 |
| **布局** | 4 列网格，`grid-template-columns: repeat(4, 1fr)`，3 行 |
| **高度** | 自适应（流式布局下由内容决定） |
| **交互** | 点击卡片 emit `open-mv-player` |
| **入场动画** | `.an-stagger-media > *` + `--i` 变量 |

### 新增代码

- `import { getAllMvs }` — API 导入
- `import MvHoverPoster` — 组件导入
- `mvList` / `mvLoading` / `mvError` — 响应式状态
- `fetchMvList()` — 数据获取函数，字段归一化
- `openMv()` — 点击跳转函数
- `emit('open-mv-player', mv)` — 事件
- `{.mv-grid}` — 4 列网格 CSS

## 6. 布局顺序总览

当前首页从上到下依次为：

```
分类（tags）
热门歌手（top-artists — 位于 section）
最新专辑推荐（albums）
本周最热音乐（list）
新歌速递（latest-music）
推荐播客（podcast）      ← 新增
推荐 MV（mv）            ← 新增
```

## 7. 注意事项

1. **Album API 兜底逻辑**：`fetchTopAlbums` 依次尝试 `/top/album` → `/album/newest` → 搜索兜底，确保数据不为空
2. **MV/Podcast 高度自适应**：移除网格系统后，高度完全由内容决定，不需要手动调整 `h` 值
3. **localStorage 布局缓存**：`tm_home_widget_layout_v1` 键值中存储了编辑器内的布局，代码改动后可通过 `hydrateHomeLayout` 中的迁移逻辑自动同步 mv/podcast 的 `h` 值
4. **min-width: 0**：在 `.mv-card` 和 `.podcast-grid > *` 上设置了 `min-width: 0`，防止网格子项受内容宽度影响导致列宽不一致
