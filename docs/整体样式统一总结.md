## 3. 统一的设计原则

### 3.1 spacing 优先用 token

本轮把大量硬编码的 `gap`、`padding`、`margin` 收敛为：

- `--space-1`
- `--space-2`
- `--space-3`
- `--space-4`
- `--space-5`
- `--space-6`
- `--space-8`

后续调整密度时，优先改 token，而不是在组件里继续加局部数值。

### 3.2 结构统一优先于局部美化

本轮优先统一了：

- 页面布局层级
- 卡片间距
- 列表项节奏
- 面板内边距
- 控件大小

这样能保证整个项目先达成一致的产品感，再做局部装饰。

### 3.3 保持兼容

样式统一时尽量保留：

- 组件结构
- 事件逻辑
- 数据流
- 播放行为

这轮主要是收敛样式，不是重写页面。

## 6. 全局封面 hover 缩放统一 (2026-05)

所有封面交互统一使用 `--image-hover-scale: 1.05`（`src/styles/animations.css`）。

### 核心规则

| 规则 | 说明 |
|------|------|
| 统一变量 | `--image-hover-scale: 1.05`，组件不得自行覆盖 |
| animation-fill-mode | 所有封面元素必须在 animations.css 中声明 `animation-fill-mode: backwards`，防止 AnimatedAppear 锁定 transform |
| overflow: hidden | 封面父容器必须有，否则缩放溢出 |
| 行级 hover | 歌单列表用 `.song-item:hover img.song-cover`，首页用 `.song:hover .cover` |
| 大封面 | 详情页 hero 使用 `--scroll-transform` CSS 变量驱动滚动动画 |

### 涉及文件

- `src/styles/animations.css` — 全局变量、hover 选择器、fill-mode 列表
- `src/styles/detail-page.css` — `.song-item { overflow: hidden }`，大封面 `.cover` 基础样式
- `src/components/DetailStickyHeroHeader.vue` — 滚动 JS 改用 `--scroll-transform` CSS 变量
- `src/components/HomePanel.vue` — 移除本地 `scale(1.06)` override
- `src/components/PlaylistDetailPage.vue` — 移除 `transform-origin: left center` override

### 禁止事项

- ❌ 禁止在组件 scoped CSS 中覆盖封面的 `transform` 或 `transform-origin`
- ❌ 禁止使用 `[class*='cover']` 通配符选择器
- ❌ 禁止 JS inline style 写 `transform`（应用 CSS 变量替代）
- ❌ 禁止单独为某个组件设置封面 hover 缩放值

## 4. `theme.css` 的定位

`theme.css` 现在主要承担三类职责：

- 设计 token
- 交互与控件状态
- 兼容兜底

新增样式时优先使用 token，不要继续在组件里堆硬编码值。

---

## 5. `detail-page.css` 的公共详情样式说明

`detail-page.css` 负责把“详情页”统一成同一套视觉框架。

它主要统一了：

- 详情页面板外壳
- 头图区域
- 返回按钮
- 封面样式
- 标题 / 副标题 / 描述
- 播放与操作按钮
- 歌曲列表行
- 状态提示区

### 维护顺序

以后要改详情页体验，建议按这个顺序：

1. 先改 `detail-page.css`
2. 再看具体页面是否还需要局部覆盖
3. 尽量避免每个详情页自己再写一套头图、按钮和列表节奏

## 6. 统一后的收益

### 6.1 视觉收益

- 页面更整齐
- 卡片更一致
- 密度更可控
- 控件更像一个系统

### 6.2 维护收益

- 调整节奏更容易
- 后续做主题切换更容易
- 新页面接入更容易复用现有规范

### 6.3 迭代收益

新增页面时，优先复用：

- token
- 公共详情样式
- 统一布局工具

避免重新设计基础样式。

## 7. 后续推荐

后续继续迭代，建议按以下顺序：

1. 新功能开发
2. 页面级最终验收
3. 个别页面精修

- 每个组件再加一层独立补丁样式
- 为同类页面重复写新的详情页外壳
