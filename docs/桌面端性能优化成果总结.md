## 优化详情

### 1. three.js / ogl 动态导入

**问题**：4 个背景效果 composable 在顶层同步导入 three.js 或 ogl，被 `PlayerExpanded.vue` 同步加载进主包，而所有背景效果默认关闭。

**修复**：将顶层 `import * as THREE from 'three'` 和 `import { ... } from 'ogl'` 改为在 `start()` 函数内动态 `import()`，仅当用户手动开启对应背景效果时才加载。

### 2. AMLL 歌词渲染器懒加载

**问题**：`LyricsPanel.vue` 顶层同步导入 `LyricPlayer` 及 AMLL CSS，`PlayerExpanded.vue` 同步导入 `BackgroundRender`。`useAmllRenderer` 默认 `false`，但 AMLL 始终被打包且在 DOM 中 `v-show`。

**修复**：改为 `defineAsyncComponent(() => import(...))`，渲染层改用 `v-if`，CSS 在异步加载时自动注入。

### 3. 高频 API 缓存

**问题**：首页 5 个接口（`/top/artists`、`/top/album`、`/top/song`、`/mv/all`、`/dj/recommend`）每次页面加载都重新请求，数据变化不频繁。

**修复**：接入 `apiCache`，TTL 5-10 分钟，后续访问直接读缓存。

### 4. 雷达歌单 14 个请求去重

**问题**：`fetchRadarPlaylists()` 通过 `Promise.allSettled` 并发调 14 次 `/playlist/detail`，无去重。

**修复**：缓存结果到 `apiCache`，TTL 5 分钟，同挂载周期内不重复请求。

### 5. resolveArtistDisplay 并发限流

**问题**：`normalizeLatestSongs()` 通过 `Promise.all` 同时发起最多 24 个 `/artist/detail` 请求。

**修复**：引入 `asyncPool` 工具函数，限制最多 4 个并发。

### 6. loadMoreHotSongs 不再阻塞

**问题**：`onMounted` 中 `await loadMoreHotSongs()` 在 `Promise.all` 之前串行执行，延迟所有后续模块加载。

**修复**：移入 `Promise.all` 并发执行，`setupHotSongsObserver()` 在 `nextTick` 后统一设置。

### 7. Vendor 拆分

**问题**：`vue`、`axios`、`lucide-vue-next` 等 vendor 库打包在主包中，每次部署后缓存失效。

**修复**：在 `vite.config.ts` 中添加 `manualChunks`：

```ts
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'vendor-vue': ['vue'],
        'vendor-axios': ['axios'],
        'vendor-icons': ['lucide-vue-next'],
      },
    },
  },
},
```

### 8. 二级页面懒加载

**问题**：`App.vue` 在顶层导入所有页面组件（Settings、Stats、Local、Podcast 等 16 个页面），全部打包进主包。

**修复**：改为 `defineAsyncComponent(() => import(...))`，仅在用户导航到对应页面时加载。
