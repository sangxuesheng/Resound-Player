# 侧栏折叠与折痕交互实现说明

本文档记录侧栏的展开/收起双状态切换机制、动画过渡、右侧边缘折痕线交互设计，以及持久化存储方案。

---

## 一、组件文件

- `src/components/Sidebar.vue` — 侧栏组件，包含折叠状态管理与折痕线交互

---

## 二、功能总览

```
┌─────────────────────┐
│  Sidebar            │ ← 固定定位，左侧边缘贴边
│  ┌───────────────┐  │
│  │ Profile       │  │ ← 头像 + 用户名（收起时仅保留头像）
│  └───────────────┘  │
│  ┌───────────────┐  │
│  │ Menu Item     │  │ ← 导航项（收起时仅保留图标）
│  │ Menu Item     │  │
│  │ ...           │  │
│  └───────────────┘  │
│               ║     │ ← 右侧边缘折痕线（点击切换）
└─────────────────────┘
      ◀—— 220px ——▶     展开态
      ◀—— 76px  ——▶     收起态
```

- **双状态**：展开（220px）/ 收起（76px）
- **文字渐隐**：收起时用户名、导航标签文字 `opacity → 0`、`max-width → 0`，保留图标
- **尺寸动画**：`width` 使用 `0.32s cubic-bezier(0.34, 1, 0.64, 1)` 缓动
- **右侧折痕线**：位于侧栏右边框中轴，常态灰色竖条，hover 变主题色 + 伸长，点击切换状态
- **本地存储**：折叠状态持久化到 `localStorage`（key: `tm_sidebar_collapsed`）

## 三、Props / Events

### Props

| Prop | 类型 | 默认 | 说明 |
|---|---|---|---|
| `activeKey` | `string` | — | 当前选中导航项 key |
| `collapsed` | `boolean` | `false` | 受控的折叠状态（可选受控模式） |

### Emits

| Event | 参数 | 说明 |
|---|---|---|
| `update:collapsed` | `v: boolean` | 折叠状态变化时通知父组件 |
| `select` | `key: string` | 导航项被选中 |

### 使用方式

```vue
<template>
  <Sidebar
    v-model:collapsed="sidebarCollapsed"
    :activeKey="currentRoute"
    @select="navigateTo"
  />
</template>
```

## 四、内部状态管理（半受控模式）

组件使用 `isCollapsed` 内部 ref 作为状态来源，同时通过 `watch` 同步父组件传入的 `collapsed` prop，实现**半受控**模式：

1. **初始化** — `onMounted` 时读取 `localStorage('tm_sidebar_collapsed')`，存在则覆盖初始值
2. **父组件同步** — `watch(() => props.collapsed)` 将外部值同步到内部
3. **状态变更** — 内部 `watch(isCollapsed)` 执行持久化 + 向上 `emit('update:collapsed')`

```
props.collapsed ──watch──▶ isCollapsed ──watch──▶ localStorage + emit
                              ▲
                         toggleCollapsed()
```

## 五、右侧边缘折痕线（Edge Crease）

### 5.1 交互区域

```
  Sidebar 右边缘        点击区域
     │                      │
  ┌──┼──────────────────┐   │
  │  │                  │◀──┤  .edge-crease (18px wide)
  │  │                  │   │  z-index: 20
  │  │    ║             │   │  .crease-line (10px × 48px 竖条)
  │  │  hover:  ║║     │   │
  │  │                   │
  └──┼──────────────────┘   │
     │                      │
```

- `.edge-crease` — 绝对定位在侧栏右侧外部（`right: -6px`），18px 宽点击区域
- `.crease-line` — 居中竖条，常态 `10px × 48px`，灰色 (`var(--border)`)
- **hover** → 宽度 `14px`、高度 `64px`、背景变 `var(--accent)`
- **active** → 高度收缩至 `40px`（模拟点击回弹）
- **点击** → 切换 `isCollapsed`，触发整体宽度动画

### 5.2 关键 CSS

```css
.edge-crease {
  position: absolute;
  top: 0;
  right: -6px;
  width: 18px;
  height: 100%;
  z-index: 20;
  cursor: pointer;
}

.crease-line {
  position: absolute;
  left: 12px;
  top: 50%;
  transform: translateY(-50%) translateX(-50%);
  width: 10px;
  height: 48px;
  border-radius: 3px;
  background: var(--border);
  opacity: 1;
  transition: opacity 0.22s ease, height 0.22s ease,
              width 0.22s ease, background 0.22s ease;
}

.edge-crease:hover .crease-line {
  width: 14px;
  height: 64px;
  background: var(--accent);
}

.edge-crease:active .crease-line {
  height: 40px;
  background: var(--accent);
}
```

### 5.3 设计要点

- 折痕线**始终可见**（`opacity: 1`），默认与右边框同色（`var(--border)`）
- hover 时变为主题色，暗示"此处可交互"
- 竖条居中跨越侧栏右边框线，视觉上像是边框本身折叠出了一个手柄
- 点击区域比视觉条宽（18px vs 10px），降低误触难度

## 六、动画系统

### 6.1 侧栏宽度动画

```css
.sidebar {
  transition: width 0.32s cubic-bezier(0.34, 1, 0.64, 1);
}
```

使用 `cubic-bezier(0.34, 1, 0.64, 1)` — 一个带轻微过冲的缓出曲线，类似 Apple 的弹性格变动画。

### 6.2 文字渐隐动画

```css
.collapsedText {
  opacity: 0;
  max-width: 0;
  margin: 0 !important;
}
```

所有需要收起时隐藏的文字（用户名、导航标签）挂 `collapsedText` class，通过 `transition: opacity 0.2s ease, max-width 0.2s ease, margin 0.2s ease` 实现平滑渐隐。

### 6.3 折痕线动画

`crease-line` 使用 0.22s ease 过渡，覆盖 `width`、`height`、`background`、`opacity` 四个属性。

## 七、本地存储

| Key | 值 | 说明 |
|---|---|---|
| `tm_sidebar_collapsed` | `"1"` | 收起 |
| `tm_sidebar_collapsed` | `"0"` | 展开 |

存储时机：`watch(isCollapsed)` 触发时自动写入。

恢复时机：`onMounted` 时读取，优先级高于父组件传入的 `collapsed` 默认值。

---

## 八、层级与定位

- 侧栏使用 `position: fixed`，由 `--layout-top` / `--layout-left` / `--sidebar-height` 变量控制位置
- 折痕线使用 `position: absolute` 相对于侧栏容器
- `z-index: 10`（侧栏）/ `z-index: 20`（折痕线交互区域）

---

## 九、复用方式

直接引入 `Sidebar.vue` 并传入 `activeKey` 与 `select` 事件即可。折叠状态由组件内部自动持久化。

如需外部控制折叠状态（如在路由变化时自动展开/收起），可通过 `v-model:collapsed` 绑定。

如需重置存储：`localStorage.removeItem('tm_sidebar_collapsed')`。
