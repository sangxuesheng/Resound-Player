# 心动模式动画效果实现说明

## 概述

进入/退出心动模式时，在页面顶层播放 Three.js 3D 彩色管道动画（Tubes Cursor），同时显示 Toast 通知。动画约 2.5 秒后自动淡出消失。

## 涉及文件

| 文件 | 类型 | 说明 |
|---|---|---|
| `src/components/effects/HeartbeatActivateEffect.vue` | 新建 | 动画组件 |
| `src/components/TopBar.vue` | 修改 | 集成动画触发逻辑 |

## 组件架构

```
TopBar.vue
  ├── [点击心动按钮] ──> handleIntelligencePlay()
  │     ├── 退出模式 ──> showGlobalToast() + showHeartbeatEffect = true
  │     └── 开启模式 ──> 调 API → showGlobalToast() + showHeartbeatEffect = true
  │
  └── <Teleport to="body">
        └── <HeartbeatActivateEffect :visible="showHeartbeatEffect" @done="..." />
              ├── canvas (position: fixed, z-index: 9999, pointer-events: none)
              ├── 动态导入 threejs-components（本地 npm 包，由 Vite 打包）
              ├── 递归查找 renderer → 设置透明清除色
              ├── 递归查找 camera → 启动随机缓动运动（替代鼠标跟踪）
              ├── 2.5s 后触发淡出 (CSS animation)
              └── 淡出完成 → emit('done') → 父组件重置状态
```

## 动画组件 HeartbeatActivateEffect.vue

### Props

| 名称 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `visible` | `boolean` | — | 控制动画显示/隐藏（v-if） |

### Events

| 名称 | 参数 | 说明 |
|---|---|---|
| `done` | 无 | 动画淡出完成时触发，父组件应重置 visible |

### 生命周期

1. `visible = true` → canvas 挂载 → `createTubes()` 执行
2. 拦截 `window.addEventListener('mousemove', ...)` → 加载 CDN 库 → 创建 Tubes 场景
3. 恢复 `addEventListener`，移除已捕获的 mousemove 监听器
4. 递归搜索 `renderer` → `setClearColor(0x000000, 0)` 使背景透明
5. 递归搜索 `camera` → 启动 `requestAnimationFrame` 随机缓动运动（easeInOutQuad，X 范围 ±1.5，Y 范围 ±0.75）
6. 2.5 秒后 `fadingOut = true` → CSS fade-out 动画（0.6s ease-in）
7. `animationend` → `destroyTubes()` → `emit('done')`

### 三层背景透明策略

Three.js WebGL 渲染器的默认清除色为黑色，通过三层方式去除：

1. **JS 层**：`makeTransparent()` 递归搜索 app 对象树，定位 `renderer`（检测 `setClearColor + render`）和 `scene`（检测 `background + children`）
2. **CSS 层**：非 scoped `<style>` 中的 `mix-blend-mode: screen`（黑色→透明，彩色保留）
3. **布局层**：canvas 本身无背景色，`pointer-events: none` 不阻断交互

### 随机运动算法（替代鼠标跟随）

```js
// 每帧以 0.015 步进从当前位置缓动到随机目标
camera.position.x ∈ [-1.5, 1.5]
camera.position.y ∈ [-0.75, 0.75]
ease = easeInOutQuad(progress)
// 每帧有约 0.5% 概率选择新目标
```

### 触发时机

| 场景 | Toast | 动画 |
|---|---|---|
| 手动退出心动模式 | `已退出心动模式` (success) | Tubes 动画 |
| 成功开启心动模式 | `已开启心动模式` (success) | Tubes 动画 |
| 切换歌单自动退出 | `已退出心动模式` (success) | Tubes 动画 |
| 无有效歌单 | `暂无心动推荐` (warning) | 无 |
| API 请求失败 | 错误消息 (warning) | 无 |

### 流程细节

```typescript
async function handleIntelligencePlay() {
  if (已激活) {
    退出 → showGlobalToast + showHeartbeatEffect = true
    return
  }
  if (!checkAuth()) return
  if (!有效歌单) { showGlobalToast(warning); return }

  const err = await playIntelligenceList()
  if (err) showGlobalToast(err, warning)
  else {
    isIntelligenceActive = true
    showGlobalToast(success)
    showHeartbeatEffect = true
  }
}

// 歌单切换时自动退出
watch(currentPlaylistId) {
  if (isIntelligenceActive) {
    isIntelligenceActive = false
    showGlobalToast(success)
    showHeartbeatEffect = true
  }
}
```

## 导入方式

动态导入而非静态 import，目的是在模块代码执行前通过 `blockMouseMove()` 拦截 `window.addEventListener('mousemove', ...)`：

```ts
// HeartbeatActivateEffect.vue
blockMouseMove()  // 先拦截 addEventListener
import('threejs-components/build/cursors/tubes1')  // 再加载库
  .then((mod) => {
    // 库初始化时会注册 mousemove 监听器（已被拦截）
    if (restoredMouseHandler) restoredMouseHandler()  // 恢复并移除捕获的监听器
    // ... 创建场景、设置透明背景、启动随机运动
  })
```

## 注意事项

1. **本地打包**：`threejs-components` 已加入 `package.json`，由 Vite 打包，无需 CDN，离线可用
2. **递归搜索**：`makeTransparent()` 和 `findCamera()` 深度限制为 4-5 层，防止栈溢出
3. **事件拦截**：`blockMouseMove()` 临时替换 `window.addEventListener`，库初始化后立即恢复并清理，不影响其他模块
4. **随机运动**：每帧约 0.5% 概率重选目标，运动方向不可预测
5. **清除条件**：`isFirstIntelligenceSuccess` flag 已被移除（早期用于限制首次 Toast，现每次进入均触发动画）
