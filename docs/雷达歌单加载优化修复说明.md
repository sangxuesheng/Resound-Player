# 雷达歌单加载优化修复说明

## 背景

雷达歌单（私人雷达、时光雷达、新歌雷达等）在详情页仅显示首次加载的 30 首歌曲，无法动态加载剩余歌曲。用户在首页点击雷达歌单卡片后，`PlaylistDetailPage.vue` 通过 `getPlaylistDetail(id, 30)` 获取到 30 首歌曲后停止，无论该歌单总共有多少首。

## 根因分析

通过 `/cloudsearch?type=1000` 搜索歌单 API 和 `/playlist/detail` 查看详情 API 的测试，发现：

### 1. 雷达歌单的 API 响应特征

所有雷达歌单的 `/playlist/detail` 响应中：

- `trackIds` 始终为空数组（`[]`）— 动态歌单无预分配 ID
- `trackCount` 正常返回（如新歌雷达 = 5689 首）
- `tracks` 仅返回前 N 首（受 `s` 参数影响，上限约 30 首）
- `/playlist/track/all` 仍然可以正常分页获取全部歌曲

### 2. 原有逻辑缺陷

```typescript
// 原有逻辑只依赖 trackIds
const trackIds = detail.trackIds || [];
if (!trackIds.length) return;  // ❌ 雷达歌单 trackIds 为空，直接退出
```

由于 `trackIds` 为空，补全逻辑被跳过，导致仅显示首次的 30 首。

## 涉及的修改

### 文件 1：`src/components/HomePanel.vue` — 修复歌单 ID

对部分雷达歌单的硬编码 ID 进行了修复/确认：

| 歌单名 | 旧 ID | 新 ID | 说明 |
|--------|-------|-------|------|
| 新歌雷达 | 5300458264 | 13139913381 | 旧 ID 已停用，新 ID trackCount=5689 |
| 时光雷达 | 5320167908 | — | 确认是官方 ID，保留不变 |
| 乐迷雷达 | 5327906368 | — | 确认是官方 ID，保留不变 |
| 神秘雷达 | 5341776086 | — | 确认是官方 ID，保留不变 |

**注意：** 时光雷达/神秘雷达/乐迷雷达的官方 ID 返回的 `trackCount` 较小（约 30），这是官方歌单本身的限制，而非 ID 错误。不应替换为用户创建的歌单。

### 文件 2：`src/components/PlaylistDetailPage.vue` — 修复补全逻辑

**修改位置：** `fetchDetail()` 函数中的歌曲补全逻辑。

```typescript
// 新增：获取 trackCount
const trackCount = detail.trackCount || 0;

// 改为：用 trackCount 兜底（雷达歌单不返回 trackIds 但返回 trackCount）
const max = trackIds.length || trackCount;
if (max <= rawTracks.length) return;

// 然后进入 /playlist/track/all 分页补全循环
```

#### 改动的核心点

1. **新增 `trackCount` 提取**：从 API 响应中获取歌单总歌曲数
2. **补全上限计算**：优先使用 `trackIds.length`，若为空则降级为 `trackCount`（`const max = trackIds.length || trackCount`）
3. **补全条件判断**：`max > rawTracks.length` 时进入补全（而非依赖 `trackIds` 非空）
4. **分页补全**：使用 `getPlaylistTrackAll` 以 `limit=30` 为步长从 `offset=30` 开始逐页补全
5. **去重保护**：合并前按 `song.id` 去重，防止 API 返回重复数据导致死循环

## 验证方法

### API 验证（无需登录）

```bash
# 搜索新歌雷达，确认正确 ID
curl 'http://127.0.0.1:38761/cloudsearch?keywords=新歌雷达&type=1000&limit=5'

# 查看歌单详情，确认 trackCount
curl 'http://127.0.0.1:38761/playlist/detail?id=13139913381'

# 分页补全测试
curl 'http://127.0.0.1:38761/playlist/track/all?id=13139913381&limit=30&offset=30'
```

### 功能验证

1. 打开首页，点击"新歌雷达"卡片
2. 进入详情页后观察：
   - 页面顶部显示"歌曲总数：5689"
   - 歌曲列表持续加载，底部显示"歌曲列表补全中…"
   - 等待加载完成后可浏览所有歌曲
3. 同样测试其他雷达歌单（时光、神秘、宝藏等）

## 已知问题

- **乐迷雷达 (5327906368)**：API 返回 `trackCount=0`，详情页无法加载任何歌曲。这是官方歌单本身的问题，与代码无关。
- **宝藏雷达等 trackCount=30 的歌单**：即使补全逻辑正常运行，官方数据本身只有 30 首，不会加载更多。
