# 详情页公共样式规范

本规范约束项目内所有详情页（PlaylistDetailPage、AlbumDetailPage、ArtistDetailPage、UserDetailPage、PodcastDetailPage、LanguageDetailPage、SearchPage 等）的样式开发方式。

## 1. 背景

2026-03 重构前，`artist-link`、`meta-pill`、`playlist-detail-page--embedded`、`song-item--playing .song-cover`、`podcast-shell` 这 5 组样式在 **10 个组件** 的 `<style scoped>` 块中各自重复定义，改一处漏多处。

重构后所有公共样式收敛到 `src/styles/detail-page.css`，组件只需 `@import '../styles/detail-page.css'` 即可使用。

## 2. 公共样式清单

以下所有 class 定义在 `src/styles/detail-page.css`，新增详情页**禁止在 scoped 样式中重新定义**同名前缀的规则。

### 2.1 页面骨架

| Class | 用途 |
|---|---|
| `.playlist-detail-page` | 详情页根容器 — flex column、`height: 100%`、自带负 margin 撑满父级宽度 |
| `.playlist-detail-body` | 详情页滚动内容区 — `flex: 1`、`min-height: 0` |
| `.playlist-detail-page--embedded` | 嵌入模式 — 取消自身滚动，`overflow: visible`、`margin: 0`、移除 padding/border/background/shadow |

### 2.2 面板

| Class | 用途 |
|---|---|
| `.panel` | 通用面板 — `border-radius`、`padding: var(--space-4)`、glass reflection 背景、自带 `::before` 封面模糊渐变 |
| `.daily-page` | 每日推荐页面板 — 与 `.panel` 同源，附加自身 `::before` 渐变 |
| `.user-detail-page` | 用户详情页面板 — 与 `.panel` 同源，附加自身 `::before` 渐变 |
| `.detail-panel` | 内嵌详情面板（UserSplitView 分屏场景）— 独立的 accent 渐变体系，依赖 `--panel-accent`、`--detail-cover`、`--detail-hero-rgb` |
| `.rank-detail-panel` | 排行榜详情面板 — `.detail-panel` 的白色渐变变体 |

**规则**：新增详情页面板时，优先套用 `.panel`。若需要 accent 颜色驱动的渐变背景（分屏场景），套用 `.detail-panel` 并通过 CSS 变量注入颜色。

### 2.3 header / 封面

| Class | 用途 |
|---|---|
| `.header` | 详情页头部网格 — `grid-template-columns: 269px 1fr`、`gap: var(--space-4)` |
| `.hero-media-shell` | 封面媒体容器 — `position: relative`、`border-radius: 16px`、`isolation: isolate` |
| `.cover` | 封面图片 — 269×269、`object-fit: cover`、hover 时通过 `--image-hover-scale` 缩放 |
| `.title` | 详情标题 — `--text-headline-xl` |
| `.sub` | 副标题 / 艺术家名 — `--text-label-md`、`--text-sub` |
| `.desc` | 描述文字 — `--text-label-md`、`--text-sub`、`white-space: pre-wrap` |
| `.desc--collapsed` | 描述折叠态 — 2 行截断 |
| `.desc-toggle` | 描述展开/收起按钮 — accent 色、hover 下划线 |
| `.ops` | 操作按钮区 — `margin-top: 12px` |
| `.back-btn` | 返回按钮 — `border-radius: 999px`、glass 背景、hover 上浮 |
| `.playlist-detail-back` / `.artist-detail-back` | 返回按钮容器 — opacity 由 `--sticky-progress` 驱动 |

**规则**：详情页头部的封面、标题、副标题、描述、操作区必须使用上述 class，不要自行定义新的 `font-size`、`margin`、`padding` 布局。

### 2.4 歌曲列表

| Class | 用途 |
|---|---|
| `.song-list` | 歌曲列表容器 — `border-radius: 18px`、半透明背景、上边框分隔 |
| `.song-list--search` | 搜索页歌曲列表变体 — 去除 padding/border/背景 |
| `.song-item` | 歌曲行 — `grid-template-columns: 40px 52px 1fr auto`、`padding: 10px 12px`、`border-radius: 16px` |
| `.idx` | 序号列 — 居中、`--text-soft` |
| `.song-cover` | 歌曲封面 — 52×52、`border-radius: 10px` |
| `.song-name` | 歌名 — `font-weight: 600`、`--text-main` |
| `.song-artist` | 歌手名 — `font-size: var(--text-label-sm)`、`--text-sub` |
| `.song-item--playing .song-cover` | 正在播放歌曲封面高亮 — accent 色 box-shadow |
| `.play-btn` | 行内播放按钮 — 32px 高、hover 上浮 |

**规则**：新增歌曲列表时，`<li>` 使用 `.song-item`，内部使用 `.idx`、`.song-cover`、`.song-name`、`.song-artist` 的结构。不要在 scoped 中重新定义这些 class。

### 2.5 操作按钮

| Class | 用途 |
|---|---|
| `.play-all` | 播放全部按钮 — 36px 高、圆角 pill、accent 渐变背景、hover 上浮 + 亮度提升 |
| `.add-to-queue` | 添加到队列按钮 — 与 `.play-all` 共用同一套样式 |

### 2.6 歌手链接

| Class | 用途 |
|---|---|
| `.artist-link` | 内联可点击歌手名 — 继承父级字体颜色、hover 变 accent + 下划线 |
| `.artist-link--inline` | 行内变体 — `display: inline` |

**用法**：

```html
<button class="artist-link">歌手A</button>
<button class="artist-link artist-link--inline">歌手B</button>
```

多个 `.artist-link` 相邻时，间隔自动添加 `/` 分隔符。

### 2.7 元数据标签

| Class | 用途 |
|---|---|
| `.meta-pill` | 小药丸标签 — `border-radius: 999px`、半透明背景 + 边框、`--text-label-sm` |

**用法**：

```html
<span class="meta-pill">123 首</span>
<span class="meta-pill">2024</span>
```

### 2.8 播客网格

| Class | 用途 |
|---|---|
| `.podcast-shell` | 播客页面网格容器 — `display: grid`、`grid-template-columns: minmax(0, 1fr)`、`gap: var(--space-4)` |

### 2.9 状态提示

| Class | 用途 |
|---|---|
| `.state` | 加载/空态提示 — `padding: var(--space-6) 0`、`--text-soft` |
| `.state.error` | 错误态 — `--danger` 色 |

## 4. 禁止行为

以下行为在新页面和组件中**严格禁止**：

1. **禁止在 scoped 中重新定义 `detail-page.css` 已有的 class 规则。** 例如：
   ```css
   /* ❌ 禁止 — detail-page.css 已定义 */
   .artist-link { border: 0; padding: 0; ... }
   .meta-pill { display: inline-flex; ... }
   .song-item--playing .song-cover { box-shadow: ... }
   .podcast-shell { display: grid; ... }
   .playlist-detail-page--embedded { overflow: visible; ... }
   ```

2. **禁止在多个页面中复制粘贴相同的 scoped 覆盖。** 如果你的 `.playlist-detail-page--embedded` 覆盖在两个地方出现，说明该提取到 `detail-page.css`。

3. **禁止在 scoped 中覆盖 `detail-page.css` 的 token 级属性为硬编码值。** 例如 `font-size: 14px` 覆盖 `var(--text-label-sm)` —— 如果确实需要不同字号，使用更高层级的 token（如 `--text-label-md`）。

4. **禁止在页面中单独定义 artist-link / meta-pill / podcast-shell 的 hover / separator 逻辑。** 这些已由公共样式统一处理。

## 5. Code Review 检查项

新增或修改详情页时，必须检查：

- [ ] 是否 `@import '../styles/detail-page.css'`？
- [ ] `artist-link`、`meta-pill`、`song-item`、`play-all` 等 class 是否使用了公共定义（而非 scoped 重写）？
- [ ] 嵌入模式是否需要 `.playlist-detail-page--embedded`（无需自行写 `overflow: visible` 等）？
- [ ] 播客页面是否使用了 `.podcast-shell`（无需自行定义 `display: grid`）？
- [ ] 歌手链接是否使用了 `.artist-link`（无需自行定义 hover 颜色和下划线逻辑）？
- [ ] 是否存在可删除的、与 `detail-page.css` 重复的 scoped 规则？

## 6. 扩展公共样式

如果多个页面出现同一组 scoped 覆盖，按以下流程处理：

1. 确认覆盖在两个以上页面中**字符级相同**
2. 在 `detail-page.css` 末尾追加（带 `/* ===== */` 分隔注释）
3. 从各组件 scoped 中删除重复规则
4. 更新本规范文档的“公共样式清单”章节
