# 返回顶部悬浮按钮复用说明（ScrollToTopFab）

## 目的

统一项目中「页面滚动到一定距离后右下角出现返回顶部悬浮按钮」的交互与视觉样式，避免不同页面各自复制一套逻辑和 CSS。

目前该组件已统一复用到以下场景：

- 详情页（歌单 / 专辑 / 歌手 / 用户详情 / 语言 / 播客）
- 收藏历史页（播放历史 + 我的收藏双栏面板）
- 用户中心分屏（左栏列表 + 右栏详情）
- MV 浏览页
- 歌单分类页
- 排行榜页
- 搜索页
- 播客列表 / 分类 / 订阅页
- 歌曲评论页

## 统一资源位置

### 组件

- `src/components/ui/ScrollToTopFab.vue`

职责：

- 渲染 48×48 圆形悬浮按钮
- 外圈 SVG 环形进度条（`--accent` 绿色）显示当前滚动进度
- 中心 ↑ 箭头
- 内部通过 `useBackToTop` composable 控制显隐与进度
- 支持 `fixed`（主内容区固定定位）和 `sticky`（分屏面板吸附定位）两种模式

### Composable

- `src/composables/useBackToTop.ts`

职责：

- RAF 节流的滚动检测
- 返回 `visible`（是否超过阈值且在滚动中）、`progress`（0~1）、`scrolling`（是否正在滚动）
- 内置 1500ms 滚动停止检测：停止滚动 1.5s 后自动隐藏
- 支持函数式 `scrollHostSelector`，与 `useDetailStickyState` 模式一致
- 组件卸载时自动清理事件和定时器

## Props

| Prop | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `scrollHostSelector` | `string \| (() => string \| undefined)` | `'.content'` | 滚动宿主的选择器，支持函数式动态获取 |
| `threshold` | `number` | `400` | 滚动超过多少像素后显示按钮 |
| `fixed` | `boolean` | `false` | `true` 时使用 `position: fixed` 固定在主区域右下角；`false` 时使用 `position: sticky` 在面板内吸附 |

## 视觉效果

统一后的返回顶部按钮具备以下特征：

- 48×48 圆形，`--bg-solid` 不透明背景 + 投影
- 外圈半透明灰色 track 环 + `--accent` 绿色前景进度环（`stroke-dashoffset` 驱动）
- 中心 ↑ 箭头，默认 `--text-sub` 色，hover 变 `--accent` 色
- hover 上浮 2px，active scale(0.96)
- 滚动超过阈值且在滚动中才显示，停止滚动 1.5s 后自动隐藏
- 尊重 `prefers-reduced-motion`

## 接入方式

### 方式一：主内容区（推荐单栏页面）

直接在 App.vue 的 `.main-area` 层级添加，使用 `fixed` 模式：

```html
<ScrollToTopFab v-if="showBackToTop" :fixed="true" />
```

`position: fixed; bottom: calc(84px + 24px); right: 32px;` 固定在播放栏上方、内容区右下角，不依赖任何滚动容器内部定位。

### 方式二：分屏面板内（双栏 / 多栏页面）

在面板的滚动容器内添加，使用默认 `sticky` 模式：

```html
<div class="scroll-container">
  <!-- 列表内容 -->
  <ScrollToTopFab scrollHostSelector=".scroll-container" />
</div>
```

`position: sticky; bottom: 32px;` 吸附在面板滚动容器底部。

### 方式三：独立列表容器

当列表容器（如 `.collection-list`）有独立 `overflow: auto` 时，在列表末尾添加：

```html
<div class="collection-list collection-list--favorites">
  <div v-for="item in items">...</div>
  <ScrollToTopFab scrollHostSelector=".collection-list--favorites" />
</div>
```

## 条件显示

`scrollHostSelector` 和 `threshold` 参数控制何时出现。组件内部通过 `useBackToTop` composable 自动管理：

- **超过阈值 + 正在滚动** → 按钮显示 + 进度环实时更新
- **超过阈值 + 停止滚动 1.5s** → 按钮隐藏
- **低于阈值** → 始终隐藏

## 当前接入清单

| 页面 / 区域 | scroll 宿主 | 定位方式 | FAB 位置 |
|---|---|---|---|
| 所有使用 `.content` 滚动的页面 | `.content` | `position: fixed` | `App.vue` → `.main-area` |
| 用户中心分屏 — 左栏 | `.left-panel` | `position: sticky` | `UserSplitView.vue` → 左栏末尾 |
| 用户中心分屏 — 右栏 | `.detail-panel` | `position: sticky` | `UserSplitView.vue` → 右栏末尾 |
| 历史页 — 收藏列表 | `.collection-list--favorites` | `position: sticky` | `HistoryPanel.vue` → 列表末尾 |
| 历史页 — 历史列表 | `.collection-list--history` | `position: sticky` | `HistoryPanel.vue` → 列表末尾 |

## 注意事项

1. 弹窗/面板内使用 `sticky` 模式时，父容器必须有 `overflow` 属性（auto/scroll）作为滚动容器
2. 主内容区始终使用 `fixed` 模式，避免 `.content-shell` 的 `height: 100%` 影响 sticky 定位
3. `useBackToTop` 的 `scrollHostSelector` 支持 `() => string` 函数式写法，适合动态切换滚动宿主的场景
