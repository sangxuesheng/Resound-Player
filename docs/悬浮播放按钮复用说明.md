# 悬浮播放按钮复用说明

## 目的

统一项目中“鼠标移入封面后显示播放按钮”的交互与视觉样式，避免不同页面各自复制一套按钮 DOM 和 CSS。

当前这套悬浮播放按钮已经统一复用到以下页面：

- MV 页面
- 排行榜页面
- 歌单页面
- 首页最新专辑推荐

## 统一资源位置

### 组件

- `src/components/HoverPlayButton.vue`

职责：

- 只负责渲染悬浮播放按钮本体
- 不关心具体业务数据
- 不绑定点击播放逻辑
- 主要用于 hover 视觉提示

### 样式

- `src/styles/hover-play-button.css`

职责：

- 定义悬浮播放按钮的通用视觉样式
- 定义 hover/focus 时按钮显示、角标隐藏的统一过渡
- 供多个页面直接引入复用

## 视觉效果

统一后的播放按钮具备以下特征：

- 默认隐藏
- 鼠标移入卡片时淡入出现
- 位置通常在封面右下角
- 毛玻璃圆形按钮
- 中间为白色三角播放图标
- 可与右上角播放量角标联动：
  - 默认显示播放量
  - hover 时播放量淡出
  - 同时播放按钮浮现

## 通用样式类

`hover-play-button.css` 中的核心类：

### 按钮本体

- `.hover-play-button`
- `.hover-play-button__icon`

### 触发器

- `.hover-play-button-trigger`

给卡片根节点或 hover 宿主加上这个类后，按钮会在 hover / focus 时出现。

### 角标联动

- `.hover-play-button-count`
- `.hover-play-button-count-trigger`

用于控制播放量、统计数等角标在 hover 时淡出。

### 尺寸分档与变量

通用样式支持通过 CSS 变量控制按钮大小，并且支持分档类。

核心变量：

- `--hover-play-button-size`
- `--hover-play-button-offset`

常用分档类：

- `.hover-play-button--sm`
- `.hover-play-button--md`
- `.hover-play-button--lg`

推荐原则：

- 大卡使用大号按钮
- 中卡使用中号按钮
- 小卡使用小号按钮
- 如需更精细控制，优先覆盖变量，不要重写整套按钮样式

## 页面接入方式

### 1. 引入通用按钮组件

在页面组件中引入：

- `src/components/HoverPlayButton.vue`

### 2. 引入通用样式

在页面组件中引入：

- `src/styles/hover-play-button.css`

例如在 `<style scoped>` 中：

```css
@import '../styles/hover-play-button.css';
```

### 3. 给卡片根节点添加触发类

例如：

```html
<button class="card hover-play-button-trigger">
```

### 4. 在封面区域插入通用按钮组件

例如：

```html
<div class="cover">
  <img src="..." alt="..." />
  <span class="count hover-play-button-count">123万</span>
  <HoverPlayButton class="hover-play-button--md" />
</div>
```

如果需要在具体卡面中自定义大小，也可以直接在封面容器或卡片容器上设置：

```css
.cover {
  --hover-play-button-size: 34px;
  --hover-play-button-offset: 9px;
}
```

## 定位规则

通用按钮样式默认定位为：

- `right: var(--hover-play-button-offset)`
- `bottom: var(--hover-play-button-offset)`

默认偏移量一般由分档或页面容器变量决定。

如果某类卡片结构特殊，可以在页面局部覆盖定位。

例如排行榜页中的 `曲风榜`：

- 通过局部类覆盖按钮位置
- 保证按钮显示在封面区域上，而不是卡片文字区

推荐原则：

- 默认优先使用通用定位
- 默认优先通过变量控制大小和边距
- 只有在卡片结构不同、按钮落点不合适时，才做局部覆盖

## 推荐复用规范

后续如果要在新页面继续接入这套按钮，建议按以下规范执行：

1. 优先复用 `HoverPlayButton.vue`
2. 优先复用 `hover-play-button.css`
3. 不再手写新的播放按钮 DOM
4. 不再复制一套新的按钮视觉样式
5. 优先使用分档类或变量控制尺寸
6. 如需特殊位置，只覆盖定位，不重写整套动画和视觉

## 建议的接入模板

可直接参考以下结构：

```html
<article class="card hover-play-button-trigger">
  <div class="cover">
    <img src="..." alt="..." />
    <span class="count hover-play-button-count">123万</span>
    <HoverPlayButton />
  </div>
  <div class="info">
    <p class="name">标题</p>
    <p class="sub">副标题</p>
  </div>
</article>
```

## 推荐尺寸分档

当前项目建议按以下思路使用：

- 大卡：`hover-play-button--lg`
  - 例如推荐榜、精选榜
- 中卡：`hover-play-button--md`
  - 例如语种榜、曲风榜、歌单卡
- 小卡：`hover-play-button--sm`
  - 例如车友榜、首页专辑卡

如果分档仍然不够精确，可在具体容器上额外覆盖：

- `--hover-play-button-size`
- `--hover-play-button-offset`

推荐做法：

- 先选最接近的分档
- 再做少量变量微调
- 不建议每个页面单独重写一套固定尺寸按钮

## 维护建议

如果后续还会继续扩展播放器视觉规范，建议下一步把“定位变体”也抽象出来，例如：

- 默认右下角
- 封面内右下角
- 紧贴封面边缘
- 中央播放按钮

这样可以让不同页面只切换定位变体，而不需要再写局部覆盖样式。
