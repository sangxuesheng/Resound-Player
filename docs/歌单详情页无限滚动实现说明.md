# 歌单详情页无限滚动实现说明

## 背景

歌单详情页（`PlaylistDetailPage.vue`）在加载雷达歌单等大歌单（如新歌雷达 5689 首）时，原有逻辑使用 while 循环一次性拉取全部歌曲，导致：

- 串行发出大量 API 请求（5689 ÷ 30 ≈ 190 次）
- 每次返回 30 首触发一次 Vue 响应式更新，造成页面卡顿
- 无法写入 localStorage（数据量太大）
- 补全数据不缓存，切页返回后重跑

## 解决方案

改为 **无限滚动（Infinite Scroll）** 模式：首次加载 30 首，用户滚动到底部时自动加载下 30 首，以此类推。

## 核心改动

### 文件：`src/components/PlaylistDetailPage.vue`

### 1. 移除 eager 补全循环

删除了 `fetchDetail()` 中的 `while (offset < max)` 循环（原 ~60 行），替换为：

```typescript
// 仅设置初始状态，不再立即加载全部
if (max > rawTracks.length) {
  infiniteState.value = {
    loading: false,
    hasMore: true,
    loaded: rawTracks.length,
    total: max,
  };
}
```

### 2. 新增 `infiniteState` 响应式状态

```typescript
const infiniteState = ref({ loading: false, hasMore: false, loaded: 0, total: 0 });
```

替代原有的 `trackEnriching` 布尔值。包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `loading` | boolean | 是否正在加载下一页 |
| `hasMore` | boolean | 是否还有更多歌曲可加载 |
| `loaded` | number | 已加载的歌曲数 |
| `total` | number | 歌单总歌曲数 |

### 3. 新增 `loadMoreSongs()` 函数

```typescript
async function loadMoreSongs(id: number) {
  if (infiniteState.value.loading || !infiniteState.value.hasMore) return;
  // ...
  const { data: chunkRes } = await getPlaylistTrackAll({ id, limit: 30, offset: st.loaded });
  // 去重后追加到 playlist.value.tracks
}
```

每次请求 30 首，从当前 `infiniteState.loaded` 位置开始偏移。

### 4. `IntersectionObserver` 哨兵监听

```typescript
const sentinelRef = ref<HTMLElement | null>(null);
let scrollObserver: IntersectionObserver | null = null;

function setupInfiniteScroll() {
  scrollObserver?.disconnect();
  scrollObserver = null;
  requestAnimationFrame(() => {
    const sentinel = sentinelRef.value;
    if (!sentinel) return;
    scrollObserver = new IntersectionObserver(
      (entries) => {
        if (entries[0]?.isIntersecting && infiniteState.value.hasMore && playlist.value?.id) {
          loadMoreSongs(Number(playlist.value.id));
        }
      },
      { rootMargin: '400px' },
    );
    scrollObserver.observe(sentinel);
  });
}
```

`rootMargin: '400px'` 表示在哨兵进入视口前 400px 就触发加载，配合虚拟滚动提前预加载。

### 5. 虚拟滚动

在无限滚动数据加载的基础上，将歌曲列表从全量 DOM 渲染改为 **虚拟滚动（Virtual Scrolling）**，避免数千首歌曲同时渲染导致卡顿。

#### 5.1 固定行高

```typescript
const ROW_HEIGHT = 68;
const OVERSCAN = 15;
```

每行固定 68px，上下各渲染 15 行（1020px）的缓冲。使用 `ResizeObserver` 测量容器实际高度：

```typescript
onMounted(() => {
  if (songListRef.value) {
    songListHeight.value = songListRef.value.clientHeight;
    songListResizeObserver = new ResizeObserver(() => {
      if (songListRef.value) {
        songListHeight.value = songListRef.value.clientHeight;
      }
    });
    songListResizeObserver.observe(songListRef.value);
  }
});
```

#### 5.2 可见范围计算

```typescript
const visibleRange = computed(() => {
  const total = filteredTracks.value.length;
  if (!total) return { start: 0, end: 0 };
  const start = Math.max(0, Math.floor(songListScrollTop.value / ROW_HEIGHT) - OVERSCAN);
  const end = Math.min(total, Math.ceil((songListScrollTop.value + songListHeight.value) / ROW_HEIGHT) + OVERSCAN);
  return { start, end };
});
```

#### 5.3 模板结构

虚拟列表使用 `position: absolute` + `transform: translateY` 定位，spacer 提供完整滚动高度：

```html
<div ref="songListRef" class="song-list song-list--virtual">
  <div :style="{ height: `${totalListHeight}px`, position: 'relative' }">
    <div
      v-for="vi in visibleTracks"
      :key="vi.track?.id || vi.index"
      class="song-item"
      :style="{
        position: 'absolute',
        top: 0,
        left: 0,
        width: '100%',
        height: `${ROW_HEIGHT}px`,
        transform: `translateY(${vi.index * ROW_HEIGHT}px)`,
      }"
    >
      ...
    </div>
  </div>
</div>
```

#### 5.4 滚动架构——与吸顶态共存

虚拟列表不再使用内部 `@scroll` 事件。`.playlist-detail-page` 自身作为唯一滚动容器（`overflow: clip auto`），`useDetailStickyState` 和虚拟滚动位置都监听该容器的 scroll（引用在 `onMounted` 中缓存，避免每帧 `document.querySelector`）：

```typescript
let cachedPageEl: HTMLElement | null = null;

function onSongListScroll(): void {
  const list = songListRef.value;
  if (cachedPageEl && list) {
    songListScrollTop.value = Math.max(0, cachedPageEl.scrollTop - list.offsetTop);
  }
}
```

| 角色 | 实现 |
|------|------|
| 滚动容器 | `.playlist-detail-page`（`overflow: clip auto` + `overflow-clip-margin: 16px`） |
| 虚拟列表容器 | `.song-list--virtual`（`overflow: visible; flex: 1`），超高 spacer 溢出传播到页面级滚动 |
| 吸顶 scroll host | `useDetailStickyState({ scrollHostSelector: '.playlist-detail-page' })` |
| 虚拟位置计算 | `cachedPageEl.scrollTop - list.offsetTop`（每帧计算，自动适应吸顶态高度变化） |

### 5. 模板哨兵元素（现嵌入虚拟列表）

```html
<div ref="songListRef" class="song-list song-list--virtual">
  <div :style="{ height: `${totalListHeight}px`, position: 'relative' }">
    ...v-for 虚拟项...
  </div>
  <div
    v-if="infiniteState.hasMore"
    ref="sentinelRef"
    class="infinite-sentinel"
  >
    <span v-if="infiniteState.loading" class="infinite-loading">加载中…</span>
    <span v-else class="infinite-tip">向下滑动加载更多</span>
  </div>
</div>
```

哨兵始终嵌入在虚拟列表容器内，随无限滚动加载更多歌曲后自动重新挂载。

### 6. 哨兵自动挂载

```typescript
watch(
  () => infiniteState.value.hasMore,
  (hasMore) => {
    if (hasMore) {
      requestAnimationFrame(() => setupInfiniteScroll());
    }
  },
  { immediate: true },
);
```

`{ immediate: true }` 确保首次渲染时立即挂载。

### 7. 观察者清理

```typescript
onBeforeUnmount(() => {
  scrollObserver?.disconnect();
  scrollObserver = null;
});
```

### 8. 模块级会话缓存

```typescript
const sessionTrackCache = new Map<number, { tracks: any[]; total: number }>();
```

内存 Map，不写入 localStorage，同一会话内导航返回不重复加载。

## 工作流程

```
进入歌单详情页
  → getPlaylistDetail(id, 30) 获取前 30 首 + trackCount
  → 设置 infiniteState = { loading: false, hasMore: true, loaded: 30, total: 5689 }
  → watch 触发 setupInfiniteScroll()，挂载 IntersectionObserver

用户滚动到底部（或接近底部 200px）
  → 哨兵元素进入视口
  → loadMoreSongs(id) 请求 offset=30，limit=30
  → 追加到 playlist.value.tracks
  → 更新 infiniteState.loaded = 60
  → 若 loaded < total，哨兵继续存在；否则自动消失

用户切换到评论标签再切回
  → sentinelRef 重新挂载（v-if hasMore）
  → watch 触发 setupInfiniteScroll()，继续监听
```

## 性能优化对比

| 指标 | 原有 eager 模式 | 无限滚动 + 虚拟滚动 |
|------|----------------|-------------------|
| API 请求数 | ~190 次（一次性） | 按需（首次 1 次，后续每次滚动 1 次） |
| DOM 节点数 | 全部歌曲的 li（大歌单数千个） | 30-60 个实时项 + spacer |
| 响应式更新次数 | ~190 次 | ~1 次/每次加载 |
| 首屏歌词加载 | 慢（等全部拉完） | 快（30 首立即展示） |
| 页面卡顿 | 持续卡顿 | 无卡顿 |
| localStorage | 无法缓存（数据太大） | 不写 localStorage（内存 Map 缓存） |
| 用户感知 | 等待全部加载完 | 边滚动边加载，体验流畅 |

## 依赖

- `IntersectionObserver` — 浏览器原生 API，无需额外依赖
- `ResizeObserver` — 测量虚拟列表容器高度
- Vue `ref` / `computed` / `watch` / `onMounted` / `onBeforeUnmount`
