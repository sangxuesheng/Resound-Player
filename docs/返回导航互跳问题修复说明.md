# 返回导航互跳问题修复说明

本文档记录本次“多个页面跳转后点击返回，会在两个页面之间反复跳转”的问题原因、修复方案和后续复核要点。

## 1. 问题现象

用户连续测试多个页面跳转后，再点击页面内返回按钮，偶发出现：

- 返回不是按真实访问顺序回退
- 某两个页面之间持续来回跳转
- 多层详情页跳转后，旧的来源页面状态污染新的返回路径

典型链路包括但不限于：

```text
歌单详情 → 用户详情 → 歌单详情 → 返回 → 用户详情 → 返回 → 歌单详情 ...
```

```text
专辑详情 → 歌手详情 → 专辑详情 → 返回 → 歌手详情 → 返回 → 专辑详情 ...
```

## 2. 根因

修复前，`src/App.vue` 的返回逻辑主要依赖各类详情页自己的 `returnPage` 状态，例如：

- `activePlaylistReturnPage`
- `activeAlbumReturnPage`
- `activeArtistReturnPage`
- `activeUserReturnPage`
- `activeLanguageReturnPage`
- `activeMvReturnPage`

这种机制只能表达“当前页面来自哪个页面类型”，不能表达真实访问历史。

当出现 A 页面打开 B 页面、B 页面又打开 A 页面时，两边的 `returnPage` 很容易互相指向：

```text
A.returnPage = B
B.returnPage = A
```

此时点击返回并不是弹出历史，而是继续把 `activePage` 设置为对方页面，最终形成 A / B 之间的循环跳转。

之前的歌手页链式跳转修复只解决了“歌手 → 歌手”的同类 ID 回退，但没有覆盖“歌单 / 专辑 / 歌手 / 用户 / 语言 / 评论 / MV / 播客”等跨详情页互相打开的场景。

## 3. 修复策略

本次修复将详情页返回机制调整为“真实访问快照栈优先，旧 `returnPage` 兜底”。

新增 `NavigationSnapshot`，在进入详情类页面前保存当前页面关键状态：

- 当前页面类型：`activePage`
- 当前详情 ID：`activePlaylistId`、`activeAlbumId`、`activeArtistId`、`activeUserId` 等
- 歌手页内部状态：`artistActiveTabState`、`activeArtistIdStack`
- 各类旧 `returnPage` 值
- MV、语言、每日推荐注入数据等必要上下文

核心函数：

- `createNavigationSnapshot()`：生成当前页面快照
- `pushNavigationSnapshot()`：进入新页面前压栈
- `restoreNavigationSnapshot()`：恢复指定快照
- `restorePreviousPage()`：返回时弹出上一个快照
- `clearNavigationHistory()`：进入顶级页面时清空历史

返回函数统一改为：

```ts
if (restorePreviousPage()) return;
// 没有历史快照时，继续使用原 returnPage 逻辑兜底
```

这样即使出现 A → B → A → B，也会按真实栈顺序逐层回退，而不是读取两个页面互相污染的 `returnPage`。

## 5. 保留旧 `returnPage` 的原因

本次没有删除旧的 `returnPage` 体系，原因是：

1. 返回按钮文案仍依赖这些值，例如“返回歌单详情”“返回搜索结果”。
2. 某些入口可能不经过新快照栈，旧逻辑可作为兜底。
3. 这是最小风险修复，避免一次性重构全部导航状态。

当前规则是：

```text
真实访问快照栈 > 旧 returnPage 兜底
```

## 6. 后续开发约束

新增详情页、评论页、播放页、聚合页时，必须判断它是否属于“可返回的二级/三级页面”。

如果是，应遵循：

1. 打开页面前调用 `pushNavigationSnapshot()`。
2. 返回函数开头优先调用 `restorePreviousPage()`。
3. 侧边栏、搜索、设置、用户中心等顶级入口调用 `clearNavigationHistory()`。
4. `returnPage` 只用于 label 和兜底，不再作为主要历史机制。

禁止新增只写固定页面的返回逻辑，例如：

```ts
activePage.value = 'playlist-detail';
```

除非该逻辑前面已经优先尝试 `restorePreviousPage()`，并且固定页面只是兜底。

## 7. 复核清单

完成导航相关改动后，至少验证以下链路：

- 歌单详情 → 用户详情 → 歌单详情 → 连续返回
- 歌单详情 → 专辑详情 → 歌手详情 → 专辑详情 → 连续返回
- 搜索页 → 歌手详情 → 专辑详情 → 评论页 → 连续返回
- 播客列表 → 播客分类 → 播客详情 → 连续返回
- MV 播放页 → 用户详情 → 连续返回
- 顶级侧边栏切换后，再进入详情页并返回，不能回到旧详情链路

同时确认：

- 页面切换后 `.content` 滚动位置归零
- 返回按钮仍可见
- sticky 头部不会卡死
- `npm run build:web` 通过
- `src/App.vue` 无 lint 报错
