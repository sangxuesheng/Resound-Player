# 桌面端窗口控制按钮检测方案

## 背景

Electron 桌面端需要在 TopBar 右上角显示自定义窗口控制按钮（最小化、最大化/还原、关闭），而 Web 端不显示。窗口控制按钮的显隐控制（桌面端显示 / Web 端隐藏）需要可靠的平台检测，避免按钮在 Web 端误显示或桌面端启动初期不渲染。

## 问题

`platform.isDesktop` 在 Electron 渲染进程中可能返回 `false` 的原因：

| 原因 | 说明 |
|------|------|
| contextBridge 时机 | `contextBridge.exposeInMainWorld` 的执行时机和 Vue 组件挂载的时序不确定，`window.appEnv` 在第一次检查时可能尚未就绪 |
| 条件覆盖 | 原有实现只检查 `window.appEnv?.isDesktop`，无兜底 |

导致：`v-if="platform.isDesktop"` 在渲染进程启动初期为 `false`，窗口控制按钮不渲染。

## 改动方案

### 1. `electron/preload.js` — DOM class 标记

在 `DOMContentLoaded` 事件中给 `<html>` 添加 `resound-desktop` class：

```js
// electron/preload.js — 末尾
document.addEventListener('DOMContentLoaded', () => {
  document.documentElement.classList.add('resound-desktop');
});
```

该 class 可在 CSS / 选择器中用于平台专属样式判断，与 `platform.ts` 解耦。

此标记的优势：
- 不依赖 contextBridge 的 `isDesktop` 属性
- 在 DOM 解析完成后立即生效，早于 Vue 组件挂载
- 可供 CSS 全局选择器使用（如 `html:not(.resound-desktop)` 规则）

### 2. `src/utils/platform.ts` — User-Agent 兜底

在 `isDesktop` getter 中增加 User-Agent 兜底检测：

```ts
get isDesktop(): boolean {
  if (typeof window === 'undefined') return false
  // 优先通过 preload contextBridge 检测
  if (!!(window as any).appEnv?.isDesktop) return true
  // 兜底：User-Agent 检测（不受 contextBridge 影响）
  return typeof navigator !== 'undefined' && navigator.userAgent.includes('Electron')
}
```

检测优先级：
1. `window.appEnv?.isDesktop` — contextBridge 暴露（最快，但时机不可靠）
2. `navigator.userAgent.includes('Electron')` — User-Agent 兜底（一定可靠）

## 相关文件与职责

| 文件 | 角色 | 关键代码 |
|------|------|---------|
| `electron/preload.js` | 桌面端标记 + 最大化状态同步 | `document.documentElement.classList.add('resound-desktop')` + `ipcRenderer.on('win-state-change')` → `data-win-maximized` |
| `src/utils/platform.ts` | 平台检测统一入口 | `isDesktop` getter（contextBridge → UA 兜底） |
| `src/components/TopBar.vue` | 按钮渲染 + 窗口控制 | `v-if="platform.isDesktop"` 控制显隐，`document.title = 'cmd:xxx'` 触发窗口操作 |
| `electron/main.js` | 窗口创建 + page-title-updated 监听 + 最大化状态广播 | `page-title-updated` 事件处理 + `win.on('maximize/unmaximize')` 广播 |

## 数据流

```
用户点击自定义按钮
  → document.title = 'cmd:maximize:1234567890'
    → page-title-updated 事件 → event.preventDefault()
      → BrowserWindow.maximize()
        → win.on('maximize')
          → webContents.send('win-state-change', true)
            → preload: ipcRenderer.on('win-state-change')
              → dataset.winMaximized = ''
                → TopBar.vue: MutationObserver
                  → isMaximized.value = true
                    → 按钮图标切换为「还原」
```

## 按钮显隐判定链路

```
检查渲染进程是否在 Electron 中运行
  → contextBridge 暴露的 window.appEnv?.isDesktop ?
    → true: 标记为桌面端
    → false/undefined: User-Agent 包含 'Electron' ?
      → true: 标记为桌面端
      → false: Web 端，隐藏按钮
```

## 开发规范

1. **禁止**在业务代码中直接检查 `window.appEnv`
2. 所有平台检测走 `src/utils/platform.ts`
3. 新增 preload 字段时同步更新 `global.d.ts` 中的 `Window.appEnv` 接口
4. CSS 层面的平台区分优先使用 `resound-desktop` class 选择器
