# 导航与吸顶状态修复说明

本文档记录本次导航返回逻辑与吸顶（sticky）状态相关的 Bug 及修复方案。

---

## 1. 返回导航 Label 缺失

### 问题

`backToAlbum` / `backToArtist` / `backToPlaylist` / `backToUserDetail` 等返回函数的 label 和分发逻辑遗漏了部分来源页面，导致：

- 从歌单详情页进入专辑详情 → 返回按钮显示「返回首页」且跳到首页
- 从用户详情页进入歌单详情 → 返回按钮显示「返回歌单分类」
- 从语言详情页进入歌单详情 → 同上
- 从歌单/歌手/专辑详情页进入用户详情 → 返回按钮显示「返回搜索结果」

### 修复

**文件**: `src/App.vue`

所有 `*BackLabel` computed 和 `backTo*` 函数补充缺失的 `returnPage` 分支：

| 函数 | 补充分支 |
|------|----------|
| `backToAlbum` | `playlist-detail` |
| `albumBackLabel` | `playlist-detail` → "返回歌单详情" |
| `playlistBackLabel` | `user-detail` → "返回用户详情", `language-detail` → "返回语言详情" |
| `userBackLabel` | `playlist-detail`, `artist-detail`, `album-detail` |
| `HomePanel` 调用 | `openPlaylistDetail` 补传 `'home'` |

## 2. 多层歌手页面链式跳转

### 问题

歌手页内点击歌手链接（歌手 A → 歌手 B → 歌手 C）时，`openArtistDetail` 无条件覆盖 `activeArtistReturnPage` 为 `'artist-detail'`，导致：

- 原始来源页面丢失（如 `'playlist-detail'`）
- `backToArtist()` 看到 returnPage 为 `'artist-detail'`，将 `activePage` 设为 `'artist-detail'`（与当前相同），**页面无变化，点击返回无效**

### 修复

**文件**: `src/App.vue`

1. 新增 `activeArtistIdStack = ref<number[]>([])` 保存歌手链式跳转历史
2. 修改 `openArtistDetail`：当从歌手页跳转到另一个歌手时，**压栈保存当前歌手ID**，且**不覆盖**原始 `activeArtistReturnPage`
3. 修改 `backToArtist`：栈非空时优先**弹栈恢复上一个歌手ID**（留在歌手页，通过 prop 变更驱动重新加载）
4. 修改 `artistBackLabel`：栈非空时显示「返回上一页」，栈空后恢复原始来源标签

### 行为

```
Playlist → 歌手A → 歌手B → 歌手C
                ← 返回上一页(歌手B)
           ← 返回上一页(歌手A)
      ← 返回歌单详情 ✓
```

## 3. 页面切换时滚动位置 / 吸顶（sticky）态卡死

### 问题

所有详情页共用同一个 `.content` 滚动容器（`App.vue` 中的 `<main>`），页面切换时：

- `scrollTop` 从未归零 → 新页面从旧页面的滚动位置开始显示
- `useDetailStickyState` 的 `isSticky` 保持 `true` → 返回按钮被 `back-fade` 隐藏、头部卡在粘顶态无法切回

触发场景：
- 跨页面跳转（歌单 → 歌手、专辑 → 歌手等）
- 同类页面跳转（歌手 A → 歌手 B）

### 修复

#### 3.1 页面切换时重置滚动位置

**文件**: `src/App.vue`

新增 `scrollContentToTop()` 函数 + 两个 watcher：

```ts
watch(activePage, () => { scrollContentToTop(); });
watch([activeArtistId, activeAlbumId, activePlaylistId, activeUserId, activeLanguage, activeRankId], 
     () => { scrollContentToTop(); });
```

#### 3.2 `refresh()` 中使用 `nextTick` 替代嵌套 RAF

**文件**: `src/composables/useDetailStickyState.ts`

`refresh()` 原本使用嵌套 `requestAnimationFrame` 等待 DOM 更新，但 RAF 不能可靠保证 Vue DOM flush 完成。改用 `nextTick`（Vue 内部使用 microtask，在 DOM 更新完成后回调）：

```ts
function refresh(): void {
  isSticky.value = false;
  nextTick(() => {
    headerEl = getHeaderEl();
    scrollHost = getScrollHost();
    if (headerEl && scrollHost) {
      const hostRect = scrollHost.getBoundingClientRect();
      const headerRect = headerEl.getBoundingClientRect();
      initialOffset = Math.max(1, headerRect.top - hostRect.top);
    }
    update();
  });
}
```

关键点：
- `nextTick` 在 Vue flush DOM 后立即执行，此时 `is-sticky-header` class 已移除
- `Math.max(1, ...)` 安全兜底，防止 `initialOffset = 0` 导致 `update()` 重新锁死 `isSticky`

#### 3.3 `initialOffset` 安全兜底

在 `Math.max()` 中最小值为 `1` 而非 `0`，确保即使异常场景下 `initialOffset` 也不会为 0，避免 `update()` 在 `scrollTop = 0` 时将 `isSticky` 重新设为 `true`。
