# 吸顶动画流畅度优化方案

> 本方案中的优化措施已全部实施。当前实现：
> - **height transition**：默认 60ms（滚动中），停止后升级 350ms
> - **RAF delta 过滤**：`--sticky-progress` 变化量 < 0.005 时跳过写入
> - **will-change**：已在 `playlist-detail-header-wrap` 上设置 `will-change: height, opacity`


## 优化方案

### 1. 恢复 `height` CSS transition（关键）

已实现。`.playlist-detail-header-wrap` 上恢复 `height` 的 CSS transition：

```css
.playlist-detail-header-wrap {
  will-change: height, opacity;
  transition:
    height 0.06s cubic-bezier(0.33, 0, 0.1, 1),
    border-radius 0.3s cubic-bezier(0.33, 0, 0.1, 1),
    box-shadow 0.3s cubic-bezier(0.33, 0, 0.1, 1);
}
```

> 实际滚动中为 60ms 极短时长消除跳动，停止滚动 200ms 后升级为 350ms 平滑过渡（详见吸顶栏实现说明第十四节）。

### 2. RAF 防冗余写入

已实现。`useDetailStickyState.ts` 中，`update()` 通过 RAF 节流 + delta 检查，当 `--sticky-progress` 变化量 < 0.005 时跳过写入：

```typescript
const delta = Math.abs(progress - lastProgress);
if (!force && delta < 0.005) return;
lastProgress = progress;
host.style.setProperty('--sticky-progress', String(progress));
```

### 3. `will-change` 提示

已实现。为以下元素添加 `will-change`：

| 元素 | 属性 | 原因 | 状态 |
|------|------|------|------|
| `.playlist-detail-header-wrap` | `height, opacity` | 吸顶态高度变化 + 背景透明度动画 | ✅ 已设置 |
| `.playlist-detail-header-wrap` | `box-shadow` | 吸顶态阴影出现 | —（由 transition 处理） |
| `.cover-motion-shell` | `transform` | 封面缩放 + 上移动画 | —（由 transition 处理） |
| `img.cover` | `transform` | 封面 hover 缩放 | ✅ 已有 `will-change: transform` |

## 预期效果

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| `--sticky-progress` 写入频率 | 每帧（60次/秒） | 平均 20-30 次/秒（delta 过滤） |
| height 过渡 | 瞬间硬切 | 60ms（滚动中）/ 350ms（停止后）平滑 |
| GPU 合成层 | 透传合成层 | 独立合成层（`will-change`） |
| 滚动时 JS 开销 | 中（RAF + progress 计算 + 3 次 style 写入 + blur sync） | 低（delta 过滤后大部分帧跳过） |
