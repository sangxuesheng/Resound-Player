# 吸顶动画流畅度优化方案


## 优化方案

### 1. 恢复 `height` CSS transition（关键）

在 `.playlist-detail-header-wrap` 上恢复 `height` 的 CSS transition：

```css
.playlist-detail-header-wrap {
  will-change: height, opacity;
  transition:
    height 0.3s cubic-bezier(0.33, 0, 0.1, 1),
    border-radius 0.3s cubic-bezier(0.33, 0, 0.1, 1),
    box-shadow 0.3s cubic-bezier(0.33, 0, 0.1, 1);
}
```

**为什么这次可以加回来**：
- `will-change: height` 提示浏览器预先创建合成层，将 height 变化提升到合成器线程处理
- `transform: translateZ(0)` 已存在，与 `will-change` 配合将整个 header 提升到独立合成层
- 布局重排（Layout）的影响范围被限制在 header 自身（`overflow: hidden` 裁剪内部溢出）

### 2. RAF 防冗余写入

`useDetailStickyState.ts` 中，`update()` 已通过 RAF 节流。再加一个 delta 检查，当 `--sticky-progress` 变化量 < 0.01 时跳过写入：

```typescript
function update(): void {
  if (!scrollHost || !headerEl) return;
  const st = scrollHost.scrollTop;
  // ...计算 progress...
  
  // 跳过写入：变化太小视觉无感知，减少 inline style 写入
  const currentStr = headerEl.style.getPropertyValue('--sticky-progress');
  const current = parseFloat(currentStr) || 0;
  if (Math.abs(progress - current) < 0.005) return;
  
  headerEl.style.setProperty('--sticky-progress', String(progress));
}
```

### 3. `will-change` 提示

为以下元素添加 `will-change`：

| 元素 | 属性 | 原因 |
|------|------|------|
| `.playlist-detail-header-wrap` | `height, opacity` | 吸顶态高度变化 + 背景透明度动画 |
| `.playlist-detail-header-wrap.is-sticky-header` | `box-shadow` | 吸顶态阴影出现 |
| `.cover-motion-shell` | `transform` | 封面缩放 + 上移动画 |
| `img.cover` | 已有 `will-change: transform` | 封面 hover 缩放 |
| `:deep(.hero-meta-shell)` | `opacity, transform` | 元数据淡出上移 |

## 预期效果

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| `--sticky-progress` 写入频率 | 每帧（60次/秒） | 平均 20-30 次/秒（delta 过滤） |
| height 过渡 | 瞬间硬切 | 0.3s 平滑 cubic-bezier |
| GPU 合成层 | 透传合成层 | 独立合成层（`will-change`） |
| 滚动时 JS 开销 | 中（RAF + progress 计算 + 3 次 style 写入 + blur sync） | 低（delta 过滤后大部分帧跳过） |
