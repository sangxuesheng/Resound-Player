# 吸顶动画流畅度优化方案

> 本方案中的优化措施已全部实施。当前实现：
> - **layout transition**：已全部移除（height / padding 等均无 transition），所有属性同时变化消除 temporal mismatch
> - **Scroll anchoring**：`.playlist-detail-page` 设置 `overflow-anchor: none`，切断高度变化导致的 scrollTop 反馈振荡
> - **RAF delta 过滤**：`--sticky-progress` 变化量 < 0.005 时跳过写入
> - **will-change**：仅保留 `will-change: opacity`（header 背景透明度）


## 优化方案

### 1. 消除 Layout CSS transition（关键）

已实施。**不再使用 layout transition**（height、padding 等），所有依赖 `--sticky-progress` 的属性在同一帧同时变化，避免 temporal mismatch：

```css
.playlist-detail-header-wrap {
  transition:
    border-radius 0.3s cubic-bezier(0.33, 0, 0.1, 1),
    box-shadow 0.3s cubic-bezier(0.33, 0, 0.1, 1);
}
```

> 曾尝试 60ms + 350ms 双层 height transition，均因与 grid 列宽、封面 transform 等无 transition 属性的时间差导致抖动。最终结论：**任何布局型 transition 都会与无 transition 属性产生不可消除的错位，唯一正解是全部移除**。

### 2. Scroll anchoring 防护

已实施。当头部高度从 380px 收缩到 80px 时，滚动容器总内容高度减少 ~300px。浏览器 scroll anchoring 会调整 `scrollTop` 保持可见内容稳定，调整后的 scrollTop 重新计算 `--sticky-progress` → **反馈振荡**（layout → scroll 循环）：

```css
.playlist-detail-page {
  overflow-anchor: none;
}
```

### 3. RAF 防冗余写入

已实现。`useDetailStickyState.ts` 中，`update()` 通过 RAF 节流 + delta 检查，当 `--sticky-progress` 变化量 < 0.005 时跳过写入：

```typescript
const delta = Math.abs(progress - lastProgress);
if (!force && delta < 0.005) return;
lastProgress = progress;
host.style.setProperty('--sticky-progress', String(progress));
```

### 3. `will-change` 提示

已实施。为以下元素添加 `will-change`：

| 元素 | 属性 | 原因 | 状态 |
|------|------|------|------|
| `.playlist-detail-header-wrap` | `opacity` | header 背景透明度动画 | ✅ 已设置 |
| `img.cover` | `transform` | 封面 hover 缩放 | ✅ 已有 `will-change: transform` |

> `will-change: height` 已移除，因为不再使用 height transition，且 `will-change: height` 与 `position: sticky` 结合可能触发浏览器合成层异常。

## 预期效果

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| `--sticky-progress` 写入频率 | 每帧（60次/秒） | 平均 20-30 次/秒（delta 过滤） |
| height 过渡 | 瞬间硬切 | 无 transition，所有属性同时变化消除时间差 |
| Scroll anchoring 反馈 | 无防护 | `overflow-anchor: none` 切断回路 |
| 滚动时 JS 开销 | 中（RAF + progress 计算 + 3 次 style 写入 + blur sync） | 低（delta 过滤后大部分帧跳过） |
