# 详情页标签栏与评论区复用说明

本文档记录 `PlaylistDetailPage.vue`、`AlbumDetailPage.vue` 和 `PodcastDetailPage.vue` 中已落地的详情页标签栏和评论区实现方式，供后续排行榜等其他详情页复用。

---

## 1. 适用场景

当详情页需要在同一个头部下切换多类内容时，优先采用本模式：

- 详情页主内容：歌曲列表、节目列表、专辑列表等
- 评论区：复用 `CommentPanel`
- 未来扩展：简介、相关推荐、动态、收藏用户等

核心目标：

- 标签栏放在统一详情页头部中，不破坏详情页 shell 结构
- tab 内容使用 `v-if / v-else-if` 互斥分支，避免多个面板同时挂载
- 切换 tab 或切换资源 ID 时强制重建内容实例，避免评论分页、输入框、内部加载状态串页
- 评论能力统一走 `CommentPanel`，只在外层注入资源类型和 API 适配函数

---

## 2. 标签栏接入位置

标签栏已提取为公共组件 `src/components/ui/DetailTabBar.vue`，`PlaylistDetailPage.vue`、`AlbumDetailPage.vue` 和 `PodcastDetailPage.vue` 均通过 `DetailStickyHeroHeader` 的 `#tabs` 插槽接入。

歌单页示例：

```vue
<template #tabs>
  <DetailTabBar
    v-model="activeTab"
    :tabs="tabs"
    aria-label="歌单详情标签"
    v-model:search-query="searchQuery"
    :show-search="activeTab === 'songs'"
  />
</template>
```

专辑详情页的标签栏与歌单页一致，仅 `aria-label` 改为 `"专辑详情标签"`。

播客详情页的标签栏类似，tab 切换为"声音"和"评论"，`aria-label` 改为 `"播客详情标签"`，搜索框仅在声音 tab 显示：

```vue
<template #tabs>
  <DetailTabBar
    v-model="activeTab"
    :tabs="tabs"
    aria-label="播客详情标签"
    v-model:search-query="searchQuery"
    :show-search="activeTab === 'episodes'"
  />
</template>
```

组件源码位于 `src/components/ui/DetailTabBar.vue`，职责：

- 渲染 tab 按钮 + 右侧搜索框（`margin-left: auto` 靠右）
- 通过 `v-model` 对外暴露 `activeTab`
- 通过 `v-model:search-query` 对外暴露搜索关键词
- 搜索框只在 `showSearch = true` 时显示，内含清空按钮，支持 `Esc` 一键清空

脚本侧维护最小状态：

```ts
const activeTab = ref<'songs' | 'comments'>('songs');
const tabs = [
  { key: 'songs', label: '歌曲' },
  { key: 'comments', label: '评论' },
] as const;
const searchQuery = ref('');
```

复用要求：

- 标签栏应放入详情页头部的 tabs 插槽，避免在内容区重复实现头部结构
- 直接使用 `<DetailTabBar>` 组件，不再在页面内重复散写 tab 按钮模板
- `role="tablist"` 和明确的 `aria-label` 已由组件内置

## 3. 内容区必须使用互斥分支

tab 内容区不要使用 `v-show`，也不要让多个重型组件同时挂载。推荐使用相邻的 `<template v-if>` / `<template v-else-if>`：

```vue
<template v-if="activeTab === 'songs'">
  <AnimatedAppear
    :key="`${tabContentKey}:songs`"
    tag="ul"
    variant="content"
    rhythm="list"
    class-name="song-list"
  >
    <!-- 歌曲列表 -->
  </AnimatedAppear>
</template>

<template v-else-if="activeTab === 'comments'">
  <div :key="`${tabContentKey}:comments`" class="playlist-comment-section">
    <!-- 评论区 -->
  </div>
</template>
```

注意事项：

- `v-else-if` 必须紧邻同级 `v-if`，中间不能插入普通元素、注释或被编译器视为非相邻的自定义组件分支
- 为避免 Vue 编译器对自定义组件相邻分支识别不稳定，推荐把互斥条件放在 `<template>` 上
- 动态 `key` 放在实际内容根节点上，用于强制重建分支实例
- 新增第三个 tab 时继续追加相邻的 `<template v-else-if="activeTab === 'xxx'">`

---

## 4. 强制重建 key 规则

歌单详情页使用 `tabContentKey` 将资源 ID 和当前 tab 绑定：

```ts
const tabContentKey = computed(() => `${playlist.value?.id || props.playlistId || 'pending'}:${activeTab.value}`);
```

再在内容根节点上拼接分支名：

```vue
<AnimatedAppear :key="`${tabContentKey}:songs`" ...>
<div :key="`${tabContentKey}:comments`" ...>
```

这样可以覆盖两类重建场景：

1. **切换 tab**：`activeTab` 改变，当前分支卸载，目标分支重新挂载
2. **切换资源**：`playlist.id / props.playlistId` 改变，同一 tab 下的内容也重新挂载

复用到其他详情页时，将资源 ID 替换为对应主 ID：

```ts
const tabContentKey = computed(() => `${artist.value?.id || props.artistId || 'pending'}:${activeTab.value}`);
```

或：

```ts
const tabContentKey = computed(() => `${album.value?.id || props.albumId || 'pending'}:${activeTab.value}`);
```

播客详情页使用电台 ID：

```ts
const tabContentKey = computed(() => `${podcastRid.value || 'pending'}:${activeTab.value}`);
```

## 5. 评论区接入方式

歌单与专辑的评论区均复用 `CommentPanel`，外层只负责注入资源 ID、资源类型和 API 适配函数。

歌单页示例：

```vue
<CommentPanel
  :resource-id="Number(playlist.id || props.playlistId || 0)"
  :resource-type="2"
  :fetcher="commentApi.getPlaylistComments"
  :sender="(params) => commentApi.sendComment({ ...params, type: 2 })"
  :liker="(params) => commentApi.likeComment({ ...params, type: 2 })"
  :deleter="commentApi.deletePlaylistComment"
  @open-user="(uid) => emit('open-user', uid)"
/>
```

专辑页示例：

```vue
<CommentPanel
  :resource-id="Number(album.id || props.albumId || 0)"
  :resource-type="3"
  :fetcher="commentApi.getAlbumComments"
  :sender="(params) => commentApi.sendComment({ ...params, type: 3 })"
  :liker="(params) => commentApi.likeComment({ ...params, type: 3 })"
  :deleter="commentApi.deleteAlbumComment"
  @open-user="(uid) => emit('open-user', uid)"
/>
```

播客页（电台频道级别）示例：

```vue
<CommentPanel
  :resource-id="Number(podcastRid || 0)"
  :resource-type="7"
  :fetcher="commentApi.getRadioComments"
  :sender="(params) => commentApi.sendComment({ ...params, type: 7 })"
  :liker="(params) => commentApi.likeComment({ ...params, type: 7 })"
  :deleter="commentApi.deleteDjComment"
  @open-user="(uid) => emit('open-user', uid)"
/>
```

> **注意**：播客评论分为两级：
> - **电台频道级别**（type: `7`，threadId 前缀 `A_DR_14_`）：使用 `getRadioComments`，传入电台 ID（`podcastRid`）
> - **节目/单集级别**（type: `4`，threadId 前缀 `A_DJ_1_`）：使用 `getDjComments`，传入节目 ID

关键约定：

- `resource-id` 必须使用当前详情页真实资源 ID，并提供 prop 兜底
- `resource-type` 必须与接口类型一致
- `sendComment` / `likeComment` 这类通用接口要在外层补齐 `type`
- 删除接口如果已有资源专用封装，直接传入专用函数
- `@open-user` 继续向页面外层派发，不在评论面板内部耦合导航逻辑

## 6. 常见资源类型参考

复用时按接口实际定义确认类型值。当前已落地场景：

| 场景 | resource-type | fetcher 示例 | 说明 |
|------|---------------|--------------|------|
| 歌单评论 | `2` | `getPlaylistComments` | `PlaylistDetailPage` 已落地 |
| 专辑评论 | `3` | `getAlbumComments` | `AlbumDetailPage` 已落地 |
| 电台/播客评论（频道级别） | `7` | `getRadioComments` | `PodcastDetailPage` 已落地（threadId `A_DR_14_`） |
| 电台/播客评论（节目级别） | `4` | `getDjComments` | 用于单集节目（threadId `A_DJ_1_`） |
| 歌曲评论 | 按接口定义 | `getMusicComments` / 专用封装 | 需确认现有 API 名称 |
| MV 评论 | 按接口定义 | `getMvComments` | 复用同一模式 |
| 电台/播客评论 | 按接口定义 | 专用评论接口 | 需确认后端支持 |

不要在 `CommentPanel` 内硬编码资源类型。资源类型属于业务详情页上下文，应由调用方注入。

## 7. 复用检查清单

新增详情页 tab + 评论区时，至少确认以下事项：

- 标签栏放在统一详情页头部插槽中，而不是另起一套 header
- 直接使用 `<DetailTabBar>` 组件，不再重复散写 tab 按钮模板
- tab 状态使用字面量联合类型，避免任意字符串
- 如果页内有歌曲列表需要搜索过滤，新增 `searchQuery` ref + `filteredTracks`/`filteredSongs` 计算属性，传入 `<DetailTabBar>` 的 `v-model:search-query` 和 `:show-search="activeTab === 'songs'"`
- 内容区使用相邻 `<template v-if>` / `<template v-else-if>`
- 内容根节点带动态 `key`，且 key 包含资源 ID 和 `activeTab`
- 评论区复用 `CommentPanel`，不复制评论列表、发送、点赞、删除逻辑
- `@open-user` 等导航事件继续向外层 emit
- 完成后运行 lint / build，确认 Vue 模板编译通过

## 8. 已知坑位

### 8.1 `v-else-if` 相邻性

不要写成下面这种依赖自定义组件相邻识别的结构：

```vue
<AnimatedAppear v-if="activeTab === 'songs'" ...>
  ...
</AnimatedAppear>
<div v-else-if="activeTab === 'comments'">
  ...
</div>
```

在当前项目的 Vite + Vue 编译链中，这种写法曾触发：

```text
v-else/v-else-if has no adjacent v-if or v-else-if
```

更稳妥的写法是：

```vue
<template v-if="activeTab === 'songs'">
  ...
</template>
<template v-else-if="activeTab === 'comments'">
  ...
</template>
```

### 8.2 评论区状态串页

如果评论区不带动态 key，可能出现：

- 切换歌单后仍保留旧评论分页状态
- 输入框、回复目标、加载状态没有重置
- 同一组件实例复用导致请求时序更难排查

因此评论区根节点必须带资源 ID 相关 key。

### 8.3 不要用 `v-show` 隐藏重型面板

评论区包含列表请求、分页、输入、点赞、删除等内部状态。使用 `v-show` 会让隐藏面板继续保留实例状态，不适合作为详情页多 tab 的默认方案。
