# 播客详情页列表设计说明

## 设计决策

播客详情页的程序节目列表采用 **倒序显示 + 自然序号** 的展示方式。

## 具体行为

- **列表显示顺序**：最新发布的节目排在最上方，越旧的节目排在越下方
- **序号规则**：从 1 开始递增（`idx + 1`），作为每条节目的位置标识

## 核心代码

```ts
// src/components/PodcastDetailPage.vue

// 源数组保持不变，展示时反转
const displayedRawItems = computed(() => [...props.items].reverse());

// normalizedItems 中：
const normalizedItems = computed(() => displayedRawItems.value.map((item, idx) => {
  const originalIndex = props.items.length - idx - 1; // 原始数组中的位置
  return {
    // ...
    index: idx + 1,       // 显示序号（1-based，对应当前倒序列表中的位置）
    originalIndex,        // 原始数组中的索引，用于 play-item 事件
  };
}));
```

## 设计意图

| 特性 | 目的 |
|---|---|
| **倒序显示（最新在前）** | 播客/有声书场景下用户最关心最新发布的内容，优先展示最近更新的节目 |
| **序号正序（1, 2, 3...）** | 为用户提供清晰的位置感，方便交流时引用节目位置，也符合常规列表的阅读习惯 |

## 播放行为

- 双击某条节目或点击 `PlayPauseButton` 触发 `play-item` 事件时，传递的 `index` 为 **`originalIndex`**（原始数组中的索引），确保播放器按源数据顺序正确播放后续节目

## 适用范围

适用于：

- 播客详情页（`PodcastDetailPage.vue`）的程序节目列表
- 任何按时间倒序展示内容、但需要维持自然序号的列表场景
