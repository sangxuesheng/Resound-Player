# 桌面端本地歌曲功能开发计划（深化版）

> **重要：本地歌曲功能仅限桌面端（Electron）。Web 端不提供本地歌曲功能。**
> 所有本地音乐相关的代码路径必须做 `platform.isDesktop` 守卫，Web 端应优雅降级。


---

## 一、桌面端专属原则

### 1.1 平台隔离策略

```
┌──────────────────────────────────────────────────────┐
│                 渲染进程 (Renderer)                     │
│                                                       │
│  Sidebar / 路由 / Store / 页面组件                      │
│       │  ▲  platform.isDesktop ? 显示 : 隐藏           │
│       │  ▲  localApi 存在 ? 调用 : 跳过                │
│       ▼  │                                             │
│  ┌──────────────────┐    ┌──────────────────┐          │
│  │  Desktop Only     │    │  Web Fallback    │          │
│  │  - localApi IPC   │    │  - 隐藏本地入口   │          │
│  │  - local:// 播放   │    │  - 空 store      │          │
│  │  - 扫描/管理/编辑  │    │  - 路由重定向     │          │
│  └──────────────────┘    └──────────────────┘          │
└──────────────┬───────────────────────────────────────┘
               │ IPC (仅 Electron)
               ▼
┌──────────────────────────────────────────────────────┐
│                 主进程 (Main Process)                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐    │
│  │ SQLite DB │  │ Scanner  │  │ local:// Protocol │    │
│  │ (better-  │  │ (music-  │  │ (registerFile-   │    │
│  │  sqlite3) │  │ metadata)│  │  Protocol)        │    │
│  └──────────┘  └──────────┘  └──────────────────┘    │
└──────────────────────────────────────────────────────┘
```

### 1.2 守卫模式

所有本地音乐相关操作必须遵循以下守卫链：

```typescript
// 1. 组件层面：v-if 控制显隐
<LocalSongsPage v-if="platform.isDesktop" />

// 2. Store 层面：方法级守卫
async scanAll() {
  if (!platform.isDesktop) return  // 静默跳过
  if (!window.localApi) return     // preload 未暴露
  // ...实际扫描逻辑
}

// 3. 播放层面：source 标记 + 协议守卫
if (track.source === 'local') {
  if (!platform.isDesktop) {
    showToast('本地歌曲仅支持桌面端播放', 'warning')
    return
  }
  playUrl = `local:///${track.path}`
}

// 4. 路由层面：桌面端才注册本地路由
const localRoutes = platform.isDesktop ? [
  { path: '/local/songs', component: LocalSongsPage },
  { path: '/local/artists', component: LocalArtistsPage },
  { path: '/local/albums', component: LocalAlbumsPage },
] : []
```

### 1.3 Web 端降级清单

| 模块 | Web 端行为 |
|---|---|
| 侧栏「本地音乐」入口 | `v-if="platform.isDesktop"` 隐藏 |
| 本地音乐路由 | 不注册，访问重定向到首页 |
| `localMusicStore` | 实例化但不初始化，所有方法空返回 |
| `player.ts` local 源 | 遇到 `source === 'local'` 跳过并提示 |
| `useLyrics.ts` 本地歌词 | `if (!isDesktop) return` |
| `localApi` 调用 | 所有 `window.localApi?.xxx()` 可选链调用 |

## 四、Phase 1 深化：基础设施（Electron 主进程）

### 4.1 依赖安装与 Native Rebuild 风险

```bash
npm install music-metadata better-sqlite3
npx @electron/rebuild
```

| 风险点 | 影响 | 应对 |
|---|---|---|
| `better-sqlite3` 在 Electron 下需要 native rebuild | 启动崩溃 | `@electron/rebuild` 必须执行，CI 中集成 |
| ARM Mac / Linux / Windows 交叉编译 | 安装失败 | 使用 prebuildify 预编译或 `@electron/rebuild` 平台矩阵 |
| `music-metadata` 解析超大文件（>500MB）| 内存溢出 | 流式解析 + 单文件超时 30s |
| `better-sqlite3` WAL 模式文件锁 | 并发写冲突 | 使用单例 DB 连接 + 串行化写操作 |

### 4.2 SQLite 数据库层

新建 `electron/services/db/LocalMusicDB.ts`

**Bug 预防：**

1. **WAL 模式必须启用**：`PRAGMA journal_mode=WAL` — 防多进程读写锁
2. **同步写必须串行化**：所有写操作通过队列保证顺序
3. **路径规范化**：Windows 统一 `path.normalize()`，macOS 统一 `path.normalize()`
4. **内存控制**：大结果集使用游标分页，避免一次性加载全部到内存

```typescript
import Database from 'better-sqlite3'
import path from 'path'
import { app } from 'electron'

const DB_PATH = path.join(app.getPath('userData'), 'local-music.db')

class LocalMusicDB implements IMetadataDB {
  private db: Database.Database
  private writeQueue: Promise<void> = Promise.resolve()

  async init() {
    this.db = new Database(DB_PATH)
    this.db.pragma('journal_mode = WAL')
    this.db.pragma('foreign_keys = ON')
    this.createTables()
  }

  private createTables() {
    this.db.exec(`
      CREATE TABLE IF NOT EXISTS tracks (
        id          TEXT PRIMARY KEY,
        path        TEXT UNIQUE NOT NULL,
        title       TEXT NOT NULL DEFAULT '',
        artist      TEXT DEFAULT '',
        album       TEXT DEFAULT '',
        albumArtist TEXT DEFAULT '',
        duration    REAL DEFAULT 0,
        bitrate     INTEGER DEFAULT 0,
        sampleRate  INTEGER DEFAULT 0,
        trackNo     INTEGER DEFAULT 0,
        discNo      INTEGER DEFAULT 0,
        genre       TEXT DEFAULT '',
        year        INTEGER DEFAULT 0,
        coverPath   TEXT DEFAULT '',
        fileSize    INTEGER DEFAULT 0,
        mtime       INTEGER DEFAULT 0,
        hasLyrics   INTEGER DEFAULT 0,
        createdAt   TEXT DEFAULT (datetime('now')),
        updatedAt   TEXT DEFAULT (datetime('now'))
      );

      CREATE INDEX IF NOT EXISTS idx_tracks_path ON tracks(path);
      CREATE INDEX IF NOT EXISTS idx_tracks_artist ON tracks(artist);
      CREATE INDEX IF NOT EXISTS idx_tracks_album ON tracks(album);
      -- 全文搜索虚拟表 (Phase 3 时启用)
      -- CREATE VIRTUAL TABLE IF NOT EXISTS tracks_fts USING fts5(title, artist, album, content='tracks', content_rowid='rowid');

      CREATE TABLE IF NOT EXISTS scan_dirs (
        id       INTEGER PRIMARY KEY AUTOINCREMENT,
        path     TEXT UNIQUE NOT NULL,
        label    TEXT DEFAULT '',
        lastScan TEXT
      );

      CREATE TABLE IF NOT EXISTS playlists (
        id          TEXT PRIMARY KEY,
        name        TEXT NOT NULL,
        description TEXT DEFAULT '',
        coverPath   TEXT DEFAULT '',
        createdAt   TEXT DEFAULT (datetime('now'))
      );

      CREATE TABLE IF NOT EXISTS playlist_tracks (
        playlistId TEXT NOT NULL,
        trackId    TEXT NOT NULL,
        sortOrder  INTEGER NOT NULL,
        addedAt    TEXT DEFAULT (datetime('now')),
        PRIMARY KEY (playlistId, trackId),
        FOREIGN KEY (playlistId) REFERENCES playlists(id) ON DELETE CASCADE,
        FOREIGN KEY (trackId) REFERENCES tracks(id) ON DELETE CASCADE
      );
    `)
  }

  /** 串行化写操作 */
  private serializedWrite<T>(fn: () => T): Promise<T> {
    this.writeQueue = this.writeQueue.then(fn, fn)
    return this.writeQueue
  }

  async upsertTracks(tracks: TrackRecord[]) {
    return this.serializedWrite(() => {
      const stmt = this.db.prepare(`
        INSERT INTO tracks (id, path, title, artist, album, albumArtist, duration,
          bitrate, sampleRate, trackNo, discNo, genre, year, coverPath, fileSize, mtime, hasLyrics, updatedAt)
        VALUES (@id, @path, @title, @artist, @album, @albumArtist, @duration,
          @bitrate, @sampleRate, @trackNo, @discNo, @genre, @year, @coverPath, @fileSize, @mtime, @hasLyrics, datetime('now'))
        ON CONFLICT(path) DO UPDATE SET
          title=excluded.title, artist=excluded.artist, album=excluded.album,
          duration=excluded.duration, mtime=excluded.mtime, updatedAt=datetime('now')
      `)
      const insertMany = this.db.transaction((tracks: TrackRecord[]) => {
        for (const t of tracks) stmt.run(t)
      })
      insertMany(tracks)
    })
  }

  async search(query: string): Promise<TrackRecord[]> {
    const like = `%${query}%`
    return this.db.prepare(`
      SELECT * FROM tracks WHERE title LIKE ? OR artist LIKE ? OR album LIKE ?
      ORDER BY title ASC LIMIT 200
    `).all(like, like, like) as TrackRecord[]
  }

  close() { this.db.close() }
}
```

### 4.3 扫描服务

新建 `electron/services/scanner/NodeMusicScanner.ts`

**Bug 预防：**

1. **单文件异常不中断整体扫描**：每个 `parseFile` 包裹 `try/catch`，失败的文件跳过
2. **并发控制**：worker 数 = `Math.min(8, Math.max(2, os.cpus().length - 1))`
3. **超时保护**：单文件解析超过 30s 强行跳过
4. **增量扫描核心逻辑**：对比 `mtime + fileSize` 双重校验
5. **大目录性能**：10000+ 文件扫描时，首次遍历 + mtime 读取可能会卡住主线程 → 使用 `setImmediate` 分片处理

```typescript
import * as mm from 'music-metadata'
import fs from 'fs/promises'
import path from 'path'
import crypto from 'crypto'

const SUPPORTED = new Set([
  '.mp3', '.flac', '.wav', '.ogg', '.m4a', '.aac',
  '.wma', '.ape', '.dsf', '.opus', '.aiff', '.alac'
])

function md5(str: string): string {
  return crypto.createHash('md5').update(str).digest('hex')
}

async function* scanDir(dirPath: string): AsyncIterable<ScanEvent> {
  // 1. 收集文件路径
  const files: string[] = []
  try {
    await collectFiles(dirPath, files)
  } catch (err) {
    yield { type: 'error', path: dirPath, message: `无法读取目录: ${err}` }
    return
  }

  const total = files.length
  let completed = 0
  const batch: TrackRecord[] = []

  // 2. 获取已缓存的 mtime Map（增量扫描）
  // const cachedMtimes = await db.getAllMtimes()  // 从 DB 读取

  // 3. 并发解析
  const concurrency = Math.min(8, Math.max(2, require('os').cpus().length - 1))
  let index = 0

  const worker = async () => {
    while (index < files.length) {
      const filePath = files[index++]
      try {
        const track = await parseFile(filePath)
        if (track) {
          batch.push(track)
          // yield batch 每 50 条
          if (batch.length >= 50) {
            yield { type: 'batch', tracks: batch.splice(0) }
          }
        }
      } catch (err) {
        yield { type: 'error', path: filePath, message: String(err) }
      }
      completed++
      yield { type: 'progress', current: completed, total }
    }
  }

  await Promise.all(Array.from({ length: concurrency }, worker))

  if (batch.length) yield { type: 'batch', tracks: batch }
  yield { type: 'complete', total }
}

async function collectFiles(dirPath: string, result: string[]) {
  const entries = await fs.readdir(dirPath, { withFileTypes: true })
  for (const entry of entries) {
    const fullPath = path.join(dirPath, entry.name)
    if (entry.isDirectory()) {
      await collectFiles(fullPath, result) // 递归
    } else if (entry.isFile()) {
      const ext = path.extname(entry.name).toLowerCase()
      if (SUPPORTED.has(ext)) result.push(fullPath)
    }
  }
}

async function parseFile(filePath: string): Promise<TrackRecord | null> {
  const stat = await fs.stat(filePath)
  let meta: mm.IAudioMetadata
  try {
    meta = await mm.parseFile(filePath, { duration: true })
  } catch {
    // 解析失败 → 用文件名兜底
    return {
      id: md5(filePath),
      path: filePath,
      title: path.basename(filePath, path.extname(filePath)),
      artist: '未知艺术家',
      album: '未知专辑',
      albumArtist: '',
      duration: 0,
      bitrate: 0,
      sampleRate: 0,
      trackNo: 0, discNo: 0, genre: '', year: 0,
      coverPath: '',
      fileSize: stat.size,
      mtime: stat.mtimeMs,
      hasLyrics: false,
      createdAt: '', updatedAt: '',
    }
  }

  const { common, format } = meta
  return {
    id: md5(filePath),
    path: filePath,
    title: common.title || path.basename(filePath, path.extname(filePath)),
    artist: common.artist || '未知艺术家',
    album: common.album || '未知专辑',
    albumArtist: common.albumartist || '',
    duration: format.duration || 0,
    bitrate: format.bitrate || 0,
    sampleRate: format.sampleRate || 0,
    trackNo: common.track?.no || 0,
    discNo: common.disk?.no || 0,
    genre: (common.genre || []).join(', '),
    year: common.year || 0,
    coverPath: '',   // 封面单独处理
    fileSize: stat.size,
    mtime: stat.mtimeMs,
    hasLyrics: Boolean(common.lyrics?.length),
    createdAt: '', updatedAt: '',
  }
}
```

### 4.4 local:// 协议注册

`electron/main.js` 中注册（必须在 `app.whenReady()` 内）：

**Bug 预防：**

1. **协议必须在 `ready` 之后注册**：`app.whenReady()` 回调内
2. **路径解碼**：`decodeURIComponent` 处理中文/空格文件名
3. **Windows 兼容**：`local:///C:/path/file.mp3` → 去除开头的 `/`
4. **文件不存在**：返回 `{error: -6}`（`net::ERR_FILE_NOT_FOUND`），播放器收到后显示提示

```typescript
import { protocol } from 'electron'

app.whenReady().then(() => {
  protocol.registerFileProtocol('local', (request, callback) => {
    try {
      let filePath = decodeURIComponent(
        request.url.replace(/^local:\/\//, '')
      )
      // Windows: local:///C:/xxx → C:/xxx
      if (/^\/[a-zA-Z]:\//.test(filePath)) {
        filePath = filePath.slice(1)
      }
      filePath = path.normalize(filePath)

      if (!fs.existsSync(filePath)) {
        return callback({ error: -6 }) // ERR_FILE_NOT_FOUND
      }
      callback({ path: filePath })
    } catch (err) {
      callback({ error: -2 }) // ERR_FAILED
    }
  })
})
```

### 4.5 IPC 通道注册

```typescript
import { ipcMain, dialog } from 'electron'

export function registerLocalMusicIpc(scanner: IMusicScanner, db: IMetadataDB) {
  ipcMain.handle('local:scan', async (_event, dirPath: string) => {
    // 通过 event.sender.send 推送进度
    const sendProgress = (event: ScanEvent) => {
      _event.sender.send('local:scan-progress', event)
    }
    for await (const event of scanner.scanDir(dirPath)) {
      sendProgress(event)
      if (event.type === 'batch') {
        await db.upsertTracks(event.tracks)
      }
    }
    return { success: true }
  })

  ipcMain.handle('local:search', async (_event, query: string) => {
    return db.search(query)
  })

  ipcMain.handle('local:select-directory', async () => {
    const result = await dialog.showOpenDialog({
      properties: ['openDirectory']
    })
    return result.canceled ? null : result.filePaths[0]
  })
}

  artist: string
  album: string
  duration: number
  coverUrl: string  // base64 data URL
  source: 'local'
}

export const localMusicStore = reactive({
  tracks: [] as LocalTrack[],
  directories: [] as string[],
  scanning: false,
  progress: { current: 0, total: 0 },
  searchKeyword: '',

  get filteredTracks() {
    if (!this.searchKeyword) return this.tracks
    const kw = this.searchKeyword.toLowerCase()
    return this.tracks.filter(t =>
      t.title.toLowerCase().includes(kw) ||
      t.artist.toLowerCase().includes(kw) ||
      t.album.toLowerCase().includes(kw)
    )
  },

  get hasLocalSupport() {
    return platform.hasLocalMusicSupport
  },

  async scanAll() {
    if (!platform.localApi || this.scanning) return
    this.scanning = true
    this.progress = { current: 0, total: 0 }

    for (const dir of this.directories) {
      try {
        await platform.localApi.scan(dir)
      } catch (e) {
        console.error('[localMusic] scan failed:', dir, e)
      }
    }

    this.scanning = false
    await this.loadFromCache()
  },

  async loadFromCache() {
    if (!platform.localApi) return
    // 从 SQLite 加载最新数据（通过 IPC）
    const tracks = await platform.localApi.search('')
    this.tracks = tracks.map(this.formatTrack)
  },

  formatTrack(t: any): LocalTrack {
    return {
      id: t.id,
      path: t.path,
      title: t.title,
      artist: t.artist,
      album: t.album,
      duration: t.duration,
      coverUrl: '',  // 懒加载
      source: 'local',
    }
  },
})
```

## 六、潜在 Bug 与风险排查（全阶段适用）

### 6.1 文件系统相关

| Bug | 触发条件 | 影响 | 修复 |
|---|---|---|---|
| 中文/日文/韩文路径乱码 | Windows + CJK 文件名 | 播放失败 | 始终 `decodeURIComponent` + `path.normalize()` |
| 符号链接/硬链接 | 用户建立了目录链接 | 重复扫描/死循环 | `fs.realpathSync()` 解析真实路径，记录已访问的 realpath |
| 权限不足目录 | `/System`, `/proc` | 扫描中断 | `try/catch` 包裹 `fs.readdir`，失败跳过不中断 |
| 文件在扫描中被删除 | 用户实时操作 | 解析崩溃 | `try/catch` 包裹 `parseFile`，`ENOENT` 跳过 |
| 网络驱动器/NAS 掉线 | 局域网不稳定 | 扫描挂起 | 目录扫描加超时（每个目录 30s） |
| 超长路径（>260 字符） | Windows 深目录 | 解析失败 | 检测路径长度，跳过超长路径 |

### 6.2 数据库相关

| Bug | 触发条件 | 影响 | 修复 |
|---|---|---|---|
| 并发扫描冲突 | 用户快速点击扫描 | 数据不一致 | 扫描锁 + 串行化写队列 |
| SQLite 数据库损坏 | 强制退出/崩溃 | 全部数据丢失 | 定期 `PRAGMA integrity_check` + 自动备份 |
| WAL 文件膨胀 | 大量写入 | 磁盘占用 | 定期 `PRAGMA wal_checkpoint(TRUNCATE)` |
| 跨平台路径格式 | Windows `\` vs macOS `/` | 查不到数据 | 存储时统一 `path.normalize()`，查询时同样处理 |

### 6.3 扫描性能

| 场景 | 预估耗时 | 问题 |
|---|---|---|
| 1000 首首次扫描 | ~10s | 可接受 |
| 10000 首首次扫描 | ~90s | 必须流式推送进度 |
| 100000 首首次扫描 | ~15min | 必须分片处理，可能需要 Rust |
| SSD vs HDD | HDD 慢 3-5 倍 | 并行 worker 数减少 |
| 大文件（DSD/FLAC 192kHz）| 解析耗时 >10s | 单文件超时 30s 保护 |

### 6.4 内存相关

| 场景 | 风险 | 应对 |
|---|---|---|
| 10000+ 封面 base64 | 渲染进程内存暴增 | 懒加载：仅加载可见项的封面 |
| 超大歌词文件 | 解析卡顿 | 歌词文件大小限制 < 512KB |
| SQLite 全表查询 | 主进程内存压力 | 分页查询，每页 500 条 |
| IPC 传输大封面 | 渲染进程消息队列积压 | 封面存文件系统，传路径而非 base64 |

### 6.5 编码兼容性

| 场景 | 问题 | 修复 |
|---|---|---|
| LRC 文件 GBK 编码（中文系统常见）| 乱码 | 使用 `jschardet` 检测编码，`iconv-lite` 转 UTF-8 |
| ID3v2.3 标签 Latin-1 编码 | 中文乱码 | `music-metadata` 已自动处理，需验证 |
| FLAC 文件 UTF-8 编码 | 正常 | 无问题 |
| 文件名编码 | Windows 系统 ANSI | `fs.readdir` 返回 UTF-8，无需额外处理 |

```typescript
// LRC 编码检测与转换
import jschardet from 'jschardet'
import iconv from 'iconv-lite'

function decodeLrc(buffer: Buffer): string {
  const result = jschardet.detect(buffer)
  const encoding = result.encoding === 'GB2312' || result.encoding === 'GBK'
    ? 'gbk' : result.encoding || 'utf-8'
  return iconv.decode(buffer, encoding)
}
```

### 6.6 启动时序

| 场景 | 问题 | 修复 |
|---|---|---|
| `local://` 协议在 `ready` 前注册 | 崩溃 | 必须在 `app.whenReady()` 内 |
| 扫描在数据库初始化前触发 | 异常 | 数据库 init() 返回 Promise，扫描等待 |
| 渲染进程在 preload 加载前调用 localApi | undefined | 全部可选链 `?.()` |
| electron-rebuild 未执行 | `better-sqlite3` 加载失败 | 启动时 try/catch 检测，失败则弹出错误提示 |
