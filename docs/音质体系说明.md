# 音质体系说明

本文档统一说明项目内音质等级、VIP 分级、降级检测、各组件使用位置与持久化策略。

---

## 1. 音质等级总览

共 9 个音质等级，按质量从低到高排列：

| 显示标签 | API level | 最低比特率 (bps) | VIP 要求 |
|---|---|---|---|
| 标准 | `standard` | 128,000 | 免费 |
| 较高 | `higher` | 192,000 | 免费 |
| 极高(HQ) | `exhigh` | 320,000 | 免费 |
| 无损(SQ) | `lossless` | 800,000 | 黑胶VIP |
| Hi-Res | `hires` | 1,920,000 | 黑胶VIP |
| 高清臻音 | `jyeffect` | 1,920,000 | SVIP |
| 沉浸环绕声 | `sky` | 1,920,000 | SVIP |
| 杜比全景声 | `dolby` | 1,920,000 | SVIP |
| 超清母带 | `jymaster` | 1,920,000 | SVIP |

> `高清环绕声` 是 `高清臻音` 的别名，映射到同一个 API level (`jyeffect`)。

---

## 2. VIP 分级

VIP 权限是**逐级包含**的：黑胶 VIP 实际可用 SVIP 及以下全部音质，SVIP 覆盖黑胶 VIP 的全部能力。

| 级别 | 可用音质 | 对应 API level 集合 |
|---|---|---|
| 免费用户 | 标准、较高、极高(HQ) | `standard`, `higher`, `exhigh` |
| 黑胶VIP | 标准~超清母带（全部 9 种） | `standard` ~ `jymaster` |
| SVIP | 标准~超清母带（全部 9 种） | `standard` ~ `jymaster` |

> **实测结论**：黑胶 VIP 账号请求 SVIP 音质（高清臻音、沉浸环绕声、杜比全景声、超清母带）时，API 正常返回对应比特率的音源，不会被降级。API 的质量门控不是按账号 VIP 等级一刀切的，而是按歌曲粒度决定。

免费用户向 API 请求 VIP 音质时，系统强制降级为 `exhigh`（极高 HQ）。黑胶 VIP 和 SVIP 用户不做此限制。

## 3. 核心数据结构

### 3.1 `QUALITY_LEVELS` — 标签 → API level 映射

定义位置：`src/stores/player.ts`

```ts
const QUALITY_LEVELS: Record<string, string> = {
  '标准': 'standard',
  '较高': 'higher',
  '极高(HQ)': 'exhigh',
  '无损(SQ)': 'lossless',
  'Hi-Res': 'hires',
  '高清臻音': 'jyeffect',
  '高清环绕声': 'jyeffect',
  '沉浸环绕声': 'sky',
  '杜比全景声': 'dolby',
  '超清母带': 'jymaster',
};
```

使用函数 `toApiLevel(label)` 做转换。

### 3.2 `QUALITY_MIN_BR` — 降级检测阈值

定义位置：`src/stores/player.ts`

每个 API level 的最低比特率阈值。当 API 返回的 `br` 低于对应阈值时，判定为**静默降级**。

### 3.3 `VIP_ONLY_API_LEVELS` — 需要 VIP 的 API level 集合

定义位置：`src/stores/player.ts`

```ts
const VIP_ONLY_API_LEVELS = new Set([
  'lossless', 'hires', 'jyeffect', 'sky', 'dolby', 'jymaster',
]);
```

## 4. 降级检测机制

位置：`src/stores/player.ts` → `playTrack()`

流程：

1. 根据 `defaultQuality` 转换为 API level
2. **仅免费用户**：若 level 在 `VIP_ONLY_API_LEVELS` 中，强制设为 `exhigh`；黑胶 VIP 和 SVIP 用户跳过此检查
3. 请求 `/api/song/url/v1`
4. 拿到返回 `br` 后，与 `QUALITY_MIN_BR[level]` 比较
5. 低于阈值 → 设置 `currentQualityDowngraded = true`，填充 `qualityDowngradeInfo`
6. **不修改 `defaultQuality`**（用户偏好保留，下一首仍以偏好请求）
7. 通过 `PlayerBar.vue` 的 toast 提示用户当前音质被降级

关键字段（`playerStore`）：

- `currentQualityDowngraded: boolean` — 当前歌曲是否被降级
- `qualityDowngradeInfo: { from: string; to: string } | null` — 降级来源和实际音质

---

### 4.1 音源替换（unblock）的音质

当歌曲官方源不可播，且用户启用了音源替换（`uiStore.unblockEnabled`）时，系统会调用 `/unblock-api/match` 从第三方源（波点/酷狗/咪咕/QQ 等）匹配替代音源。

音源替换返回的比特率由第三方源决定，**通常为 320kbps 级别（对应极高 HQ）**，不区分用户 VIP 等级。匹配成功后 `currentQualityBr` 会被设为替换源的 `br` 值，但不影响 `defaultQuality`（用户偏好的音质保持）。

> 注意：音源替换的音质与用户设置的 `defaultQuality` 无关。即使默认音质设为超清母带，替换源返回的仍是第三方源自身提供的品质。

## 7. 持久化

- `defaultQuality` 持久化到 localStorage key `gm_quality_v1`
- 通过 `playerStore.setDefaultQuality()` 写入
- 恢复时通过 `playerStore.hydrate()` 读取，优先取 `gm_quality_v1`，兜底到 `gm_player_state_v1` 中的值

## 8. `formatQualityBr()` — br → 标签还原

位置：`src/stores/player.ts`

```ts
function formatQualityBr(br: number): string {
  if (br >= 1920000) return 'Hi-Res';
  if (br >= 999000) return '无损(SQ)';
  if (br >= 320000) return '极高(HQ)';
  if (br >= 192000) return '较高';
  if (br >= 128000) return '标准';
  return '';
}
```

用于降级检测后把 API 返回的比特率还原为显示标签。

## 10. 维护提示

- 新增音质时，需同步更新：`QUALITY_LEVELS`、`QUALITY_MIN_BR`、`VIP_ONLY_API_LEVELS`、`qualityOptions`（PlayerBar）、`optionVipLabels`（SettingsPage）
- VIP 标签文案（`黑胶VIP` / `SVIP`）在 PlayerBar 和 SettingsPage 各定义一份，修改时两边保持一致
- 降级逻辑不修改 `defaultQuality`，确保切歌后自动恢复偏好的音质请求
