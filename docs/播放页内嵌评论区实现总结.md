# 播放页内嵌评论区实现总结

## 功能概述

在全屏播放器（PlayerExpanded）中嵌入评论区，点击底部控制栏的评论区图标按钮，评论区覆盖封面/歌词区域滑入显示。再次点击切换回正常播放模式。评论区底部与底部控制栏顶部共用同一条水平线，不产生显示重叠。

## 最终实现方案

### 布局架构

```
expanded-panel (grid: auto 1fr auto, gap: 0)
├── panel-head          (row 1, auto, bottom margin)
├── panel-body          (row 2, 1fr, overflow: hidden, grid: 40% 60%)
│   ├── cover-hidden-head   (v-show="!showComments")
│   ├── left-zone           (v-show="!showComments && showLeftZone")
│   ├── LyricsPanel         (v-show="!showComments")
│   └── comments-overlay    (v-show="showComments", grid-column:1/-1, height: 100%)
└── bottom-console          (row 3, auto, position: static, width: 100%)
```

### 关键 CSS

```css
/* 三行 grid + 无间隙 */
.expanded-panel {
  display: grid;
  grid-template-rows: auto 1fr auto;
  gap: 0;
}

/* 评论区覆盖双列 + 填满高度 + 内部滚动 */
.comments-overlay {
  grid-column: 1 / -1;
  align-self: stretch;
  min-height: 0;
  overflow-y: auto;
  display: flex;
  flex-direction: column;
}

/* 底部控制栏改为普通 grid 项，不再 fixed */
.bottom-console {
  position: static;  /* 移除原来的 position: fixed */
}
```

### v-show 切换

- 评论区使用 `v-show="showComments"`（始终在 DOM，不销毁）
- 正常内容（封面/歌词）使用 `v-show="!showComments"`（始终挂载）
- `v-show` 改为 `display: none`，不触发组件销毁重建
- 歌词组件不会因 `v-if` 卸载导致初始化失败

## 踩坑记录

### 坑1：CSS Grid 子项全部 display:none 时容器塌缩

**表现**：panel-body 高度为 0，评论区不显示。

**原因**：当 panel-body 内所有子节点都被 `v-show` 设为 `display: none` 时，CSS Grid 容器会认为没有可见的 grid 子项，导致容器高度塌缩为 0。即使其他子项随后变为可见，grid 容器也不恢复高度。

**错误尝试**：
- 尝试 `height: 100%` → 不生效，因为 grid 容器的父级来自 grid 轨道
- 尝试 `min-height: 100%` → 同样不生效
- 尝试 `grid-template-rows: 1fr` → 不解决塌缩问题

**正确做法**：确保 grid 容器始终有至少一个可见子项。通过 `v-show` 而非 `v-if` 控制两个状态，保持至少一个子项可见。

### 坑2：v-if 导致歌词空白

**表现**：使用 `v-if/v-else` 分离正常模式和评论区模式后，歌词面板显示空白。

**原因**：`LyricsPanel` 组件在初始化时（`onMounted`）会请求歌词数据和绑定事件。使用 `v-if` 切换时，组件被销毁后重建，破坏了内部状态（如歌词解析器、AMLL 渲染器初始化序列）。

**错误尝试**：
- 将评论区放在 panel-body 外作为兄弟元素 → 布局错位
- 调整生命周期顺序 → 无效

**正确做法**：正常内容（含 LyricsPanel）始终使用 `v-show="!showComments"` 保持挂载，评论区使用 `v-show="showComments"` 同样保持挂载。两者都在 panel-body 内作为直接子节点。

### 坑3：评论区内容延伸到底部控制栏后面

**表现**：评论区滚动到最后，评论内容跑到了底部控制栏后方，视觉上被截断。

**原因**：底部控制栏使用 `position: fixed; bottom: 0`，悬浮覆盖在 panel-body 之上。panel-body 的高度包含整个视口（越过控制栏位置），所以 panel-body 内的评论区也延伸到了控制栏后方。

**错误尝试**：
- 加 `padding-bottom` 推高评论区底部 → 内容不跑了但容器仍在控制栏后面
- 加 `position: relative` + `z-index` 叠层 → 无法解决根本问题

**正确做法**：将底部控制栏从 `position: fixed` 改为 `position: static` 正常 grid 项。`expanded-panel` grid 改为 `auto 1fr auto`（三行：头部/内容/控制栏），panel-body 自然在控制栏上方结束。

### 坑4：`gap: var(--space-3)` 创建了 12px 间隙

**表现**：评论区底部和底部控制栏顶部之间有 12px 空隙，无法对齐。

**原因**：`expanded-panel` 的 `gap: var(--space-3)` 在 grid 行之间都添加 12px 间距。

**正确做法**：评论区模式下将 `gap` 设为 0。最终方案直接统一设为 `gap: 0`，用 panel-head 的内边距提供正常模式的视觉间距。

### 坑5：评论区只占左侧 40% 列

**表现**：评论区内容只显示在 panel-body 的左半部分（40% 列），右侧空白。

**原因**：panel-body grid 使用 `grid-template-columns: 40% 60%`（封面列 + 歌词列）。评论区作为 grid 子项只占一列（默认列 1）。

**正确做法**：评论区添加 `grid-column: 1 / -1` 跨两列。

### 坑6：评论区无法滚动

**表现**：评论区内容超过 panel-body 高度时不出现滚动条，内容溢出隐藏。

**原因**：评论区作为 grid 子项，默认高度为内容高度（`align-items: start` 不拉伸），不触发 `overflow-y: auto`。

**正确做法**：评论区添加 `align-self: stretch` 拉伸填满 grid 行 + `min-height: 0` 允许收缩 + `overflow-y: auto` 启用滚动。

---

## 关于底部控制栏 position:fixed 的取舍

原始设计中，底部控制栏使用 `position: fixed` 悬浮覆盖在 panel-body 底部。这在正常模式下提供更好的视觉效果（控制栏覆盖歌曲封面底部）。但评论区打开时，这种覆盖导致评论区内容延伸到控制栏后方。

**权衡结果**：将底部控制栏改为 `position: static`（正常 grid 项），彻底解决评论区与控制栏的边界问题。正常模式的视觉效果略有变化（控制栏不再覆盖封面底部），但换来评论区模式的无缝对齐。

## 验证清单

1. 全屏播放器 → 底部控制栏评论区按钮（消息气泡图标）
2. 点击按钮 → 评论区显示，占据完整内容区（左右两列）
3. 评论区顶部显示歌曲封面(80px)、歌手（可点击跳转）、专辑
4. 评论区可正常发送/回复/点赞/删除评论
5. 评论区内容可滚动，最后一条评论停在底部控制栏上方
6. 评论区底部 = 底部控制栏顶部（0 间隙）
7. 再次点击评论区按钮 → 评论区关闭，封面/歌词恢复
8. 右侧控制栏（设置/延迟/翻译等）隐藏
9. 切换回正常模式 → 歌词正常显示，不空白
