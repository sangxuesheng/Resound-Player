### 核心原则

- **仅桌面端（Electron）**：Web 端不提供本地歌曲功能
- **platform.isDesktop 守卫**：所有本地入口、路由、Store 方法都需加守卫
- **播放方案**：通过 IPC (`local:read-file`) 读取文件内容，在渲染进程创建 `Blob` + `URL.createObjectURL`，避免 `local://` 协议的 CORS 问题

## 3. 存储方案

采用 **JSON 文件存储**（`local-music.json` 位于 Electron `userData` 目录），完全避免了原生模块和 WASM 依赖问题。

### 演进历史

| 方案 | 版本 | 问题 | 结论 |
|-----|------|------|------|
| `better-sqlite3` | 初始 | Electron 42 的 V8 API 变更导致 `electron-rebuild` 编译失败 | ❌ 放弃 |
| `sql.js` (WASM SQLite) | 方案二 | WASM 在 Electron 主进程加载失败，`initSqlJs()` 不工作 | ❌ 放弃 |
| **JSON 文件** | 当前 | 纯 Node.js `fs` 读写，零外部依赖，稳定可靠 | ✅ 采用 |

### 数据结构

```json
{
  "tracks": [{ "id": "md5(path)", "path": "...", "title": "...", "artist": "...", "album": "...", "duration": 0, ... }],
  "playlists": [{ "id": "...", "name": "...", "description": "...", "createdAt": "...", "updatedAt": "..." }],
  "playlistTracks": [{ "playlistId": "...", "trackId": "...", "sortOrder": 0, "addedAt": "..." }],
  "scanDirs": [{ "id": 1, "path": "...", "label": "", "lastScan": "" }]
}
```

### 读写机制

- **写入**：所有写操作通过 `serializedWrite(fn)` 串行化，完成后调用 `persist()` 写入磁盘
- **读取**：直接读取 `this.data.tracks`（内存中），仅在 `init()` 时从磁盘加载
- **搜索**：JavaScript 字符串匹配（`toLowerCase().includes()`），降级到 `filter` 而非 FTS5

## 4. IPC 三层链路

所有本地音乐操作都经过 **DB → IPC → Preload → Platform Type → Store → View** 六层：

```
LocalMusicDB.js           ← JSON 文件查询
    ↓
localMusicIpc.js          ← ipcMain.handle('local:xxx', ...)
    ↓
preload.js               ← contextBridge.exposeInMainWorld('localApi', {...})
    ↓
src/utils/platform.ts    ← LocalApi 接口类型定义
    ↓
src/stores/localMusic.ts ← 响应式 Store，调用 platform.localApi.xxx()
    ↓
src/views/Local*.vue     ← UI 组件读写 Store
```

### 21 个 IPC 通道一览

| 通道 | 用途 | 依赖 |
|---|---|---|
| `select-directory` | 选择扫描目录 | 无（模块级注册） |
| `local:scan` | 扫描目录 | Scanner + DB |
| `local:search` | 搜索曲库 | DB (字符串匹配) |
| `local:get-all` | 获取全部歌曲 | DB |
| `local:track-count` | 获取歌曲总数 | DB |
| `local:get-lyric` | 读取歌词文件 | fs |
| `local:get-cover` | 获取封面(带缓存) | CoverCache |
| `local:read-file` | 读取文件 | fs |
| `local:playlist-create` | 创建歌单 | DB |
| `local:playlist-list` | 歌单列表 | DB |
| `local:playlist-get` | 歌单详情 | DB |
| `local:playlist-delete` | 删除歌单 | DB |

### 添加新 IPC 的步骤

1. **DB 层** (`LocalMusicDB.js`) — 添加查询方法
2. **IPC 层** (`localMusicIpc.js`) — 注册 `ipcMain.handle`
3. **Preload** (`preload.js`) — `contextBridge` 暴露 API
4. **类型** (`platform.ts`) — `LocalApi` 接口添加签名
5. **Store** (`localMusic.ts`) — 添加调用方法
6. **重启 Electron**

## 5. 平台检测与守卫

### `platform.isDesktop` 检测逻辑

```typescript
// src/utils/platform.ts
get isDesktop(): boolean {
    if (typeof window === 'undefined') return false
    // Electron preload 注入 appEnv
    if (!!(window as any).appEnv?.isDesktop) return true
    // 降级：UA 检测
    return typeof navigator !== 'undefined' && navigator.userAgent.includes('Electron')
}
```

### 前置条件：preload 脚本

`electron/preload.js` 必须使用 **CommonJS** (`require`) 而非 ESM (`import`)：

```js
// ✅ Electron 沙盒环境兼容
const { contextBridge, ipcRenderer } = require('electron');

// ❌ 不支持（Sandbox 模式下 SyntaxError）
import { contextBridge, ipcRenderer } from 'electron';
```

### 守卫应用位置

| 位置 | 守卫方式 |
|---|---|
| `App.vue` 路由 | `v-if="activePage === 'local-xxx' && platform.isDesktop"` |
| `Sidebar.vue` 菜单 | `...(platform.isDesktop ? [{...}] : [])` |
| Store 方法 | `if (!platform.localApi) return` |
| SettingsPage 设置项 | `!platform.isDesktop` 时从 `currentGroups` 过滤 |
| `apiBaseUrl` | Dev 模式返回 `/api`（走 Vite 代理），打包后返回直连 URL |

### 视图页面

本地音乐页面统一由 `LocalMusicHub.vue` 作为入口。页面结构从上到下依次为：
1. **页面标题** — "本地音乐"
2. **公共统计栏 + 标签栏** — 统计卡片（歌曲/歌手/专辑/总时长/总大小）与 6 个标签按钮左右并排
3. **内容区** — 左栏（歌手/专辑/目录索引，条件渲染）+ 右栏（子视图内容）

子页面不自带分栏结构，统一由 Hub 管理。

| Hub 标签 | 子组件 | 左栏 | 右栏内容 |

| 专辑 | `LocalAlbumsPage` | 专辑索引列表（点击过滤） | 专辑歌曲列表（`selectedAlbumTracks`） |
| 目录 | `LocalFoldersPage` | 可折叠目录树（点击过滤） | 目录歌曲列表（`selectedFolderTracks`） |

| 独立路由 | 组件 | 布局 |
|---|---|---|
| `local-playlist-detail` | `LocalPlaylistDetailPage` | 单列：歌曲列表 + 操作按钮 |

### 跨页面导航

本地页面统一由 `LocalMusicHub.vue` 承载，通过 `activeView` 在 6 个标签间切换。子页面间导航使用 `CustomEvent('local-navigate', { detail: { page: 'local-xxx' } })`，App.vue 监听并处理：`activePage.value = detail.page`（用于 "定位到目录" 等场景）。

## 7. 常见 Bug 与修复

### 运行时错误

| 错误 | 原因 | 修复 |
|---|---|---|
| `ERR_MODULE_NOT_FOUND: Cannot find module ...LocalMusicDB.js` | TypeScript 文件未编译为 JS | 用 esbuild 编译 `.ts` → `.js` |
| `Cannot use import statement outside a module` (preload 脚本) | preload 沙盒不支持 ESM | 改用 CommonJS `require()` |
| `No handler registered for 'select-directory'` | IPC handler 未在模块顶层注册 | 移到 `localMusicIpc.ts` 模块级 |
| `Attempted to register a second handler for 'select-directory'` | handler 被注册了两次 | 只保留一处注册 |
| `TypeError: SQLite3 can only bind numbers, strings, bigints, buffers, and null` | 布尔值传入 SQLite | `Boolean()` → `(... ? 1 : 0)` |
| `TypeError: fs.existsSync is not a function` | `fs/promises` 无 `existsSync` | 改用 `import { existsSync } from 'fs'` |
| CORS: `Access-Control-Allow-Origin: *` with credentials | Dev 模式直连 API 端口 | `apiBaseUrl` 返回 `/api` 走 Vite 代理 |
| `[Vue warn]: Failed to resolve component: LocalMusicHub` | `App.vue` 中使用了 `<LocalMusicHub>` 但未 import | 添加 `import LocalMusicHub from './components/LocalMusicHub.vue'` |

### Store / Type 错误

| 错误 | 原因 | 修复 |
|---|---|---|
| 本地歌曲被识别为 `source: 'song'` 而非 `'local'` | `formatTrack` 中未处理 `local` | 添加 `raw.source === 'local' ? 'local'` 分支 |
| 播放时 `id: NaN` | md5 字符串 ID 被当数字处理 | 保留 `source: 'local'` 即可走 local:// 协议 |
| `hasLocalSupport` 返回 `false` | `isDesktop` 为真但 `window.localApi` 不存在 | preload 脚本加载失败 → 改用 `require` |
| `Cannot set properties of undefined (setting 'selectedFolderPath')` | 将 reactive 方法直接作为回调传递导致 `this` 丢失 | 模板中使用箭头函数包裹：`@select="(p) => store.method(p)"` |
| `directories` 数组始终为空 | `loadDirectories()` 从未被调用 | 在需要的组件 `onMounted` 中调用 `localMusicStore.loadDirectories()` |

### 写入队列时序

| Bug | 原因 | 修复 |
|---|---|---|
| `clearAll()` 后刷新数据"复活" | `_dataCleared` 护城河掩盖了 DB 删除失败；`clearAllTracks` 跳过 `serializedWrite` 导致写冲突 | 移除 `_dataCleared`；`clearAll()` 末尾调用 `loadTracks()` 确保 store 与 DB 同步；`clearAllTracks` 使用 `serializedWrite` |
| `removeTracks` 删除未生效 | `serializedWrite` 队列导致 `loadTracks()` 在 SQL 执行前读取旧数据 | 写操作改直接同步事务（`this.db.transaction`），删除立即生效后再 `loadTracks()` |

### CSS 布局

| Bug | 原因 | 修复 |
|---|---|---|
| 歌曲页歌手名覆盖歌名 | 网格固定 8 列但 checkbox `v-if` 条件渲染，非选中模式时只剩 7 个元素，内容左移一列 | 基柱使用 7 列网格（默认），`.has-checkbox` 变体使用 8 列网格，动态切换 |

### Store 持久化

| Bug | 原因 | 修复 |
|---|---|---|
| 移除目录后自动恢复 | `saveDirectories()` 先读 localStorage 旧数据合并到 `this.directories`，把刚删的目录又加回来 | `removeDir()` 和 `clearAll()` 中先手动从 localStorage 移除/清空该目录，再调 `saveDirectories()` |

### 封面加载

| Bug | 原因 | 修复 |
|---|---|---|
| 首次扫描后封面不显示，需刷新才出现 | `lazyLoadCovers()` 使用 `setTimeout(loadNext, 30)` 逐首加载，100 首歌需 3 秒；且首次渲染时 `coverUrl` 全为空 | ① `loadTracks()` 中 `coverUrl` 从 `_coverCache` 预填充（缓存命中即显示）② `lazyLoadCovers()` 改为 `Promise.all` 全量并行，无分批延迟 |

### 最近添加不刷新

| Bug | 原因 | 修复 |
|---|---|---|
| 扫描完成后「最近添加」列表不更新，需切标签页 | `recent` 列表仅在 `onMounted` 加载一次，扫描后未自动刷新 | 在 `LocalStatsPage.vue` 添加 `watch(() => localMusicStore.tracks.length)` 自动重新加载 |

### Electron 兼容性

| Bug | 原因 | 修复 |
|---|---|---|
| `npm run dev:desktop` 没有弹出桌面窗口 | Electron 42.1.0 的 pnpm store 缓存中二进制文件损坏（extract-zip symlink 冲突） | 清除 pnpm store 中 `electron@42.1.0` 缓存后重新 `pnpm install` |
| `No handler registered for 'local:*'` IPC 全部报错 | `better-sqlite3` 原生模块不兼容 Electron 42 的 V8 API，编译失败导致 `registerLocalMusicIpc` 未执行 | 改用 JSON 文件存储，移除 `better-sqlite3` 和 `sql.js` 依赖 |
| `SyntaxError: Unexpected strict mode reserved word` | `.js` 文件中混入了 TypeScript 语法（`interface`、`private`、类型标注） | 重写 `.js` 文件为纯 JavaScript |
| `Failed to construct 'URL': Invalid URL`（Vite 开发模式） | Vite 模块热替换系统在解析动态 `import()` 路径时的内部错误，仅开发模式出现 | 不影响功能，已添加全局 `unhandledrejection` 日志捕获 |

### @tanstack/vue-virtual v3 兼容性

| Bug | 原因 | 修复 |
|---|---|---|
| `getVirtualItems is not a function` | v3 的 `useVirtualizer` 返回 `ShallowRef<Virtualizer>` 而非直接返回实例 | 加 `.value` 访问 `virtualizer.value.getVirtualItems()` |
| `getTotalSize()` 始终返回 0 | v3 的 `getMeasurements()` memo 缓存永不失效，`getSize()` 依赖 `scrollRect` 形成死锁 | 移除 `@tanstack/vue-virtual` 依赖，改用自定义虚拟滚动实现 |
| 自定义虚拟滚动无法显示内容 | v3 的 `getMeasurements()` 内部 `initialMeasurementsCache` 未设置时返回空数组 | 自行实现定高虚拟滚动（38px/行），`ResizeObserver` 测量容器高度 + `scrollTop` 计算可见范围 |

### 本地播放 CORS

| Bug | 原因 | 修复 |
|---|---|---|
| 本地歌曲播放时报 CORS: `local://` 被拦截 | 开发模式页面从 `http://localhost:5173` 加载，`local://` 协议跨源被浏览器拦截 | 通过 IPC (`local:read-file`) 读取文件 ArrayBuffer，创建 `Blob` + `URL.createObjectURL` |

### 扫描进度

| Bug | 原因 | 修复 |
|---|---|---|
| 扫描大目录（27000+ 文件）时 UI 卡在"正在扫描 0/0" | `collectFiles` 遍历目录树期间没有 yield 任何进度事件，持续数十秒 | 将 `collectFiles` 放到后台运行，轮询 `state.found` 计数器，每 300ms yield 一次进度 |

## 9. 新增功能

## 8. 新增功能的操作步骤

### 新增 IPC 通道

以新增 `local:xxx` 为例：

```
1. electron/services/db/LocalMusicDB.js
   → 添加查询方法 getXxx()

2. electron/services/ipc/localMusicIpc.js
   → ipcMain.handle('local:xxx', async () => db.getXxx())

3. electron/preload.js
   → xxx: () => ipcRenderer.invoke('local:xxx'),

4. src/utils/platform.ts
   → LocalApi 接口添加: xxx(): Promise<...>

5. src/stores/localMusic.ts
   → 添加 async method: this.platform.localApi.xxx()

6. 重启 Electron
   → main.js 改动需完全重启，Vite HMR 不影响主进程
```

### 新增页面

```
1. 创建 src/views/LocalMyNewPage.vue（遵循 .local-page shell 模式）
2. src/App.vue 中:
   - import LocalMyNewPage
   - 添加 v-else-if="activePage === 'local-my-new' && platform.isDesktop"
3. src/components/Sidebar.vue 中:
   - 添加 { key: 'local-my-new', label: '...', icon: ... }
4. 如果涉及 IPC, 执行上面的新增 IPC 步骤
5. lint + build 验证
```

### 新增 DB 字段/索引

```json
// JSON 存储时，直接在对象上添加字段即可
{
  "tracks": [{ "id": "...", "path": "...", "newField": "" }],
}
```

无需 SQL 迁移、无需编译。
